# v1.5 / 模型页宽屏布局策略

## 版本号和分配依据

- 版本号：`v1.5`
- 分配依据：`update_log.md` 最新正式版本为 `v1.4 / Mac iPad Composer输入焦点与辅助语义`，本轮继续 Agent X 的 Mac/iPad 体验审计主线，递增为 `v1.5`。

## 背景

当前主界面已经支持 iPhone、iPad 和 Mac Catalyst 宽屏双栏，并完成工作区快捷键、系统菜单、侧栏选择语义和 composer 输入焦点。模型页内部仍用 `size.width > size.height && size.width > 620` 判断是否显示内部双栏，这会让一些足够宽但高度更大的 iPad/Mac 窗口退回单栏，降低模型部署控制台的扫描效率。

本轮目标是只优化模型页内部布局策略，不改全局 `WorkspaceLayoutMode`，不改模型状态、artifact 校验、下载/导入/卸载和 runtime 边界。

## 目标

1. 在 `ModelLibraryView` 内抽出可测试的模型页布局策略，例如 `ModelLibraryLayoutMode`。
2. 让模型页在足够宽的容器中进入内部双栏：
   - 左侧：模型选择、部署按钮、本地 artifact 操作。
   - 右侧：模型摘要、参数、性能和建议。
3. 窄 iPhone、窄 split view 或不足以并列显示的容器继续保持单栏。
4. 新增纯逻辑 XCTest 锁住窄屏、宽横屏、宽竖向 iPad/Mac pane 和窄 Mac split 的布局结果。
5. 同步 README、测试规范、核心流程文档、流程图和更新日志。

## 非目标

- 不修改 `WorkspaceLayoutMode` 的主界面断点。
- 不修改 v1.1 快捷键、v1.2 `工作区` CommandMenu、v1.3 选择语义、v1.4 composer 焦点策略。
- 不改模型导入、扫描、卸载、部署状态、artifact validator 或 runtime plan。
- 不新增原生 macOS target。
- 不下载模型权重，不接入真实 Gemma 推理，不接入外部服务。

## 当前架构依据

- `md/flow/flow.md`：`ContentView` 管 UI 工作区，`ModelCatalog` 管模型选择和 artifact 状态；UI 不应绕过状态层直接改核心状态。
- `LocalGemma/ContentView.swift`：
  - `ModelLibraryView.deploymentContent(size:)` 当前用局部 `isLandscape = size.width > size.height && size.width > 620` 决定模型页内部单栏/双栏。
  - 双栏分支仅重排现有 `ModelSelectorPanel`、`DeploymentPowerButton`、`ArtifactActionPanel` 和模型详情面板。
- `LocalGemmaTests/LocalGemmaTests.swift` 已有 `WorkspaceLayoutMode` 主界面断点测试；本轮应新增模型页内部布局策略测试，避免混淆主界面布局。

## 实现步骤

1. Agent B 开始前同步远端：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short --branch`
2. 在 `LocalGemma/ContentView.swift` 中新增模型页内部布局策略：
   - 可以是 `enum ModelLibraryLayoutMode { case singleColumn, twoColumn }`。
   - 提供 `static func resolve(for size: CGSize) -> ModelLibraryLayoutMode`。
   - 提供左侧栏宽度 helper，保证宽屏双栏有稳定宽度区间。
3. 将 `deploymentContent(size:)` 的 `isLandscape` 替换为新策略：
   - `twoColumn` 时使用现有 `HStack` 双栏布局。
   - `singleColumn` 时保持现有顺序。
4. 新增 XCTest，例如 `testModelLibraryLayoutModeSupportsWideModelWorkflows`：
   - iPhone 390x844 -> singleColumn。
   - 窄 Mac split 620x900 或类似 -> singleColumn。
   - iPad/Mac 宽竖向 pane 820x1180 或类似 -> twoColumn。
   - 横向宽窗口 1000x760 或 1180x820 -> twoColumn。
   - 左侧栏宽度在合理 min/max 之间。
5. 更新文档：
   - `README.md` 当前范围、运行方式/模型页说明、已完成验证。
   - `md/test/test.md` 当前测试数量和覆盖范围。
   - `md/flow/flow.md` UI 工作区流和核心状态对象。
   - `md/flow/flowchart.md` UI 布局图。
   - `update_log.md` 追加 v1.5 记录。
