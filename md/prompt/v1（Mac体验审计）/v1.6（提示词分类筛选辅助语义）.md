# v1.6 / 提示词分类筛选辅助语义

## 版本号与分配依据

- 版本号：`v1.6`
- 分配依据：`update_log.md` 当前最新正式版本为 `v1.5 / 模型页宽屏布局策略`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增到 `v1.6`。

## 背景

当前 `PromptTemplatesWorkspace` 已支持按“全部、部署、隐私、性能、写作、产品、排障”筛选提示词模板，但 `PromptCategorySelector` 里的分类 chip 主要依赖视觉高亮表达选中状态。v1.3 已为工作区和会话选择建立 `SelectionAccessibilityMetadata`，v1.4 已为 composer 输入建立 `ComposerInputMetadata`；提示词分类筛选仍缺少同等级的稳定 label/value/hint/input labels 和可测试选中语义。

本轮目标服务于 Mac Catalyst 和 iPad 外接键盘/辅助技术体验：让 VoiceOver、Voice Control 和键盘用户更清楚地理解当前筛选范围，并能用稳定语音命令触发筛选。

## 目标

1. 为提示词分类筛选 chip 增加可测试的辅助语义元数据。
2. 将“全部”和每个 `PromptTemplateCategory` 的筛选按钮暴露为清晰 label、value、hint 和 Voice Control input labels。
3. 当前选中的分类 chip 使用 `.isSelected` trait，未选中的 chip 不添加该 trait。
4. 新增纯 XCTest 锁住元数据文本、分类全集、选中/未选中 value 和 input labels，不依赖 SwiftUI UI runtime。
5. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md` 中的当前能力、测试数量和验证记录。

## 非目标

- 不改变 `PromptTemplateLibrary` 的模板内容、分类枚举或筛选业务结果。
- 不改全局布局断点、模型页布局、会话状态、composer 焦点策略、command menu 或真实 runtime。
- 不创建原生 macOS target。
- 不下载模型权重，不接入云端推理，不绕过 artifact `verified` 门禁。
- 不做 `ContentView.swift` 大拆分或无关视觉重设。

## 当前架构依据

- `md/flow/flow.md`：提示词页是 `ContentView` 的四个 workspace 之一，负责模板筛选、填入和发送；业务状态仍由 `InferenceEngine` 处理。
- `LocalGemma/ContentView.swift`：
  - `SelectionAccessibilityMetadata` 和 `ComposerInputMetadata` 已作为 UI 层纯元数据 helper，可被 XCTest 直接验证。
  - `PromptTemplatesWorkspace` 持有 `selectedCategory`，并通过 `PromptTemplateLibrary.templates(in:)` 获取筛选结果。
  - `PromptCategorySelector` 当前渲染横向 chip，视觉上区分选中/未选中。
- `LocalGemma/AppState.swift`：
  - `PromptTemplateCategory.allCases` 定义 6 个分类和中文标题。
  - `PromptTemplateLibrary.templates(in:)` 保持筛选 source of truth。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - 现有 `testPromptTemplateLibraryProvidesMultipleCategories` 覆盖模板分类业务。
  - 现有 `testSelectionAccessibilityMetadataDescribesWorkspaceAndSessions`、`testComposerInputMetadataAndFocusPolicyDescribeEntryPoints` 是本轮纯元数据测试风格参考。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 添加 `PromptCategoryAccessibilityMetadata`，建议放在 `SelectionAccessibilityMetadata` 附近。
2. 元数据至少包含：
   - `allCategoryTitle = "全部"`
   - `allCategoryInputLabels`
   - `label(for:)` 或等价方法，返回“筛选提示词 全部 / 筛选提示词 部署”等清晰文案。
   - `value(isSelected:)`，返回“当前筛选”或“未选中”。
   - `hint(for:)`，说明会显示全部模板或对应分类模板。
   - `inputLabels(for:)`，提供 Voice Control 可用短语，例如“全部提示词”“筛选部署”“部署提示词”。
3. 修改 `PromptCategorySelector.categoryButton`：
   - 继续使用现有视觉样式和按钮动作。
   - 增加 `.accessibilityLabel(...)`、`.accessibilityValue(...)`、`.accessibilityHint(...)`、`.accessibilityInputLabels(...)`。
   - 对选中 chip 增加 `.accessibilityAddTraits(.isSelected)`。
   - 不改变按钮业务 action。
4. 新增 XCTest，例如 `testPromptCategoryAccessibilityMetadataDescribesFilterSelectionAndInputLabels`：
   - 验证 “全部” label/hint/input labels/value。
   - 验证 `PromptTemplateCategory.allCases` 的中文标题集合仍为 `["部署", "隐私", "性能", "写作", "产品", "排障"]`。
   - 验证某个分类（建议 `.privacy` 或 `.deployment`）的 label/hint/input labels。
   - 验证选中/未选中 value。
5. 同步文档：
   - `AGENTS.md`：核心架构边界加入提示词筛选辅助语义需要测试锁住。
   - `md/flow/flow.md`：UI 工作区和铁律加入提示词分类筛选辅助语义。
   - `md/flow/flowchart.md`：UI 布局图中提示词页节点加入“分类筛选辅助语义”。
   - `md/test/test.md`：测试数量从 39 更新到 40，覆盖范围加入提示词分类筛选辅助语义。
   - `README.md`：当前范围、运行说明、模型状态和验证章节同步 v1.6。
   - `update_log.md`：追加 v1.6 记录，写明本地轻量检查命令与结果。

## 风险与禁止项

- 风险：辅助语义文案过长会影响 VoiceOver 可扫读性；应保持 label 短、hint 解释动作、input labels 提供短语。
- 风险：SwiftUI `.accessibilityAddTraits` 条件传空数组的写法应沿用项目现有模式，避免引入平台差异。
- 禁止下载模型权重、云端推理、绕过 verified 门禁、改真实 runtime、引入新第三方依赖、扩大到无关视觉重构。
- 禁止伪造本地或云端测试结果；没有跑的测试必须明确说明。
