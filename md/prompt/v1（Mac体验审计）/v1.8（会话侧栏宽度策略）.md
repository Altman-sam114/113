# v1.8 / 会话侧栏宽度策略

## 版本号与分配依据

- 版本号：`v1.8`
- 分配依据：`update_log.md` 当前最新正式版本为 `v1.7 / 会话命令菜单发现层`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增到 `v1.8`。

## 背景

项目已经为主工作区建立 `WorkspaceLayoutMode`，为模型页内部宽屏布局建立 `ModelLibraryLayoutMode`。但推理页中大屏会话列表的宽度仍直接写在 `ChatWorkspace` 里：

```swift
.frame(width: min(max(proxy.size.width * 0.28, 240), 310))
```

这会让 Mac Catalyst 和 iPad 大屏会话栏的宽度策略缺少独立测试，后续调整时容易破坏会话列表和输入区的比例。本轮目标是只抽取这条布局策略，保持当前视觉行为不变，并用纯 XCTest 锁住宽度边界。

## 目标

1. 新增可测试的会话侧栏宽度策略类型，例如 `SessionSidebarLayoutPolicy`。
2. 保持现有行为：会话侧栏宽度仍按容器宽度 28% 计算，并限制在 240 到 310 之间。
3. 窄屏或非侧栏布局不启用会话侧栏宽度，策略应能表达 portrait/单栏下宽度为 0 或不使用侧栏。
4. `ChatWorkspace` 使用新策略替代内联 `min(max(...))` 公式。
5. 新增纯 XCTest 锁住 iPhone 窄屏、iPad regular、Mac Catalyst/桌面宽窗口下的会话侧栏宽度。
6. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md` 中的当前能力、测试数量和验证记录。

## 非目标

- 不改变 `InferenceEngine` 会话创建、切换、删除、导出或流式输出逻辑。
- 不改变 v1.7 的 `会话` CommandMenu、focused action route 或快捷键。
- 不改变主工作区 `WorkspaceLayoutMode` 断点或模型页 `ModelLibraryLayoutMode` 断点。
- 不做全局视觉重设，不新增 UI Test target。
- 不创建原生 macOS target。
- 不下载模型权重，不接入云端推理，不绕过 artifact `verified` 门禁。

## 当前架构依据

- `md/flow/flow.md`：当前 UI 工作区已经由 `WorkspaceLayoutMode` 控制单栏、compact 双栏和 regular 大屏双栏；模型页内部由 `ModelLibraryLayoutMode` 控制单栏/双栏。
- `LocalGemma/ContentView.swift`：
  - `WorkspaceLayoutMode.sidebarWidth(for:)` 已用纯策略锁住主侧栏宽度。
  - `ModelLibraryLayoutMode.controlColumnWidth(for:)` 已用纯策略锁住模型页控制列宽度。
  - `ChatWorkspace` 当前只在 `layoutMode.usesSidebar` 为 true 时显示竖向 `SessionBar`，但侧栏宽度公式仍内联。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - `testWorkspaceLayoutModeConstrainsSidebarWidth` 是主侧栏宽度测试参考。
  - `testModelLibraryLayoutModeSupportsWideModelWorkflows` 是内部布局策略测试参考。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的布局策略区域增加 `SessionSidebarLayoutPolicy` 或同等命名类型：
   - `minimumWidth = 240`
   - `maximumWidth = 310`
   - `widthRatio = 0.28`
   - 提供 `static func width(for size: CGSize, layoutMode: WorkspaceLayoutMode) -> CGFloat` 或等价 API。
   - 当 `layoutMode.usesSidebar == false` 时返回 `0`，避免 portrait 单栏误用竖向会话栏。
   - 当使用侧栏时返回 `min(max(size.width * 0.28, 240), 310)`。
2. 在 `ChatWorkspace.body` 中用新策略替代内联宽度公式：
   - 保持 `SessionBar` 的 padding、material、divider 和 `chatSurface` 布局不变。
   - 不改变新建/切换/删除/导出 closure。
3. 新增 XCTest，例如 `testSessionSidebarLayoutPolicyConstrainsWideChatHistory`：
   - iPhone 竖屏或 `WorkspaceLayoutMode.portrait` 返回 0。
   - 宽度 700 的 compact 双栏返回 240。
   - iPad regular 宽度 1024 返回 286.72 左右，使用 `accuracy` 断言。
   - 宽度 1366 或更大返回不超过 310。
   - 所有侧栏布局宽度都在 240...310。
4. 同步文档：
   - `AGENTS.md`：核心架构边界加入会话侧栏宽度策略。
   - `md/flow/flow.md`：UI 工作区说明加入 `SessionSidebarLayoutPolicy`。
   - `md/flow/flowchart.md`：UI 布局图的推理页节点注明会话侧栏宽度策略。
   - `md/test/test.md`：测试数量从 42 更新到 43，覆盖范围加入会话侧栏宽度策略。
   - `README.md`：当前范围、运行说明、验证章节和历史记录同步 v1.8。
   - `update_log.md`：当前核心测试改为 43，追加 v1.8 记录，写明本地轻量检查命令与结果。

## 风险与禁止项

- 风险：如果策略返回值与旧内联公式不一致，会导致 Mac/iPad 推理页会话列表宽度变化；测试必须覆盖关键宽度并保持旧公式结果。
- 风险：如果把策略和 `WorkspaceLayoutMode` 的主侧栏宽度混淆，可能影响全局工作区侧栏；本轮只替换 `ChatWorkspace` 内部会话栏宽度。
- 禁止下载模型权重、云端推理、绕过 verified 门禁、改真实 runtime、引入新第三方依赖、扩大到无关视觉重构。
- 禁止伪造本地或云端测试结果；没有跑的测试必须明确说明。
