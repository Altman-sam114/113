v1.3

## 版本分配依据

- v1.2 已在 `origin/main` 通过 Agent C 验收：commit `3aeb187bc2b64165e6d3bb973af851e217fc691b`，run `28704420287`，artifact `localgemma-ci-v1.2-main-3aeb187-run28704420287-attempt1`。

- 当前 tab/sidebar 和会话 chip 主要靠视觉高亮表达“已选中/当前会话”，对 VoiceOver、Voice Control 和键盘用户不够明确；regular 大屏侧栏也还有空间承载一行用途说明，帮助 Mac/iPad 大画布用户扫描工作区。
- 人工总目标要求持续优化 UI、Mac 和 iPad 版。本轮选择一个小而可验证的 UI/可访问性目标：regular 大屏侧栏增加工作区用途说明，并为工作区和会话选择建立稳定语义基线。
- 因此版本递增为 `v1.3`，主题继续归档在 `md/prompt/v1（Mac体验审计）/`。Agent B commit 主题必须以 `v1.3:` 开头。

## 背景

`ContentView.swift` 中已有顶部 tab picker、侧栏 tab picker 和会话列表。v1.1/v1.2 解决了键盘和菜单入口，但当前选中状态仍主要通过颜色、背景和图标变化表达。SwiftUI 可访问性应把选中状态和图标按钮意图明确暴露给辅助技术；regular 大屏侧栏可显示一行短说明，compact 双栏仍应保持现有密度。

本轮应继续保持窄范围：增加可测试的 sidebar/accessibility 元数据，并把这些元数据应用到现有按钮。不要改变颜色、动画、模型状态或 runtime。

## 目标

1. 为 regular 大屏 sidebar 增加一行工作区用途说明：
   - `landscapeRegular` 使用详细侧栏，显示标题 + 一行短说明。
   - `landscapeCompact` 保持现有紧凑侧栏，不显示说明行。
   - 顶部 portrait tab picker 不新增说明行。
2. 为 workspace tab/sidebar 按钮补充可访问性语义：
   - label 明确为“推理工作区 / 模型工作区 / 提示词工作区 / 设置工作区”或等价短文本。
   - value 明确区分“已选中 / 未选中”。
   - 当前 tab 按钮增加 `.isSelected` trait。
3. 为会话 chip 补充可访问性语义：
   - 选择会话按钮 label 包含会话标题，例如“选择会话 新对话”。
   - 当前会话 value 使用“当前会话”，非当前使用“未选中”或等价短文本。
   - 当前会话选择按钮增加 `.isSelected` trait。
   - 删除会话图标按钮必须有 label，例如“删除会话 新对话”。
4. 抽出可测试的轻量元数据 helper，避免测试只能依赖 SwiftUI modifier：
   - `WorkspaceLayoutMode` 可增加 `usesDetailedSidebar`，只有 `.landscapeRegular == true`。
   - `WorkspaceTab` 增加 `sidebarSubtitle`，每个工作区一行短说明。
   - 可放在 `ContentView.swift` 中，例如 `enum SelectionAccessibilityMetadata`。
   - helper 只生成 label/value 字符串，不持有业务状态。
5. 新增 XCTest 锁住上述语义：
   - `WorkspaceLayoutMode.usesDetailedSidebar` 只在 regular 大屏为 true。
   - `WorkspaceTab.sidebarSubtitle` 非空、顺序覆盖四个 workspace。
   - workspace label/value。
   - session select/delete label。
   - 当前/非当前会话 value。
6. 同步文档和 CI 说明：
   - README、测试规范、flow、flowchart、update_log 必须说明 v1.3 的 Mac/iPad 侧栏选择语义基线。
   - 测试数量从 36 增加到 37；如果最终数量不同，以实际 `grep -n "func test"` 为准并同步。
7. 完成后 commit 到 `main` 并 `git push origin main`，由 GitHub Actions 生成最新 v1.3 未加密结果包，交给 Agent C 下载验收。

## 风险

- 不要用可访问性标签替代真实按钮文本；视觉 UI 仍保持现有结构。
- `.accessibilityAddTraits(.isSelected)` 必须只加在当前选中项上，避免 VoiceOver 误报。
- 会话标题来自用户输入，label 拼接时保持短句，不做复杂清洗或截断。
- 本轮不尝试修复全局固定字号问题；那需要更大 UI 策略，不适合混入本轮。
- regular 侧栏说明要短，不能挤爆 320-390pt 侧栏；compact 侧栏保持现有密度。
