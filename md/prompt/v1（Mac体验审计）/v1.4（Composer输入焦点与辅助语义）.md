# v1.4 / Composer 输入焦点与辅助语义

## 版本号和分配依据

- 版本号：`v1.4`
- 分配依据：`update_log.md` 最新正式版本为 `v1.3 / Mac iPad侧栏选择语义基线`，本轮继续 Agent X 的 Mac/iPad 体验审计主线，递增为 `v1.4`。

## 背景

当前项目已经完成 Mac Catalyst build/run 基线、Mac/iPad 快捷键、`工作区` command menu、regular 大屏侧栏说明和选择语义。下一处高收益小目标是推理页 composer：Mac Catalyst 与 iPad 外接键盘用户在切回推理页、新建/切换会话、从提示词页填入模板后，应能更自然地回到输入区；辅助技术也应能读出输入框、发送、停止和禁用状态。

本轮仍以本地模拟 runtime 为准，不下载 Gemma 权重，不接入云端推理，不改变 artifact 校验门禁。

## 目标

1. 为 composer 建立 view 层焦点请求机制：
   - 切回推理页后可请求聚焦输入框。
   - 新建会话、切换会话后请求聚焦输入框。
   - 提示词模板“填入”或“发送后回到推理页”时请求聚焦输入框。
2. 为 composer 输入框和发送/停止按钮补齐稳定的 accessibility metadata：
   - 输入框有明确 label、hint 和 Voice Control input label。
   - 发送按钮、停止按钮、空输入禁用状态有可测试文案。
   - 保留 `Command+Return` 发送或停止，不让普通 Return 破坏多行输入。
3. 新增纯逻辑 XCTest 锁住焦点策略和 composer 文案。
4. 同步 README、测试规范、核心流程文档、流程图和更新日志。

## 非目标

- 不新增原生 macOS target。
- 不改 v1.1 的快捷键映射、不改 v1.2 的 command menu 映射、不改 v1.3 的 regular 侧栏说明和选择语义。
- 不把焦点状态放进 `InferenceEngine`、`ModelCatalog` 或其他业务状态对象。
- 不改变模型导入、artifact 校验、runtime plan、会话导出或壁纸逻辑。
- 不下载模型权重，不接入真实 Gemma 推理，不接入外部服务。

## 当前架构依据

- `md/flow/flow.md`：`ContentView` 负责 UI 工作区，`InferenceEngine` 负责会话、输入、输出和导出；composer 快捷键目前属于 UI 工作区流。
- `LocalGemma/ContentView.swift`：
  - `ContentView.selectedTab` 管 workspace selection。
  - `PromptTemplatesWorkspace` 通过 `openChat()` 回到推理页。
  - `ChatWorkspace` 组合 `SessionBar`、`ChatTranscript` 和 `ComposerBar`。
  - `ComposerBar` 当前使用 `TextField("问本地模型任何问题", text: $text, axis: .vertical)`，发送/停止按钮已有 `Command+Return`。
- `LocalGemmaTests/LocalGemmaTests.swift` 已有 workspace 快捷键、command menu、选择语义和布局断点测试，本轮应新增同类纯逻辑测试，避免依赖 UI automation。

## 实现步骤

1. Agent B 开始前同步远端：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short --branch`
2. 在 `ContentView` 增加本轮所需的轻量 view 层焦点请求状态，例如递增 token 或类似机制；该状态只用于 UI focus，不进入业务层。
3. 将焦点请求 token 传入 `ChatWorkspace` 和 `ComposerBar`：
   - `ChatWorkspace` 使用 `@FocusState` 或等价 SwiftUI 机制聚焦 composer 输入框。
   - `SessionBar` 的新建和选择动作完成后请求聚焦。
   - `PromptTemplatesWorkspace.openChat()` 切回推理页时请求聚焦。
4. 新增一个可测试 helper，例如 `ComposerInputMetadata` / `ComposerFocusPolicy`：
   - 描述输入框 label/hint/inputLabels。
   - 描述发送、停止、禁用状态的 label/value/hint。
   - 描述哪些交互会请求 composer focus。
5. 把 metadata 应用到 `ComposerBar` 的 `TextField` 和发送/停止按钮。
6. 在 `LocalGemmaTests.swift` 新增测试，覆盖：
   - composer 输入框文案非空且稳定。
   - 空输入非生成中时按钮为发送但禁用；非空输入可发送；生成中为停止。
   - 模板填入、模板发送、新建会话、切换会话属于请求 composer focus 的场景。
7. 更新文档：
   - `README.md` 当前范围、运行方式/交互说明、已完成验证。
   - `md/test/test.md` 当前测试数量和覆盖范围。
   - `md/flow/flow.md` UI 工作区流和铁律。
   - `md/flow/flowchart.md` UI 布局与工作区图。
   - `update_log.md` 追加 v1.4 记录。

## 风险和禁止项

- 不要让普通 Return 变成发送，避免破坏 vertical `TextField` 多行输入。
- 不要把焦点请求写进 `InferenceEngine`，避免 UI transient 状态污染会话状态。
- 不要扩大到全局视觉重做、文件拆分或 runtime 接入。
- 禁止下载模型权重、禁止云端推理、禁止绕过 artifact `verified` 门禁。
- 禁止伪造测试、伪造 push、伪造 CI 或验收旧 artifact。
