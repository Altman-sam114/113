# v1.9 / 模型部署控件辅助语义

## 版本号与分配依据

- 版本号：`v1.9`
- 分配依据：`update_log.md` 当前最新正式版本为 `v1.8 / 会话侧栏宽度策略`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增到 `v1.9`。

## 背景

项目已经为工作区选择、会话选择、composer 输入框和提示词分类筛选建立了可测试的辅助语义 metadata。模型页部署控制台已有清晰可见 UI，但部署电源按钮和模型文件操作按钮仍主要依赖可见文案，缺少统一的 `accessibilityLabel`、`accessibilityValue`、`accessibilityHint`、Voice Control 输入标签和稳定 identifier。

这会让 iPad 外接键盘、Mac Catalyst 和 VoiceOver / Voice Control 用户在模型页执行“启动/关闭部署、模拟暂存、卸载、扫描、导入”时缺少可预测入口。本轮只补 UI 辅助语义，不改变模型文件、部署状态、runtime 计划或 artifact 校验规则。

## 目标

1. 新增可测试的模型部署控件辅助语义 metadata，例如 `ModelDeploymentControlAccessibilityMetadata`。
2. 为 `DeploymentPowerButton` 补充 label、value、hint、input labels，并保留现有 `model-deployment-power` identifier。
3. 为 `ArtifactActionButton` 支持稳定的 action metadata，并为下载/卸载操作补充 label、value、hint、input labels 和 identifier。
4. 为“扫描本地”和“导入文件”按钮补充同一套辅助语义。
5. 文案必须明确“下载模型”当前只是模拟暂存，不联网下载模型权重；未 verified 时不得暗示真实 runtime 可运行。
6. 新增纯 XCTest 锁住部署电源按钮和 artifact 操作按钮的 metadata，测试函数数从 43 增加到 44。
7. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md` 中的当前能力、测试数量和验证记录。

## 非目标

- 不修改 `ModelCatalog`、`ModelArtifactStore`、`LocalArtifactValidator`、`LocalRuntimePlanner`、`InferenceEngine` 或 runtime 行为。
- 不改变模型页可见布局、按钮位置、颜色、图标或操作 closure。
- 不改变 `ModelLibraryLayoutMode`、`WorkspaceLayoutMode` 或 `SessionSidebarLayoutPolicy`。
- 不新增 UI Test target，不做全局视觉重设，不创建原生 macOS target。
- 不下载模型权重，不接入云端推理，不绕过 artifact `verified` 门禁。

## 当前架构依据

- `md/flow/flow.md`：模型页由 `ModelCatalog` 管理选择、artifact validation 和部署状态；`ContentView` 负责 UI，不应绕过状态层直接修改核心状态。
- `LocalGemma/ContentView.swift`：
  - `PromptCategoryAccessibilityMetadata` 和 `ComposerInputMetadata` 已提供可测试 metadata 模式。
  - `DeploymentPowerButton` 目前只有 `accessibilityIdentifier("model-deployment-power")`。
  - `ArtifactActionPanel` 创建“下载模型、卸载模型、扫描本地、导入文件”四个操作入口。
- `LocalGemma/AppState.swift`：
  - `simulateDownload(for:)` 只模拟 staged，不联网下载权重。
  - `uninstallArtifacts(for:)` 删除 App 托管目录文件并停止部署。
  - 只有 `verified` artifact 才允许真实 runtime 计划。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - `testPromptCategoryAccessibilityMetadataDescribesFilterSelectionAndInputLabels` 和 `testComposerInputMetadataAndFocusPolicyDescribeEntryPoints` 是本轮 metadata 测试参考。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的辅助语义 metadata 区域新增 `ModelDeploymentControlAccessibilityMetadata`：
   - 提供部署电源按钮的 `powerLabel(deploymentState:)`、`powerValue(model:validation:deploymentState:)`、`powerHint(validation:deploymentState:)`、`powerInputLabels(deploymentState:)`。
   - 提供 `ArtifactAction` 枚举，覆盖 `download`、`uninstall`、`scan`、`importFiles`。
   - 提供 artifact action 的 `label`、`value`、`hint`、`inputLabels`、`identifier`。
2. 修改 `DeploymentPowerButton`：
   - 添加 `.accessibilityLabel(...)`、`.accessibilityValue(...)`、`.accessibilityHint(...)`、`.accessibilityInputLabels(...)`。
   - 保持现有可见文本和 `model-deployment-power` identifier 不变。
3. 修改 `ArtifactActionButton`：
   - 增加 `metadataAction: ModelDeploymentControlAccessibilityMetadata.ArtifactAction` 或等价属性。
   - 在按钮上设置 label、value、hint、input labels 和 identifier。
4. 修改 `ArtifactActionPanel`：
   - 下载按钮传入 `.download`，卸载按钮传入 `.uninstall`。
   - 扫描和导入 utility buttons 直接接入 metadata。
   - 所有 hint 必须保留本地/手动/不联网边界。
5. 新增 XCTest，例如 `testModelDeploymentControlsExposeAccessibilityMetadata`：
   - missing/stopped 电源按钮 value/hint 必须说明本地模拟或未校验，不包含“已校验”。
   - verified/running 电源按钮 value/hint 可说明已校验和真实 runtime 计划，但不得暗示会下载权重。
   - download hint 必须包含“模拟”和“不联网”。
   - uninstall/scan/import labels、values、hints、identifiers 和 input labels 稳定且非空。
   - `ArtifactAction.allCases` 顺序与四个操作一致，便于 future UI 遍历。
6. 同步文档：
   - `AGENTS.md`：一句话总览和架构边界加入模型部署控件辅助语义。
   - `md/flow/flow.md`：UI 工作区和核心模块加入 `ModelDeploymentControlAccessibilityMetadata`。
   - `md/flow/flowchart.md`：模型页节点注明部署控件辅助语义。
   - `md/test/test.md`：测试数量从 43 更新到 44，覆盖范围加入模型部署控件辅助语义。
   - `README.md`：当前范围、运行说明、模型状态、验证章节和历史记录同步 v1.9。
   - `update_log.md`：当前核心测试改为 44，追加 v1.9 记录，写明本地轻量检查命令与结果。

## 风险与禁止项

- 风险：辅助文案如果写成“下载模型”但不说明模拟，可能误导用户认为会联网下载真实权重；测试必须锁住“不联网”和“模拟暂存”边界。
- 风险：verified/running 文案可以说明真实 runtime 计划，但不能宣称真实推理已经接入。
- 风险：如果 action identifier 不稳定，后续 UI Test 或辅助技术命令会退化。
- 禁止下载模型权重、云端推理、绕过 verified 门禁、改真实 runtime、引入新第三方依赖、扩大到无关视觉重构。
- 禁止伪造本地或云端测试结果；没有跑的测试必须明确说明。
