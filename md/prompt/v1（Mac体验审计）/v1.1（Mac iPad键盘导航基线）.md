## 背景

当前项目是 SwiftUI iOS 工程，支持 iPhone、iPad 与 Mac Catalyst。`ContentView.swift` 中已有四个 workspace、会话栏、提示词卡片和 composer，但没有明确键盘路径。Mac Catalyst 和 iPad 外接键盘用户自然期望：

- `Command+1/2/3/4` 在推理、模型、提示词、设置之间切换。
- `Command+N` 新建对话。
- `Command+Shift+E` 导出当前对话。
- `Command+Return` 发送 prompt 或停止生成。

这些快捷键不应替代可见按钮；只是在现有按钮上增加桌面/iPad 可达性。当前架构中 `WorkspaceTab` 是 workspace 枚举，`SessionBar` 和 `ComposerBar` 已经把 action 作为闭包传入，适合做小范围 SwiftUI modifier 改动。

## 目标

1. 为 workspace 切换建立稳定快捷键映射：
   - `Command+1`：推理
   - `Command+2`：模型
   - `Command+3`：提示词
   - `Command+4`：设置
2. 为高频会话和 composer 操作添加快捷键：
   - 新建对话按钮：`Command+N`
   - 导出当前对话按钮：`Command+Shift+E`
   - 发送/停止按钮：`Command+Return`
3. 用 XCTest 锁住 workspace 快捷键映射：
   - 在 `WorkspaceTab` 上增加可测试的快捷键字符属性，例如 `shortcutKey: Character`。
   - 新增测试确认四个 tab 的 shortcut 与标题一一对应。
4. 同步文档和 CI 说明：
   - README、测试规范、flow、flowchart、update_log 必须说明 v1.1 的键盘导航基线。
   - 测试数量从 34 增加到 35；如果最终数量不同，以实际 `grep -n "func test"` 为准并同步。
5. 完成后 commit 到 `main` 并 `git push origin main`，由 GitHub Actions 生成最新 v1.1 未加密结果包，交给 Agent C 下载验收。

## 非目标

- 不创建原生 macOS target。
- 不引入 AppKit responder chain、FocusedValue 或复杂 CommandMenu；本轮只在现有 SwiftUI 按钮上加 `keyboardShortcut`。
- 不重做视觉风格，不新增可见快捷键说明文本。
- 不改变 `InferenceEngine`、`ModelCatalog`、artifact validation、runtime plan 或真实模型门禁。
- 不下载模型权重，不接云端推理，不新增大体积测试数据。
- 不让快捷键绕过空 prompt 禁用、停止生成逻辑或当前按钮状态。
- 不改变 iPhone 触控路径；快捷键只作为额外输入路径。

## 风险

- 重复注册同一个快捷键可能导致行为不明确；workspace 快捷键只能加在当前可见的 tab/sidebar 按钮路径，不要在 scene command 和按钮上重复实现。
- `Command+Return` 必须继续尊重空 prompt 禁用和生成中停止逻辑。
- SwiftUI `keyboardShortcut` 在 iOS、iPadOS、Mac Catalyst 下都应能编译；如编译失败，做最小条件编译。
- 本轮不做可见快捷键说明，避免 UI 文案膨胀；后续如需要可统一设计 help/command surface。
