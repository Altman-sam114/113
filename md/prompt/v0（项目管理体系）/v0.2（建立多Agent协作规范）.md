# v0.2（建立多 Agent 协作规范）

## 版本号

`v0.2`

## 版本分配依据

人工未指定版本号。本轮不是功能开发，而是为现有 SwiftUI iOS 原型建立长期迭代的 Agent 协作系统、记忆文档、测试规范和核心流程文档，因此沿用 v0 阶段项目管理体系，小版本递增为 `v0.2`。

## 背景

当前项目已有可运行的 `Local Gemma iOS Prototype`：包含 SwiftUI UI、模型管理、artifact 校验、模拟推理、会话导出、横屏布局、相册壁纸和 XCTest 覆盖。但后续迭代需要更严格的多 Agent 工作流，避免 README、测试规范、核心流程和实现状态漂移。

## 目标

创建或更新以下结构：

- `AGENTS.md`
- `update_log.md`
- `md/prompt/`
- `md/test/test.md`
- `md/flow/flow.md`
- `md/flow/flowchart.md`

这些文档必须适配当前项目，不写空泛模板。

## 非目标

- 不开发新 App 功能。
- 不修改 Swift 源码。
- 不重构 Xcode 工程。
- 不运行真实模型。
- 不改变模型下载和隐私边界。

## 当前架构依据

- `LocalGemma/AppState.swift`：核心状态、模型、artifact、runtime、会话、导出和优化逻辑。
- `LocalGemma/ContentView.swift`：SwiftUI UI、横屏布局、壁纸、分享视图。
- `LocalGemmaTests/LocalGemmaTests.swift`：当前 32 个 XCTest 方法。
- `README.md`：现有用户文档和验证记录。

## 实现步骤

1. 阅读附件任务要求和当前项目文档。
2. 检查 `git status` 和当前已有 `AGENTS.md`。
3. 创建 `md/prompt`、`md/test`、`md/flow` 目录。
4. 重写 `AGENTS.md` 为多 Agent 入口文档。
5. 新增 `update_log.md`。
6. 新增 `md/test/test.md`。
7. 新增 `md/flow/flow.md`。
8. 先写 `flow.md`，再写 `flowchart.md`，并保证 Mermaid 图与核心流程一致。
9. 在 README 中加入项目管理文档入口和本轮完成情况。
10. 做文档结构和关键内容校验。

## 测试要求

本轮为文档体系搭建，未修改 Swift 源码。至少执行：

```sh
find md -maxdepth 4 -type f | sort
grep -n "Agent A\\|Agent B\\|Agent C\\|README\\|测试规范" AGENTS.md
grep -n "flowchart\\|mermaid\\|LocalArtifactValidator\\|InferenceEngine" md/flow/flowchart.md md/flow/flow.md
git status --short
```

如修改 README，也检查对应章节可读性。

## 文档更新要求

- `AGENTS.md` 必须包含项目总览、必读文件、基本规则、架构边界、Agent A/B/C 工作流、测试规则、文档规则、交付格式、禁止项。
- `update_log.md` 必须包含当前状态、历史记录、维护规则和本轮 v0.2。
- `md/test/test.md` 必须包含 Probe/Fast、Smoke、Stage Regression、Full 四层测试。
- `md/flow/flow.md` 必须写当前真实链路。
- `md/flow/flowchart.md` 必须有 Mermaid 图和中文读图说明。
- README 必须加入这些项目管理文档的入口说明。

## 验收标准

- 所有指定文件存在。
- 文档内容明确引用当前项目真实模块和流程。
- Agent A/B/C 职责清晰。
- 测试分层可直接执行。
- 流程图能解释启动、artifact、推理、布局、壁纸、分享核心流。
- `git status` 只显示本轮预期文档变更。

## 风险和禁止项

- 禁止写成空模板。
- 禁止把真实模型推理说成已完成。
- 禁止改 Swift 源码。
- 禁止删除用户已有文档内容而不迁移有效信息。
- 禁止伪造测试结果。
