# Agent B 提示词：v0.7 / 云端验收闭环与文档校准

## 版本号

v0.7

## 版本分配依据

## 背景总结

当前项目已经完成以下阶段：

- v0.1 建立 SwiftUI iOS 原型，覆盖模型清单、artifact 校验、模拟 runtime、会话导出、横屏布局、相册壁纸和 Apple Silicon 运行计划。
- v0.2 建立 `AGENTS.md`、`update_log.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `md/prompt/` 文档体系。
- v0.3 明确 Agent C 版本提交和退回规则。
- v0.4 建立 `main` 直推、GitHub Actions 云端重验证和未加密 CI 结果包制度。
- v0.5 修复 GitHub Actions 暴露的 Swift 6 actor isolation 构建问题。
- v0.6 修复 CI artifact 命名沿用旧版本号的问题，使版本号从 commit subject 的 `vX.Y` 提取。

当前本地核对结果：

- `main` 与 `origin/main` 对齐。
- `git fetch origin` 可执行成功。
- `git remote -v` 当前已经配置 `origin`，因此 README 和 `update_log.md` 中“本地仓库尚未配置 origin remote”的当前状态描述已经过期。
- Swift 业务核心仍维持模拟 runtime，不下载权重，不执行真实模型推理。
- `LocalGemmaTests.swift` 当前有 32 个 XCTest 方法，覆盖 artifact、模型部署、runtime plan、提示词、会话、导出、横屏布局、壁纸处理和分享兜底。

## 下一步优化目标

优先目标不是接真实 Gemma 权重，而是先把云端闭环做实：

1. 修正文档中的远端状态漂移，区分“v0.4 历史环境曾无 origin”和“当前仓库已配置 origin”。
2. 强化 GitHub Actions 结果包自描述能力，让 Agent C 下载后能直接核对 artifact 名称、commit subject、仓库、分支、run URL、run attempt 和各阶段 outcome。
3. 更新测试规范、README、核心流程文档和流程图，使 `v0.7` 后的 Agent B / Agent C 有一致验收入口。
4. 不改业务 UI、不改模型 runtime、不引入真实推理，避免把流程修复扩大成产品功能重构。

## 非目标

- 不下载 Gemma、Qwen、LFM 或其他模型权重。
- 不接入云端推理、外部模型 API、MLX、llama.cpp 或 Core ML 真实推理。
- 不把 `manual-import-required` 提升成 verified。
- 不修改 `RealGemmaRuntimePlaceholder` 的语义。
- 不新增 UI Test target。
- 不拆分 `AppState.swift` 或 `ContentView.swift`。
- 不改 Swift UI 体验，除非 workflow 或文档检查发现必须修正的事实错误。

## 云端 CI 触发要求

Agent B 必须在 `main` 上 push：

```sh
git push origin main
```

GitHub Actions 应触发 `.github/workflows/ci-results.yml`，生成未加密结果包：

```text
localgemma-ci-v0.7-main-<short_sha>-run<run_id>-attempt<run_attempt>
```

如果 push 或权限失败，必须明确写为阻塞；不得声称 GitHub Actions 已运行。

## Agent C 验收标准

Agent C 必须只验收最新 `origin/main` 对应的结果包：

1. `git fetch origin`
2. 核对 `main`、`origin/main` 和 Agent B commit SHA。
3. `gh run list --workflow ci-results.yml --branch main --limit 5`
4. 下载最新 run artifact 到 `/private/tmp/localgemma-c-review-<run_id>/`。
5. 打开 `ci-artifact-manifest.json`，核对：
   - `branch == main`
   - `commitSha == origin/main 最新 commit`
   - `runId` 与下载 run 一致
   - `runAttempt` 与下载 run attempt 一致
   - `artifactName` 与 `artifact-name.txt` 一致
   - `version == v0.7`
6. 打开 `ci-failure-summary.md`、`junit.xml`、`static-checks.log`、`logic-smoke.log`、`xcodebuild.log`、`test.log` 和 `.xcresult` 或等价结果。
7. 如果任一 outcome 不是 success，或 test 被 skipped 但无明确原因，则不通过，退回 Agent B 追加修复 commit。
8. 通过时输出版本号、commit hash、run id、run attempt、artifact 名称和已核对文件。

## 风险

- 当前文档存在“当前仓库无 origin”的过期描述，容易误导 Agent B / Agent C 判定云端阻塞。
- 如果 workflow manifest 缺少 artifact name 或 run URL，Agent C 仍能手工验收，但容易错验旧 artifact。
- GitHub-hosted runner 的 iPhone Simulator 名称可能变化，应以 workflow 自动选择结果和 manifest `destination` 为准。
- 若 `gh` 未登录或无 artifact 下载权限，Agent C 必须先登录或报告权限阻塞。

## 禁止项

- 禁止下载模型权重。
- 禁止接入云端推理或外部模型 API。
- 禁止绕过 artifact `verified` 门禁。
- 禁止把 `manual-import-required` 视为真实可运行权重。
- 禁止把 `RealGemmaRuntimePlaceholder` 描述成真实推理已完成。
- 禁止伪造测试、push、run id、artifact 名称或 Agent C 验收结果。
- 禁止无关重构。
- 禁止删除用户或其他 Agent 的改动。
