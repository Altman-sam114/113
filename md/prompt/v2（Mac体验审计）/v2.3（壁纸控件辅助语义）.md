# v2.3 / 壁纸控件辅助语义

## 版本号与分配依据

- 版本号：`v2.3`
- 分配依据：`update_log.md` 当前最新正式版本为 `v2.2 / 导出分享复制辅助语义`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增到 `v2.3`。

## 背景

v2.2 已为导出弹层分享和复制动作补齐辅助语义，并通过 GitHub Actions 结果包验收。继续审计 Mac Catalyst 与 iPad 大屏设置体验时，设置页 `WallpaperPreferencePanel` 中的相册壁纸选择和恢复系统背景按钮仍只有英文 `accessibilityLabel`，缺少统一的 `accessibilityValue`、`accessibilityHint`、Voice Control 输入标签和稳定 identifier。

这会让 VoiceOver / Voice Control 用户无法明确区分“选择相册图片并经过本地压缩保存为背景”和“清空自定义壁纸恢复系统背景”，也无法从辅助语义中理解导入中、已启用相册图片、当前系统背景和禁用状态。本轮只补壁纸控件辅助语义，不改变相册读取、图片压缩、`AppStorage` 写入、背景渲染或主题视觉。

## 目标

1. 新增可测试的 `WallpaperPreferenceAccessibilityMetadata` 或等价 metadata。
2. 覆盖两类设置页壁纸动作：
   - 从系统相册选择壁纸。
   - 清空自定义壁纸并恢复系统背景。
3. 将 metadata 应用到 `WallpaperPreferencePanel` 的 `PhotosPicker` 和清空按钮。
4. label / value / hint 必须说明：
   - 选择壁纸会打开系统相册选择图片。
   - 图片会在本地压缩后写入 App 背景数据。
   - 清空动作只移除自定义壁纸并恢复系统背景。
   - 导入中或没有自定义壁纸时对应按钮会不可用。
   - 这些动作不下载模型权重、不触发真实 runtime、不把图片或提示词发送到云端服务。
5. 增加 Voice Control 输入标签和稳定 identifier，例如 `wallpaper-action-choose-photo`、`wallpaper-action-clear-custom`。
6. 新增纯 XCTest 锁住 label、value、hint、input labels、identifier 和 disabled 状态文案；测试函数数从 47 增加到 48。
7. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md` 中的当前能力、测试数量和验证记录。

## 非目标

- 不修改 `WallpaperImageProcessor.optimizedJPEGData` 或缩放/压缩参数。
- 不修改 `@AppStorage("customWallpaperImageData")` 的存储方式。
- 不改变 `PhotosPicker` 的选择范围、导入错误处理、alert 文案或背景渲染。
- 不重设设置页视觉，不拆分 `ContentView.swift`，不新增 UI Test target，不创建原生 macOS target。
- 不下载模型权重，不接入云端推理，不上传图片、会话或提示词到外部服务。

## 当前架构依据

- `md/flow/flow.md`：壁纸流程要求 `PhotosPicker` 读取相册图片，`WallpaperImageProcessor` 压缩后写入 `customWallpaperImageData`，`AppBackground` 展示壁纸并叠加主题遮罩。
- `LocalGemma/ContentView.swift`：
  - `SettingsWorkspace` 持有 `selectedWallpaperItem`、`isImportingWallpaper` 和 `wallpaperImportError`。
  - `WallpaperPreferencePanel` 展示壁纸状态、`PhotosPicker` 选择按钮和清空按钮。
  - 当前 `PhotosPicker` 只有英文 `Choose wallpaper from photo library` label；清空按钮只有英文 `Clear custom wallpaper` label。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - `testWallpaperProcessorScalesImagesWithoutUpscaling` 只覆盖压缩尺寸逻辑。
  - 本轮应新增壁纸控件 metadata 测试，不重复测试图片处理实现。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的壁纸相关类型附近新增 `WallpaperPreferenceAccessibilityMetadata`：
   - `Action`：`.choosePhoto`、`.clearCustomWallpaper`。
   - `label(for:)`
   - `value(for:hasCustomWallpaper:isImporting:)`
   - `hint(for:hasCustomWallpaper:isImporting:)`
   - `inputLabels(for:)`
   - `identifier(for:)`
2. 修改 `WallpaperPreferencePanel`：
   - `PhotosPicker` 接入 `.choosePhoto` 的 label/value/hint/input labels/identifier。
   - 清空按钮接入 `.clearCustomWallpaper` 的 label/value/hint/input labels/identifier。
   - 保留现有按钮尺寸、图标、禁用逻辑和视觉状态。
3. 新增 XCTest，例如 `testWallpaperPreferenceControlsExposeAccessibilityMetadata`：
   - 两个 action 的 label/value/hint/input labels/identifier 均非空。
   - 选择壁纸 value/hint 覆盖系统背景、相册图片已启用、导入中三种状态。
   - 清空按钮 value/hint 覆盖可清空、无自定义壁纸、导入中不可用。
   - 文案包含“系统相册”“本地压缩”“系统背景”“不会发送到云端服务”等边界。
   - 确认 `WallpaperImageProcessor` 现有测试保持不变。
4. 同步文档：
   - `AGENTS.md`：一句话总览和核心边界加入设置页壁纸控件辅助语义。
   - `md/flow/flow.md`：UI 工作区、壁纸流、核心模块加入 `WallpaperPreferenceAccessibilityMetadata`。
   - `md/flow/flowchart.md`：相册壁纸流注明选择/清空动作辅助语义。
   - `md/test/test.md`：测试数量从 47 更新到 48，覆盖范围和 Stage Regression 触发条件加入壁纸控件辅助语义。
   - `README.md`：当前范围、设置页说明、验证章节和历史记录同步 v2.3。
   - `update_log.md`：当前核心测试改为 48，追加 v2.3 记录，写明本地轻量检查命令与结果。

## 风险与禁止项

- 风险：相册选择按钮如果只有英文 label，会继续削弱中文 VoiceOver / Voice Control 体验；本轮必须统一中文 metadata。
- 风险：壁纸文案不能暗示上传图片、联网下载或触发模型 runtime；必须明确是系统相册、本地压缩和本地背景数据。
- 风险：清空按钮 disabled 状态必须从 `wallpaperData.isEmpty` 和 `isImporting` 推导，不改现有业务逻辑。
- 禁止下载模型权重、云端推理、上传图片或会话、绕过 verified 门禁、改真实 runtime、引入新第三方依赖、扩大到设置页视觉重构。
- 禁止伪造本地或云端测试结果；没有跑的测试必须明确说明。
