# v2.26 模型卸载确认弹层 - Agent B 实现提示词

## 版本号与分配依据

- 版本号：v2.26。
- 分配依据：`update_log.md` 当前最新正式版本为 v2.25，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增。

## 背景

模型页的 `ArtifactActionPanel` 中，“卸载模型”是破坏性操作。当前按钮在宽屏双栏和窄屏单栏路径都会直接调用 `uninstall(model)`，进而通过 `ModelCatalog.uninstallArtifacts(for:)` 删除 App 托管目录中的 manifest 必需文件并停止部署。Mac Catalyst 和 iPad 上指针/触控混用时，破坏性操作需要更清晰的确认层。

## 目标

- 为模型 artifact 卸载动作增加确认弹层。
- 卸载按钮只打开确认，不直接删除文件。
- 确认弹层说明本地删除范围：只移除 App 托管目录里的模型 artifact/tokenizer，并停止当前模型部署。
- 确认弹层说明安全边界：不下载模型权重、不启动真实 runtime、不发送到云端服务、不绕过 verified 门禁。
- 为确认弹层新增独立辅助语义元数据，包括 title、message、confirm/cancel label、hint、input labels 和稳定 identifier。
- 更新现有 artifact action / panel metadata 文案，让 VoiceOver 和 Voice Control 明确“打开确认，确认后才删除”。
- 新增 XCTest 锁住确认弹层元数据；更新既有部署控件和模型文件面板辅助语义测试。
- 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md`。

## 非目标

- 不修改 `ModelCatalog.uninstallArtifacts(for:)` 的状态流或文件删除逻辑。
- 不新增真实模型 runtime，不下载 Gemma 权重，不接入外部推理服务。
- 不改变 artifact `missing` / `staged` / `verified` 校验规则。
- 不改模型选择、模拟暂存、扫描、导入文件的业务行为。
- 不做大文件拆分或无关 UI 重构。

## 当前架构依据

- `md/flow/flow.md` 规定：`ContentView` 只负责 UI，模型文件删除必须通过 `ModelCatalog` / `ModelArtifactStore`；`ModelArtifactPanelAccessibilityMetadata` 负责模型文件工作流整体辅助语义。
- `LocalGemma/ContentView.swift` 中：
  - `ModelLibraryView` 管理模型页 UI、file importer 和错误 alert。
  - `ArtifactActionPanel` 在宽屏和窄屏路径中接收 `uninstall` closure。
  - `uninstall(_:)` 是 UI 层 wrapper，内部调用 `catalog.uninstallArtifacts(for:)` 并处理错误。
  - `ModelDeploymentControlAccessibilityMetadata` 和 `ModelArtifactPanelAccessibilityMetadata` 生成 artifact 操作与面板辅助语义。
- `LocalGemma/AppState.swift` 中：
  - `LocalModel` 已是 `Identifiable`。
  - `ModelCatalog.uninstallArtifacts(for:)` 已覆盖删除托管文件、停止部署和刷新 validation 的核心行为。

## 实现步骤

1. 在 `ModelLibraryView` 增加 `@State private var pendingUninstallModel: LocalModel?`。
2. 增加 `requestUninstall(_ model: LocalModel)`，只设置 pending model。
3. 将两个 `ArtifactActionPanel` 调用里的 `uninstall: { uninstall(model) }` 改为 `uninstall: { requestUninstall(model) }`。
4. 在 `ModelLibraryView` 外层挂载 `.confirmationDialog(item:)`，绑定 `pendingUninstallModel`。
5. 确认按钮使用 `.destructive` role，调用既有 `uninstall(model)` 后清空 pending；取消按钮使用 `.cancel` role 并清空 pending。
6. 新增 `ModelUninstallConfirmationAccessibilityMetadata` 或等价 enum，提供：
   - `title(model:)`
   - `message(model:)`
   - `confirmLabel(model:)`
   - `confirmHint(model:)`
   - `confirmInputLabels(model:)`
   - `confirmIdentifier(model:)`
   - `cancelLabel`
   - `cancelHint`
   - `cancelInputLabels`
   - `cancelIdentifier`
7. 更新卸载按钮的 value/hint：
   - value 应说明当前 artifact 状态和“打开确认弹层”。
   - hint 应说明“确认后移除 App 托管目录中的模型文件并停止当前模型部署”。
8. 更新 `ModelArtifactPanelAccessibilityMetadata.value`，将“卸载本地文件”改成“卸载前确认”或等价表达。
9. 保持 `ArtifactActionPanel` 的 `.accessibilityElement(children: .contain)`，让面板整体和四个按钮都可达。

## 风险与禁止项

- 禁止把卸载按钮继续做成直接删除。
- 禁止把取消按钮接到删除逻辑。
- 禁止修改 `ModelCatalog.uninstallArtifacts(for:)` 来绕过现有状态流。
- 禁止下载模型权重、提交模型文件、大数据、证书、密码或 secret。
- 禁止新增云端推理或外部网络模型服务。
- 禁止绕过 artifact `verified` 门禁。
- 禁止伪造测试、CI 或 Agent C artifact 验收结果。
