# v2.29（提示词模板文本动态排版）

## 版本号与分配依据

- 版本号：`v2.29`
- 分配依据：`update_log.md` 最新正式版本为 `v2.28 / 提示词分类筛选换行布局`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版”下继续递增。

## 背景

v2.21 已让提示词模板网格在 iPad/Mac 宽区域多列伸展，v2.25 锁住模板动作 44pt 触控目标，v2.28 让提示词分类筛选 chip 在窄屏换行。但 `PromptTemplateCard` 内部仍存在固定小字号、标题/副标题单行截断和固定最小高度，较大 Dynamic Type、iPad 分屏和 Mac Catalyst 窗口下可读性不足。

## 目标

1. 新增提示词模板卡片文本排版策略，集中定义标题、副标题、正文、分类标签行数和卡片最小高度。
2. 将模板卡片主要文本从固定小字号改为 SwiftUI 语义字体，尊重 Dynamic Type。
3. 允许标题、副标题和正文展示更多行，减少 iPad/Mac 宽区域和较大文字下的截断。
4. 新增 XCTest 锁住排版策略常量，并确认不破坏现有动作区最小宽度与 44pt 触控目标。
5. 同步 README、测试规范、核心流程文档、流程图、入口规则和更新日志。

## 非目标

- 不新增真实模型 runtime。
- 不下载 Gemma 或其他模型权重。
- 不接入云端推理或外部服务。
- 不改变提示词模板库内容。
- 不改变模板填入、直接发送、生成中禁用、composer 聚焦或辅助语义状态流。
- 不改 `PromptTemplateGridLayoutPolicy` 的列数阈值和最大卡片宽度。
- 不创建原生 macOS target，不新增 UI Test target。

## 当前架构依据

- `md/flow/flow.md` 中提示词页由 `PromptTemplatesWorkspace`、`PromptTemplateGrid` 和 `PromptTemplateCard` 负责 UI。
- `PromptTemplateGridLayoutPolicy` 负责卡片网格宽度和列数。
- `PromptCategoryLayoutPolicy` 负责分类筛选 chip 换行和 44pt 触控目标。
- `PromptTemplateActionLayoutPolicy` 负责模板卡片动作区 spacing、padding、发送按钮尺寸和最小动作区宽度。
- `PromptTemplateActionAccessibilityMetadata` 负责“填入”和“发送”的 VoiceOver / Voice Control 语义，本轮不得改变其文案和 identifier。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的提示词模板布局策略附近新增 `PromptTemplateTextLayoutPolicy`。
2. 将 `PromptTemplateCard` 标题、副标题、正文、分类标签和动作文字改为语义字体，例如 `headline`、`subheadline`、`callout`、`caption`，避免主要文本继续使用固定小字号。
3. 使用 `PromptTemplateTextLayoutPolicy` 的 line limit、line spacing、header spacing 和 minimum card height，允许标题/副标题/正文多行显示。
4. 保留按钮 action、disabled/opacity、`PromptTemplateActionAccessibilityMetadata`、`PromptTemplateActionLayoutPolicy` 和网格列数逻辑。
5. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增测试，锁住文本排版策略常量、最小高度和动作区兼容性。
6. 同步文档和日志，将测试函数数从 72 更新到 73。
