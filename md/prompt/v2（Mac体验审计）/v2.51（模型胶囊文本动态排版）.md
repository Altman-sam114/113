# v2.51 模型胶囊文本动态排版

## 版本号与分配依据

- 版本号：v2.51。
- 分配依据：`update_log.md` 最新正式轮次为 v2.50，Agent C 已验收 `4c183ca` / run `29164309862`。

## 背景

顶部 `ModelCapsule` 与 `HeaderMetricChip` 已有整体辅助语义，但模型名、状态摘要和指标 chip 仍使用固定小字号，并依赖 `minimumScaleFactor` 单行压缩。iPad split view、Mac Catalyst 窄窗口和较大 Dynamic Type 下可读性不足。

## 目标

- 新增 `ModelCapsuleTextLayoutPolicy`。
- 模型名、状态摘要、HeaderMetricChip 标题/数值改用 Dynamic Type 语义字体。
- 移除名称/状态/数值上的 `minimumScaleFactor` 压缩；名称允许两行，状态允许两行，指标标题/数值允许多行策略。
- 新增 `testModelCapsuleTextLayoutPolicySupportsDynamicTypeRows`（93 -> 94）。
- 同步文档；完整验证走 GitHub Actions，禁止本机 Xcode/Simulator。

## 非目标

- 不改变 `ModelCapsuleAccessibilityMetadata`、主题切换、工作区切换、模型文件、runtime、verified 门禁。
- 不改变 SIM/REAL 徽章视觉徽标样式的业务含义。
- 不下载模型权重，不接真实 runtime，不创建原生 macOS target。

## 实现要点

1. 策略：`verticalSpacing`、`titleStatusSpacing`、`nameLineLimit=2`、`statusLineLimit=2`、`metricTitleLineLimit=1`、`metricValueLineLimit=2`、`metricMinHeight`。
2. 模型名 `.subheadline.weight(.bold)`；状态 `.footnote.weight(.medium)`；metric title `.caption2.weight(.black)`；metric value `.caption.weight(.heavy)`。
3. fixedSize vertical；不使用 minimumScaleFactor。
4. 测试锁住策略与多行能力。
