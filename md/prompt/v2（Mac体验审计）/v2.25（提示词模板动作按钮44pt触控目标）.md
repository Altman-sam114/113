# v2.25 / 提示词模板动作按钮 44pt 触控目标

## 版本号和分配依据

- 版本号：`v2.25`
- 分配依据：`update_log.md` 最新正式版本为 `v2.24 / Composer宽屏输入宽度`，本轮继续递增到 `v2.25`。
- 所属总目标：Agent X 持续优化 UI、Mac 和 iPad 体验。

## 背景

v2.21 已经让提示词模板卡片在 iPad/Mac 宽区域多列伸展，v2.10 已经为模板卡片“填入”和“发送”动作建立辅助语义。但只读子 agent 审计指出 `PromptTemplateCard` 动作区仍有可用性缺口：发送按钮固定 `36x34`，填入按钮仅靠垂直 padding 形成约 34pt 高度，低于 Apple 常规 44x44 触控目标。

本轮应只修复提示词模板动作区触控目标和布局常量，不修改模板内容、筛选业务、`PromptTemplateActionAccessibilityMetadata` 文案、`InferenceEngine.applyTemplate` / `useTemplate` 状态流或生成中禁用语义。

## 目标

1. 新增 `PromptTemplateActionLayoutPolicy`，集中定义动作区 spacing、卡片 padding、最小触控目标、发送按钮尺寸和动作区最小宽度。
2. 将 `PromptTemplateCard` 的“填入”按钮显式保证最小 44pt 高度。
3. 将“发送”图标按钮从旧 `36x34` 调整到 44x44。
4. 保留 `PromptTemplateActionAccessibilityMetadata` 的 label/value/hint/input labels/identifier。
5. 保留 `PromptTemplateCard.disabled(isGenerating)`，生成中仍禁用填入和发送。
6. 新增 XCTest 锁住 44pt 触控目标和动作区宽度策略。
7. 同步 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 非目标

- 不修改 `PromptTemplateLibrary` 模板内容、分类或筛选结果。
- 不修改 `PromptTemplateActionAccessibilityMetadata` 语义。
- 不修改 `InferenceEngine.applyTemplate`、`useTemplate` 或生成中禁用状态流。
- 不修改 `PromptTemplateGridLayoutPolicy` 的列数、最小/最大卡片宽度或网格策略。
- 不修改 composer 焦点策略、会话、模型、artifact、runtime、导出、壁纸或设置页。
- 不新增 UI Test target。
- 不下载模型权重，不接入云端推理，不接入真实 runtime，不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `PromptTemplatesWorkspace` 把 `inference.isGenerating` 传给 `PromptTemplateGrid`，模板填入调用 `inference.applyTemplate(template)` 后切回推理页，模板发送调用 `inference.useTemplate(... availability:)` 后切回推理页。
  - `PromptTemplateCard` 内部动作区当前为 `HStack(spacing: 8)`，填入按钮靠 padding 控制高度，发送按钮固定 `36x34`。
  - `PromptTemplateGridLayoutPolicy` 已锁住卡片宽度和列数，本轮不改。
  - `PromptTemplateActionAccessibilityMetadata` 已锁住模板动作辅助语义，本轮只改视觉/触控布局。
- `LocalGemma/AppState.swift`
  - `InferenceEngine.applyTemplate` 在非生成中只写入 `inputText`，不发送消息。
  - `InferenceEngine.useTemplate` 在非生成中写入模板并走本地模拟发送。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试基线为 68 个 XCTest 方法。
  - 已有 `testPromptTemplateActionsExposeAccessibilityMetadata`、`testInferenceAppliesPromptTemplateWithoutSending` 和 `testInferenceUsesPromptTemplateForGeneration` 锁住动作语义与状态流。

## 实现步骤

1. 在 `ContentView.swift` 中新增 `PromptTemplateActionLayoutPolicy`。
2. 策略建议：
   - `minimumTouchTarget = 44`
   - `spacing = 8`
   - `cardPadding = 13`
   - `sendButtonSize = minimumTouchTarget`
   - `minimumApplyButtonWidth = 112`
   - `minimumCardWidthForActionRow()` 用于测试动作区是否适配现有 `PromptTemplateGridLayoutPolicy.minimumCardWidth`。
3. `PromptTemplateCard`：
   - 动作 `HStack` spacing 改为策略常量。
   - “填入”按钮 label 增加最小高度 44pt。
   - “发送”按钮 frame 改为策略的 44x44。
   - 卡片 padding 改用策略常量，但保持数值不变。
4. 在 `LocalGemmaTests.swift` 新增 `testPromptTemplateActionLayoutPolicyMaintains44PointTouchTargets`，测试函数数从 68 增加到 69。

## 风险和禁止项

- 风险：触控目标变大可能改变模板卡片动作区视觉密度，必须确保现有最小卡片宽度仍可容纳动作区。
- 风险：不得把 icon-only 发送按钮改成无文本语义的图片按钮；必须保留 `Label` 和现有辅助语义。
- 风险：不得移除 `.disabled(isGenerating)`，否则生成中可触发模板动作。
- 禁止下载模型权重、云端推理、绕过 verified 门禁。
- 禁止把未跑测试写成已通过。
- 禁止验收旧 run 或旧 artifact。
