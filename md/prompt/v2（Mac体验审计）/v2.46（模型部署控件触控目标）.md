# v2.46 模型部署控件触控目标

## 版本号与分配依据

- 版本号：v2.46。
- 分配依据：`update_log.md` 最新正式轮次为 v2.45，本轮在 Agent X “持续优化 UI、Mac、iPad 版体验”总目标下继续递增。

## 背景

模型页已经通过 `ModelDeploymentControlAccessibilityMetadata` 锁住模型选择器、部署电源和 artifact 操作按钮的辅助语义，也通过 `ModelArtifactActionLayoutPolicy` 锁住扫描本地和导入文件 utility 按钮的 44pt 触控目标。但模型选择器和部署电源按钮仍主要依赖现有 padding / 固定 minHeight，缺少独立布局策略和测试，后续 UI 调整可能让 Mac Catalyst、iPad split view 和触控设备上的核心模型部署入口回退。

## 目标

- 新增模型部署控件布局策略，集中定义模型选择器和部署电源按钮的 44pt 最小触控目标。
- 让 `ModelSelectorPanel` 的 picker 触发区复用该策略，保持至少 44pt 命中高度。
- 让 `DeploymentPowerButton` 复用该策略，保留当前大按钮视觉高度，同时明确它高于 44pt 最低要求。
- 新增 XCTest 锁住策略常量、控件枚举、identifier 映射和 44pt 达标判断。
- 同步 README、测试规范、核心流程、流程图、入口规则和更新日志。

## 非目标

- 不改变 `ModelCatalog`、artifact 校验、部署状态流、卸载确认状态流或 runtime 计划。
- 不改变模型选择器、部署电源按钮或 artifact 操作按钮的辅助语义文案。
- 不下载模型权重，不接入云端推理，不绕过 artifact `verified` 门禁。
- 不创建原生 macOS target，不做完整 UI Test target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `ModelSelectorPanel` 渲染模型选择器和状态徽章。
  - `DeploymentPowerButton` 渲染启动/关闭模型部署入口。
  - `ModelDeploymentControlAccessibilityMetadata` 管控模型部署控件辅助语义。
  - `ModelArtifactActionLayoutPolicy` 仅覆盖模型文件 utility 按钮。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 已有模型选择器和部署控件辅助语义测试。
  - 已有模型 artifact utility 按钮 44pt 测试。
- `md/flow/flow.md` 和 `md/test/test.md`
  - 当前 Mac/iPad UI 策略要求所有被锁住控件同步测试和文档。

## 实现步骤

1. 在 `ContentView.swift` 中新增 `ModelDeploymentControlLayoutPolicy`。
2. 定义 `minimumTouchTarget = 44`、`modelSelectorMinHeight = minimumTouchTarget`、`powerButtonMinHeight = 92`。
3. 为策略增加 `Control` 枚举，覆盖 `.modelSelector` 和 `.powerButton`，并提供稳定 identifier 映射。
4. `ModelSelectorPanel` picker 背景外层使用 `modelSelectorMinHeight`。
5. `DeploymentPowerButton` 使用 `powerButtonMinHeight` 替代裸写 `92`。
6. 在 `LocalGemmaTests.swift` 新增 `testModelDeploymentControlLayoutPolicyMaintainsTouchTargets`，测试数从 88 增至 89。
7. 更新 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md`。
