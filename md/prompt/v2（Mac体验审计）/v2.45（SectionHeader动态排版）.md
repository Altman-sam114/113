# v2.45 Agent B 提示词：SectionHeader 动态排版

## 版本号与依据

- 版本号：v2.45。
- 分配依据：`update_log.md` 最新正式实现版本为 v2.44，本轮继续 Agent X 的 UI、Mac、iPad 体验迭代，并处理 v2.44 遗留的 `SectionHeader` Dynamic Type 多行策略。

## 背景

当前 `SectionHeader` 被提示词页、模型页、设置页和优化区复用，但仍使用固定 `.system(size:)` 字号，并把主标题限制为单行再压缩。该写法不利于 iPad/Mac 窄 split view 和较大 Dynamic Type 设置，也和项目已为提示词模板、分类 chip 建立的语义字体策略不一致。

## 目标

- 新增 `SectionHeaderTextLayoutPolicy`，集中定义共享小节标题的文本行数、间距和多行能力。
- 将 `SectionHeader` 的 eyebrow、title、subtitle 改为 SwiftUI 语义字体。
- title 允许两行，subtitle 允许多行，避免通过 `minimumScaleFactor` 压缩主标题。
- 新增纯 policy XCTest，锁住 Dynamic Type 多行排版策略。
- 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md`、`update_log.md`。

## 非目标

- 不改变工作区切换、会话、composer、模型 artifact、runtime、verified 门禁或 command menu。
- 不改变现有宽屏内容宽度策略、触控目标策略或辅助语义 metadata。
- 不新增真实模型推理、模型下载、云端推理或原生 macOS target。
- 不做大文件拆分或无关 SwiftUI 重构。

## 架构依据

- `ContentView.swift` 负责 UI 组合，`SectionHeader` 是提示词页、模型页、设置页和优化区共用标题组件。
- 既有 `PromptTemplateTextLayoutPolicy`、`PromptCategoryTextLayoutPolicy` 已证明本项目用纯 policy + XCTest 锁住 Dynamic Type 文本策略。
- `md/flow/flow.md` 要反映当前真实 UI 策略；`md/test/test.md` 要反映当前测试数量和触发条件。

## 实现步骤

1. 基于最新 `origin/main` 同步 `main`，确认工作区无无关改动。
2. 在 `ContentView.swift` 新增 `SectionHeaderTextLayoutPolicy`。
3. 更新 `SectionHeader`：
   - eyebrow 使用 `.caption.weight(.black)`、单行和稳定 tracking。
   - title 使用 `.title2.weight(.heavy)`、两行和垂直自适应。
   - subtitle 使用 `.subheadline.weight(.medium)`、三行、line spacing 和垂直自适应。
4. 在 `LocalGemmaTests.swift` 新增 `testSectionHeaderTextLayoutPolicySupportsDynamicTypeHeadings`。
5. 同步文档与版本记录。
