# v2.20 / 优化指标网格宽度策略

## 版本号和分配依据

- 版本号：`v2.20`
- 分配依据：`update_log.md` 最新正式版本为 `v2.19 / 模型文件工作流面板辅助语义`，本轮继续递增到 `v2.20`。
- 所属总目标：Agent X 持续优化 UI、Mac 和 iPad 体验。

## 背景

设置页和独立优化 dashboard 的 Apple Silicon 指标区域当前使用固定双列 `LazyVGrid`。在 iPhone、窄 split view 或 Mac Catalyst 窄窗口中，两列指标卡容易压缩卡片文本；在 iPad 和 Mac 宽窗口中，双列仍然适合快速扫描。需要一个小型、可测试的布局策略锁住窄屏单列、宽屏双列，避免后续迭代破坏 Mac/iPad 与窄屏体验。

## 目标

1. 新增 `OptimizerMetricGridLayoutPolicy`，集中定义优化指标网格的最小卡片宽度、间距、最大列数和宽度断点。
2. 新增或调整视图，让 `SettingsWorkspace` 和 `OptimizerDashboard` 共用同一套指标网格布局。
3. 窄屏或窄分栏使用单列；达到策略阈值的 iPad/Mac 宽区域使用双列。
4. 新增 XCTest 锁住 1/2 列策略、阈值边界、最大列数和 GridItem 数量。
5. 同步 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 非目标

- 不修改 `DeviceOptimizer` 指标数据、开关状态、准备度计算或隐私保护摘要。
- 不修改 `OptimizerMetricAccessibilityMetadata`、`OptimizationToggleAccessibilityMetadata` 或已有辅助语义文案。
- 不修改模型、artifact、runtime、会话、导出、壁纸或提示词行为。
- 不新增 UI Test target。
- 不下载模型权重，不接入云端推理，不接入真实 runtime，不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `SettingsWorkspace` 和 `OptimizerDashboard` 各自声明固定双列 `LazyVGrid(columns: [GridItem(.flexible()), GridItem(.flexible())])`。
  - `OptimizerMetricCard` 已有稳定辅助语义，本轮只调整网格布局策略。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试基线为 63 个 XCTest 方法。
  - 已有 `testOptimizerMetricCardsExposeAccessibilityMetadata`，但没有锁住指标网格的窄屏/宽屏列数策略。

## 实现步骤

1. 在 `ContentView.swift` 中新增 `OptimizerMetricGridLayoutPolicy`。
2. 新增 `OptimizerMetricGrid` 或等价共享视图，复用 `OptimizerMetricCard` 渲染指标。
3. 将 `SettingsWorkspace` 和 `OptimizerDashboard` 的固定双列网格替换为共享网格。
4. 策略建议：
   - 最大列数为 2。
   - 最小卡片宽度约 170。
   - 两列阈值为 `minimumCardWidth * 2 + spacing`。
   - 使用 `ViewThatFits` 或等价 SwiftUI 写法保证窄屏回退单列。
5. 在 `LocalGemmaTests.swift` 新增 `testOptimizerMetricGridLayoutPolicyUsesSingleColumnOnNarrowSettingsWidth`，测试函数数从 63 增加到 64。

## 风险和禁止项

- 风险：网格策略必须同时用于设置页和优化 dashboard，避免两处布局漂移。
- 风险：窄屏单列会改变指标卡垂直长度；本轮只改变排列，不改变内容和状态。
- 禁止下载模型权重、云端推理、绕过 verified 门禁。
- 禁止把未跑测试写成已通过。
- 禁止验收旧 run 或旧 artifact。
