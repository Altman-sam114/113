# v2.12（优化指标卡辅助语义）Agent B 实现提示词

## 版本号和分配依据

- 版本号：`v2.12`
- 分配依据：`update_log.md` 最新正式版本为 `v2.11 / Composer控件标识与语音控制语义`。Agent X 继续执行“优化 UI、Mac、iPad 版”的总目标；v2.10 和 v2.11 遗留事项均明确还有“优化指标卡辅助语义”未完成，本轮递增到 `v2.12`。

## 背景

设置页和优化 dashboard 复用 `OptimizerMetricCard` 展示 Apple Silicon 部署优化指标：

- `Neural Engine`
- `Metal Kernels`
- `Memory Budget`
- `Battery Profile`

当前这些卡片只有可见文本、色点和 `ProgressView`，缺少整体 VoiceOver / Voice Control 语义，也没有稳定 identifier。Mac Catalyst、iPad 外接键盘和后续 UI 自动化需要能稳定定位这些只读指标卡，并让辅助技术一次读出指标名称、状态、进度、说明和本地边界。

## 目标

1. 新增 `OptimizerMetricAccessibilityMetadata`，为每个 `OptimizerMetric` 生成：
   - label
   - value
   - hint
   - Voice Control input labels
   - stable identifier
2. `OptimizerMetricCard` 接入整体辅助语义：
   - 使用 `.accessibilityElement(children: .combine)` 或等价只读合并语义。
   - 不给指标卡增加按钮 trait。
   - 不改变可见 UI、网格布局、颜色、进度条或卡片高度。
3. `value` 必须合并：
   - 指标名称
   - 当前状态值
   - progress 百分比
   - detail 描述
4. `hint` 必须明确：
   - 这是本地 Apple Silicon 优化指标摘要。
   - 不下载模型权重。
   - 不启动真实 runtime。
   - 不发送到云端服务。
   - 不绕过 artifact verified 门禁。
5. 新增 XCTest，测试默认 4 个指标的 metadata、百分比 clamp、identifier、input labels、本地/云端/verified 边界；测试函数数从 55 增加到 56。

## 非目标

- 不修改 `DeviceOptimizer.metrics` 默认内容、顺序、progress 或 tint。
- 不修改 `DeviceOptimizer.toggle`、`deploymentReadiness` 公式或 `OptimizationSwitch` 行为。
- 不修改 `ChipReadinessCard`、`ReadinessRing` 或运行策略开关行为。
- 不修改模型 artifact validation、runtime plan、`InferenceEngine`、会话、提示词或导出逻辑。
- 不创建原生 macOS target。
- 不下载 Gemma 或任何模型权重。
- 不接入云端推理、外部模型服务或真实 Core ML 推理。
- 不新增 UI Test target，不做无关重构。

## 当前架构依据

- `md/flow/flow.md`：`DeviceOptimizer` 是 Apple Silicon 优化状态对象；设置页和优化 dashboard 读取它展示指标、开关和准备度，不能写入模型 artifact 或 runtime 状态。
- `LocalGemma/AppState.swift`：
  - `OptimizerMetric` 包含 `label`、`value`、`detail`、`progress`、`tint`。
  - `DeviceOptimizer.metrics` 默认提供 4 个稳定指标。
- `LocalGemma/ContentView.swift`：
  - 设置页和优化 dashboard 都通过 `ForEach(optimizer.metrics)` 渲染 `OptimizerMetricCard(metric:)`。
  - `OptimizerMetricCard` 当前缺整体辅助语义。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - `testChipReadinessCardDescribesPrivacyGuardAndAccessibilityMetadata`
  - `testOptimizationToggleRowsExposeAccessibilityMetadata`
  可作为 metadata 测试模板。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 中新增 `OptimizerMetricAccessibilityMetadata`。
2. 推荐接口：
   - `static let hint`
   - `static func percent(for progress: Double) -> Int`
   - `static func label(for metric: OptimizerMetric) -> String`
   - `static func value(for metric: OptimizerMetric) -> String`
   - `static func inputLabels(for metric: OptimizerMetric) -> [String]`
   - `static func identifier(for metric: OptimizerMetric) -> String`
3. `percent` 必须 clamp 到 `0...100`，避免异常 progress 影响辅助 value。
4. `identifier` 推荐使用稳定 slug：
   - `optimizer-metric-neural-engine`
   - `optimizer-metric-metal-kernels`
   - `optimizer-metric-memory-budget`
   - `optimizer-metric-battery-profile`
5. 在 `OptimizerMetricCard` 上接入 label/value/hint/input labels/identifier。
6. 新增 `testOptimizerMetricCardsExposeAccessibilityMetadata`：
   - 断言默认指标数为 4。
   - 断言 4 个 identifier。
   - 断言 percent clamp：`0.62 -> 62`、负数为 0、大于 1 为 100。
   - 断言 label/value 包含指标名、状态、detail、百分比。
   - 断言 input labels 非空且包含指标名。
   - 断言 hint 包含本地 Apple Silicon 优化指标、不下载权重、不启动真实 runtime、不发送云端服务、不绕过 verified 门禁。
7. 同步文档：
   - `AGENTS.md`
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`

## Agent C 验收标准

Agent C 必须只验收最新 `origin/main` commit 对应的 run 和 artifact：

1. `git fetch origin`
2. 核对 `git rev-parse origin/main` 等于 v2.12 commit。
3. `gh run list --workflow ci-results.yml --branch main --limit 5` 找到最新 run。
4. `gh run download <run_id> --dir /private/tmp/localgemma-c-review-<run_id>`
5. 核对：
   - manifest `branch=main`
   - manifest `version=v2.12`
   - manifest `commitSha` 等于 `origin/main`
   - manifest `runId` / `runAttempt` 等于下载 run
   - `artifactName` 等于 `artifact-name.txt`
   - required outcomes：static、logic、iOS build、XCTest、Mac Catalyst build、Mac run script 均为 success
   - optional Codex Run environment 允许 skipped 但必须有 skipped reason
   - `junit.xml` failures 为 0
   - `test.log` 包含 `testOptimizerMetricCardsExposeAccessibilityMetadata()` 通过记录
   - `xcodebuild.log`、`test.log`、`mac-catalyst-build.log` 有成功记录
   - `.xcresult` Info.plist 可通过 `plutil -lint`
