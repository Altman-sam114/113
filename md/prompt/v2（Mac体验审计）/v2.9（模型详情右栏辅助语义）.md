# v2.9（模型详情右栏辅助语义）Agent B 实现提示词

## 版本号和分配依据

- 版本号：`v2.9`
- 分配依据：`update_log.md` 最新正式版本为 `v2.8 / 模型胶囊整体辅助语义`，其遗留事项明确“没有做模型详情右栏辅助语义”。本轮按 Agent X 总目标“持续优化 UI、Mac、iPad 版”继续递增到 `v2.9`。

## 背景

当前模型页已经具备：

- `ModelLibraryLayoutMode`：宽屏 iPad / Mac Catalyst 内部双栏，左侧为选择、部署和文件操作，右侧为模型详情。
- `ModelDeploymentControlAccessibilityMetadata`：模型选择器、部署电源和 artifact 操作按钮的稳定辅助语义。
- `ModelCapsuleAccessibilityMetadata`：顶部模型胶囊整体状态辅助语义。

缺口是宽屏模型页右侧详情区仍由多个视觉面板分散表达模型摘要、参数、性能和建议，VoiceOver / Voice Control 没有一个可稳定命名、可测试、能说明 verified 门禁和本地边界的整体入口。

## 目标

1. 新增 `ModelDetailAccessibilityMetadata`，为模型详情右栏生成稳定的：
   - label
   - value
   - hint
   - Voice Control input labels
   - accessibility identifier
2. `value` 必须聚合当前选中模型详情：
   - 模型名、家族、参数量、量化格式、上下文长度、文件格式、包体大小
   - artifact missing / staged / verified 状态和 validation summary
   - 预计速度、内存预算、主后端、回退后端、KV cache
   - runtime preparation 的 blocker / next step 摘要
3. `hint` 必须说明：
   - 这是本地模型详情摘要
   - 不会下载模型权重
   - 不会启动真实 runtime
   - 不会发送到云端服务
   - 不会绕过 verified 门禁
4. 在 `ModelLibraryView` 的模型详情右栏容器接入该 metadata；窄屏单栏下也应让模型详情区域具备同一整体辅助语义，避免 iPad 分屏或窄窗口退化。
5. 新增 XCTest 锁住 metadata 文案和状态覆盖；测试函数数从 53 增加到 54。

## 非目标

- 不创建原生 macOS target。
- 不下载 Gemma 或任何模型权重。
- 不接入云端推理、外部模型服务或真实 Core ML 推理。
- 不修改 `ModelCatalog`、`LocalArtifactValidator`、`InferenceEngine`、`LocalRuntimePlanner` 的状态流和 verified 门禁。
- 不改变模型页可见布局、按钮行为、文件导入、模拟暂存、卸载或扫描逻辑。
- 不新增 UI Test target，不做无关重构。

## 当前架构依据

- `md/flow/flow.md`：模型页由 `ModelLibraryLayoutMode` 控制内部单栏/双栏，状态来源仍为 `ModelCatalog`、`LocalArtifactValidator` 和 `LocalRuntimePlanner`。
- `LocalGemma/ContentView.swift`：
  - `ModelLibraryView` 计算 `model`、`validation`、`report` 和 `deploymentState`。
  - 宽屏时右侧 VStack 包含 `ModelSummaryPanel`、`ModelParametersPanel`、`ModelPerformancePanel`、`ModelAdvicePanel`。
  - 窄屏时相同详情面板在控制面板之后顺序展示。
  - 现有 metadata enum 放在同文件顶部附近，遵循纯函数、可测试、不依赖 SwiftUI view state 的模式。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - 现有 `testModelCapsuleExposesOverallAccessibilityMetadata`、`testModelSelectorExposesAccessibilityMetadata`、`testModelDeploymentControlsExposeAccessibilityMetadata` 可作为测试风格参考。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 新增 `ModelDetailAccessibilityMetadata`。
2. 推荐接口：
   - `static let identifier = "model-detail-summary"`
   - `static func label(model: LocalModel) -> String`
   - `static func value(model: LocalModel, validation: ArtifactValidationResult, report: RuntimePreparationReport) -> String`
   - `static func hint(validation: ArtifactValidationResult, report: RuntimePreparationReport) -> String` 或稳定 `hint`
   - `static func inputLabels(model: LocalModel) -> [String]`
   - 必要的 `artifactDescription`、`runtimeReadinessDescription`、`blockingSummary` 等小 helper。
3. 将右侧详情面板抽成一个小的 `ModelDetailColumn` view 或在现有 VStack 容器上接入 metadata。若新增 view，保持同文件局部实现即可，避免修改 Xcode target。
4. 宽屏双栏右侧和窄屏单栏详情区必须复用同一接入点，避免语义分叉。
5. 不改变现有视觉 panel 的文本、排序和业务行为。
6. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增 `testModelDetailColumnExposesAccessibilityMetadata`：
   - 覆盖 label / identifier / input labels。
   - 覆盖 missing、staged、verified value 中的模型参数、artifact 状态、validation summary、后端、KV cache、速度、内存、blocker / next step。
   - 覆盖 hint 包含“不下载模型权重”“不会启动真实 runtime”“不会发送到云端服务”“verified 门禁”。
7. 同步文档：
   - `AGENTS.md`
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`

## Agent C 验收标准

Agent C 必须只验收最新 `origin/main` commit 对应的 run 和 artifact：

1. `git fetch origin`
2. 核对 `git rev-parse origin/main` 等于 v2.9 commit。
3. `gh run list --workflow ci-results.yml --branch main --limit 5` 找到最新 run。
4. `gh run download <run_id> --dir /private/tmp/localgemma-c-review-<run_id>`
5. 核对：
   - manifest `branch=main`
   - manifest `version=v2.9`
   - manifest `commitSha` 等于 `origin/main`
   - manifest `runId` / `runAttempt` 等于下载 run
   - `artifactName` 等于 `artifact-name.txt`
   - required outcomes：static、logic、iOS build、XCTest、Mac Catalyst build、Mac run script 均为 success
   - optional Codex Run environment 允许 skipped 但必须有 skipped reason
   - `junit.xml` failures 为 0
   - `xcodebuild.log`、`test.log`、`mac-catalyst-build.log` 有成功记录
   - `.xcresult` Info.plist 可通过 `plutil -lint`

- 禁止伪造测试或云端验收。
- 禁止把旧 run / 旧 artifact 冒充 v2.9。
- 禁止无关重构或删除其他 Agent / 用户改动。
