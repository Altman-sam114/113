# v2.41 Agent B 提示词：模型页整体宽屏内容宽度

## 版本号和分配依据

- 版本号：`v2.41`
- 分配依据：`update_log.md` 当前最新正式版本为 `v2.40: 优化会话 Chip 选择触控目标`，本轮继续 Agent X 的 Mac/iPad/UI 体验审计，递增为 `v2.41`。

## 背景

模型页已有 `ModelLibraryLayoutMode` 控制内部单栏/双栏，`ModelDetailColumnLayoutPolicy` 限制双栏右侧详情阅读宽度。但 `ModelLibraryView` 外层仍把 `GeometryReader` 的完整 `proxy.size` 传给 `deploymentContent`，超宽 Mac Catalyst 窗口中模型页内容会占用整行并靠左，整体工作区没有像设置页和提示词页一样居中并限制最大宽度。

## 目标

1. 新增模型页整体宽屏内容宽度策略，让模型页标题、选择/部署/文件操作和模型详情作为整体在 iPad/Mac 超宽窗口中居中并限制最大宽度。
2. 保留 iPhone 和窄 split view 原有可用宽度。
3. 保留 `ModelLibraryLayoutMode` 双栏断点、控制列宽和 `ModelDetailColumnLayoutPolicy` 详情列最大阅读宽度。
4. 增加 XCTest 锁住策略常量、窄屏回退、iPad/Mac 宽屏封顶和无效宽度 clamp。
5. 同步 README、测试规范、核心流程、流程图、入口记忆和更新日志。

## 非目标

- 不改变模型选择、部署启动/关闭、模拟暂存、扫描、导入、卸载确认或 artifact 校验状态流。
- 不改变模型页辅助语义、状态徽章、详情行级语义、模型文件工作流面板语义或 verified 门禁。
- 不改变 `ModelDetailColumnLayoutPolicy` 已有最小/最大阅读宽度策略。
- 不下载模型权重，不接入云端推理，不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `ModelLibraryView` 外层通过 `GeometryReader` 和 `ScrollView` 渲染模型页。
  - `ModelLibraryLayoutMode` 控制模型页内部单栏/双栏。
  - `ModelDetailColumnLayoutPolicy` 控制双栏右侧详情列宽度。
  - `SettingsWorkspaceLayoutPolicy` 和 `PromptTemplatesWorkspaceLayoutPolicy` 是现有整体宽屏内容宽度策略参考。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 已有 `testModelLibraryLayoutModeSupportsWideModelWorkflows`。
  - 已有 `testModelDetailColumnLayoutPolicyConstrainsWideReadingWidth`。
  - 可新增模型页整体宽度策略测试。

## 实现步骤

1. 新增 `ModelLibraryWorkspaceLayoutPolicy`：
   - `horizontalPadding = 18`
   - `minimumReadableWidth = 320`
   - `maximumContentWidth = ModelLibraryLayoutMode.maximumControlColumnWidth + ModelDetailColumnLayoutPolicy.interColumnSpacing + ModelDetailColumnLayoutPolicy.maximumReadableWidth`
   - `contentWidth(forContainerWidth:)` 复用设置页/提示词页模式：无效宽度返回最小可读宽度；窄屏返回扣除左右 padding 后宽度；宽屏封顶。
2. 调整 `ModelLibraryView`：
   - `ScrollView` 内层内容先计算 `contentWidth`。
   - `deploymentContent` 使用内容宽度构造 layout size，让内部双栏断点基于实际内容区域。
   - 外层 `.frame(width: contentWidth)` 并保留现有 padding 语义。
3. 新增测试：
   - 断言最大内容宽度从控制列最大值、列间距和详情最大阅读宽度派生。
   - 断言 320/390 窄屏扣 padding，834 iPad 使用可用宽度，1280/1600 Mac 封顶。
   - 断言无效宽度返回 `minimumReadableWidth`。
   - 断言最大内容宽度足以触发 `ModelLibraryLayoutMode.twoColumn`，且封顶后详情列仍可达到 `ModelDetailColumnLayoutPolicy.maximumReadableWidth`。
4. 同步文档：
   - `AGENTS.md`
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`
