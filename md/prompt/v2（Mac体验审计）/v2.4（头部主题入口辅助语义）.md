# v2.4 / 头部主题入口辅助语义

## 版本号和分配依据

- 版本号：`v2.4`
- 分配依据：`update_log.md` 当前最新正式记录为 `v2.3 / 壁纸控件辅助语义`，本轮在同一条 Mac/iPad 辅助语义审计线上继续递增。

## 背景

当前 `LocalGemma/ContentView.swift` 已为模型选择器、部署控件、提示词筛选、会话栏操作、导出弹层和壁纸控件建立可测试的辅助语义。但全局头部的主题切换和打开模型库按钮仍存在明显缺口：

- `HeaderView` 的主题切换按钮只有英文 `Toggle theme` label。
- `HeaderView` 的打开模型库按钮只有英文 `Open model library` label。
- `ThemePreferencePanel` 的主题切换按钮是 icon-only button，没有稳定 label/value/hint、Voice Control 输入标签或 identifier。

这些按钮是 iPad 和 Mac Catalyst 大屏工作台中的高频入口，应该与已有中文辅助语义体系保持一致。

## 目标

1. 新增一个小型 metadata 类型，例如 `HeaderActionAccessibilityMetadata` 或 `ThemePreferenceAccessibilityMetadata`，为以下动作提供稳定辅助语义：
   - 切换外观主题。
   - 打开模型库 / 模型工作区。
2. 主题切换语义必须表达：
   - 当前主题是暗色或亮色。
   - 激活后会切换到另一个主题。
   - 只改变本地 UI 外观，不下载模型权重、不启动真实 runtime、不发送到云端服务。
3. 打开模型库语义必须表达：
   - 动作会切换到模型工作区。
   - 可继续管理本地模型、artifact 和部署状态。
   - 不下载模型权重、不启动真实 runtime、不绕过 verified 门禁。
4. 将 metadata 应用到：
   - `HeaderView` 主题按钮。
   - `HeaderView` 模型库按钮。
   - `ThemePreferencePanel` 主题按钮。
5. 新增 XCTest 锁住 label/value/hint/input labels/identifier，当前测试数量应从 48 增加到 49。
6. 同步更新 `AGENTS.md`、`README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 非目标

- 不改视觉布局、颜色系统或主题持久化 key。
- 不改 `AppThemeMode` 的业务语义，除非只是增加只读展示 helper。
- 不改 `ModelCatalog`、artifact 扫描、模型导入、下载模拟、部署状态流或 runtime 计划。
- 不接入真实模型推理，不下载模型权重，不创建原生 macOS target。
- 不在本轮处理工作区导航 hint/input labels 或运行策略开关辅助语义；这些可作为后续 Agent X 轮次目标。

## 当前架构依据

- `md/flow/flow.md` 规定 `ContentView` 负责 UI，不能绕过状态层直接改核心状态。
- `WorkspaceTab.shortcutKey`、工作区 command menu 和现有辅助语义已锁住 Mac/iPad 工作区导航。
- `WallpaperPreferenceAccessibilityMetadata`、`ExportSessionActionAccessibilityMetadata`、`SessionBarActionAccessibilityMetadata`、`ModelDeploymentControlAccessibilityMetadata` 和 `PromptCategoryAccessibilityMetadata` 是本轮 metadata 命名与测试风格参考。
- `HeaderView` 只通过传入闭包执行 `toggleTheme` 和 `showModels`，本轮只补按钮辅助语义，不改变闭包调用行为。
- `ThemePreferencePanel` 只通过传入闭包执行 `toggleTheme`，本轮只补按钮辅助语义。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 中新增 metadata 类型。
2. 为主题切换提供：
   - `label(themeMode:)`
   - `value(themeMode:)`
   - `hint(themeMode:)`
   - `inputLabels(themeMode:)`
   - `identifier`
3. 为打开模型库提供：
   - `label`
   - `value`
   - `hint`
   - `inputLabels`
   - `identifier`
4. 将 `HeaderView` 的两个 icon-only 按钮接入 metadata，替换英文 label。
5. 将 `ThemePreferencePanel` 的主题按钮接入同一套主题 metadata。
6. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增 `testHeaderAndThemePreferenceActionsExposeAccessibilityMetadata`：
   - 验证主题按钮 identifier 稳定。
   - 验证暗色/亮色 value 分别包含当前主题和切换目标。
   - 验证主题 hint 包含本地 UI、不下载模型权重、真实 runtime、不发送到云端服务。
   - 验证主题 input labels 包含“切换主题”或等价中文命令。
   - 验证模型库 label/value/hint/input labels/identifier 非空且 hint 包含模型工作区、不下载模型权重、真实 runtime、verified 门禁。
7. 更新文档中的当前测试数量、功能范围和流程说明。
