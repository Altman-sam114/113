# v2.33（提示词页整体宽屏内容宽度）

## 版本号与分配依据

- 版本号：`v2.33`
- 分配依据：`update_log.md` 最新正式版本为 `v2.32 / 设置页图标动作触控目标`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版”下继续递增。

## 背景

v2.21、v2.25、v2.28、v2.29 和 v2.30 已分别优化提示词模板网格、模板动作触控目标、分类筛选换行、模板文本动态排版和分类文本动态排版。只读子 agent 继续审计提示词页后指出，`PromptTemplatesWorkspace` 外层内容目前只有固定水平 padding，Mac Catalyst / iPad 超宽窗口下标题、分类筛选和模板网格整体没有最大内容宽度；内部卡片虽然由 `PromptTemplateGridLayoutPolicy` 限制列宽，但页面整体仍会吃满工作区，影响宽屏扫描和阅读。

## 目标

1. 新增提示词页整体宽屏内容布局策略，集中定义提示词页外层水平 padding、最小可读宽度和最大内容宽度。
2. 让 `PromptTemplatesWorkspace` 的标题、分类筛选和模板网格作为整体在宽屏下居中并限制最大宽度。
3. 最大内容宽度应从现有模板网格四列最大卡片宽度派生，避免与 `PromptTemplateGridLayoutPolicy` 脱节。
4. iPhone 和窄 split view 保持原有可用宽度；iPad/Mac 宽区域限制超宽内容行。
5. 新增 XCTest 锁住提示词页整体内容宽度策略、无效宽度 clamp 和与四列模板网格最大宽度的关系。
6. 同步 README、测试规范、核心流程文档、流程图、入口规则和更新日志。

## 非目标

- 不改变提示词模板数据、分类、筛选状态、模板卡片内容或排序。
- 不改变 `PromptCategorySelector` 换行布局、分类辅助语义、`.isSelected` 语义或 Voice Control 输入标签。
- 不改变模板“填入”和“发送”动作辅助语义、生成中禁用、填入后聚焦输入、直接发送状态流或 verified 门禁说明。
- 不改变 `PromptTemplateGridLayoutPolicy` 的列数阈值、最小/最大卡片宽度或卡片动作触控目标，除非为派生最大内容宽度新增只读 helper。
- 不新增真实模型 runtime，不下载模型权重，不接入云端推理。
- 不创建原生 macOS target，不新增 UI Test target。

## 当前架构依据

- `md/flow/flow.md` 中提示词页由 `PromptTemplatesWorkspace`、`PromptCategorySelector`、`PromptTemplateGrid` 和 `PromptTemplateCard` 组成。
- `PromptTemplateGridLayoutPolicy` 控制模板卡片网格列数、最小/最大卡片宽度和 spacing。
- `PromptCategoryLayoutPolicy` 控制分类筛选 chip 换行布局。
- `PromptTemplateActionAccessibilityMetadata` 控制模板填入/发送动作辅助语义；本轮不得改变这些语义。
- 当前测试覆盖网格列数、模板文本、分类筛选、模板动作触控目标和动作辅助语义，但没有锁住提示词页外层整体宽屏内容宽度。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的提示词页布局策略附近新增 `PromptTemplatesWorkspaceLayoutPolicy`。
2. 定义 `horizontalPadding = 18`、`minimumReadableWidth = 320`、`maximumContentWidth`，其中 `maximumContentWidth` 从 `PromptTemplateGridLayoutPolicy.maximumWidth(forColumnCount: 4)` 或等价 helper 派生。
3. 为 `PromptTemplateGridLayoutPolicy` 增加只读 `maximumWidth(forColumnCount:)` helper，按最大卡片宽度和 spacing 计算列总宽。
4. 让 `PromptTemplatesWorkspace` 用 `GeometryReader` 读取容器宽度，并将 `ScrollView` 内的 `VStack` 设置为 `PromptTemplatesWorkspaceLayoutPolicy.contentWidth(forContainerWidth:)`，保持 `.frame(maxWidth: .infinity)` 外层居中。
5. 保留 `apply` 只调用 `inference.applyTemplate` + `openChat(.applyTemplate)`，保留 `send` 通过 `validation.availability` 调用 `inference.useTemplate`。
6. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增 `testPromptTemplatesWorkspaceLayoutPolicyConstrainsWidePromptContent`，覆盖 iPhone、iPad、Mac 宽窗口、负数/NaN clamp 和最大宽度派生关系；测试函数数从 76 更新到 77。
7. 同步文档和日志，将提示词页整体宽屏内容宽度纳入当前架构边界和测试基线。
