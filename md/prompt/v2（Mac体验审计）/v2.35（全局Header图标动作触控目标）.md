# Agent B 提示词：v2.35 全局 Header 图标动作触控目标

## 版本号和依据

- 版本号：v2.35。
- 版本分配依据：`update_log.md` 最新正式版本为 v2.34，本轮按顺序递增。
- 总目标 X 当前小目标：继续优化 UI、Mac 和 iPad 体验，修正全局 Header 可点击图标动作低于 44pt 触控目标的问题。

## 背景

v2.31、v2.32 已分别为会话栏和设置页图标动作建立 44pt 触控目标策略。并发只读子 agent 指出全局 Header 的主题切换和打开模型工作区按钮仍硬编码为 `42x42`，低于 Apple 推荐的 44pt 最小触控目标。这两个入口在推理、模型、提示词和设置工作区顶部都可见，影响 iPhone、iPad 和 Mac Catalyst 用户的全局导航体验。现有 `HeaderActionAccessibilityMetadata` 只覆盖辅助语义，没有测试布局尺寸。

## 目标

1. 新增 `HeaderActionLayoutPolicy`，集中定义全局 Header 图标动作最小触控目标。
2. 将全局 Header 主题切换和打开模型工作区按钮从 42pt 提升到 44pt。
3. 保留现有 Header 辅助语义、主题切换、工作区切换、模型/runtime/artifact 状态和 `ModelCapsule` 装饰图标。
4. 新增 XCTest 锁住 Header 图标动作触控目标策略。
5. 同步 README、测试规范、核心流程文档、流程图、AGENTS 和 update_log。

## 非目标

- 不改 `HeaderActionAccessibilityMetadata` 文案。
- 不改 `selectedTab`、workspace command menu、会话 command menu 或 composer focus 状态流。
- 不改模型文件、artifact 校验、runtime、真实推理、模型下载或云端服务。
- 不改 `ModelCapsule` 的 42pt 非按钮装饰图标。
- 不新增原生 macOS target、完整 UI Test target 或大体积测试数据。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `HeaderView` 内有两个全局 Header 图标按钮：主题切换和打开模型工作区。
  - `HeaderActionAccessibilityMetadata` 只控制 label/value/hint/input labels/identifier。
  - `SessionBarActionLayoutPolicy` 与 `SettingsIconActionLayoutPolicy` 是本轮布局策略命名和测试模式参考。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试函数数为 78。
  - 已有 Header 辅助语义测试，但没有 Header 图标动作触控目标测试。

## 实现步骤

1. 在开始前执行：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short --branch`
2. 在 `HeaderActionAccessibilityMetadata` 附近新增 `HeaderActionLayoutPolicy`：
   - `enum Action: CaseIterable { case toggleTheme, openModelLibrary }`
   - `minimumTouchTarget = 44`
   - `iconButtonSize = minimumTouchTarget`
   - `usesMinimumTouchTarget(for:)`
3. 替换 `HeaderView` 中两个 `.frame(width: 42, height: 42)` 为 `HeaderActionLayoutPolicy.iconButtonSize`。
4. 新增 `testHeaderActionLayoutPolicyMaintainsTouchTargets`：
   - 最小触控目标为 44。
   - icon button size 大于等于最小触控目标。
   - action 覆盖两个 Header 可点击图标动作。
   - 所有 action 都满足最小触控目标。
5. 更新文档，把测试函数数从 78 改为 79，并加入 `HeaderActionLayoutPolicy` 说明。
