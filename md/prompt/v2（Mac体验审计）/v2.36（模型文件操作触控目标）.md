# Agent B 提示词：v2.36 模型文件操作触控目标

## 版本号和依据

- 版本号：v2.36。
- 版本分配依据：`update_log.md` 最新正式版本为 v2.35，本轮按顺序递增。
- 总目标 X 当前小目标：继续优化 UI、Mac 和 iPad 体验，修正模型文件工作流面板中扫描和导入 utility 按钮缺少 44pt 触控目标策略的问题。

## 背景

v2.31、v2.32 和 v2.35 已分别为会话栏、设置页图标动作和全局 Header 图标动作建立 44pt 触控目标策略。模型页 `ArtifactActionPanel` 中的“下载模型”和“卸载模型”是大卡片按钮，但“扫描本地”和“导入文件”仍只依赖 `compactUtilityStyle()` 的垂直 padding，缺少可测试的最小触控目标策略。这两个入口是 iPad/Mac 模型文件管理工作流的高频动作，需要与既有触控目标基线一致。

## 目标

1. 新增 `ModelArtifactActionLayoutPolicy`，集中定义模型文件工作流 utility 动作的最小触控目标。
2. 将“扫描本地”和“导入文件”按钮显式保持至少 44pt 高度。
3. 保留现有模型文件工作流辅助语义、Files 导入、扫描本地、模拟暂存、卸载确认和 artifact verified 门禁。
4. 新增 XCTest 锁住模型文件 utility 动作触控目标策略。
5. 同步 README、测试规范、核心流程文档、流程图、AGENTS 和 update_log。

## 非目标

- 不改 `ModelDeploymentControlAccessibilityMetadata` 或 `ModelArtifactPanelAccessibilityMetadata` 文案。
- 不改下载/模拟暂存、卸载确认、扫描、Files 导入或 `ModelCatalog` 状态流。
- 不改模型详情宽屏布局、模型选择器、状态徽章或部署电源按钮。
- 不下载模型权重，不接入真实 runtime，不引入云端推理。
- 不新增原生 macOS target、完整 UI Test target 或大体积测试数据。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `ArtifactActionPanel` 负责模型文件工作流面板。
  - `ArtifactActionButton` 覆盖下载和卸载两张大按钮。
  - 扫描和导入按钮目前使用 `compactUtilityStyle()`，但没有独立布局策略。
  - `ModelDeploymentControlAccessibilityMetadata.ArtifactAction` 已覆盖 `.scan` 和 `.importFiles`，可复用为布局策略 action 枚举的覆盖关系参考。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试函数数为 79。
  - 已有模型文件工作流辅助语义测试，但没有扫描/导入 utility 按钮触控目标测试。

## 实现步骤

1. 在开始实现前执行：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short --branch`
2. 在模型部署控制辅助语义或 `ArtifactActionPanel` 附近新增 `ModelArtifactActionLayoutPolicy`：
   - `enum UtilityAction: CaseIterable { case scan, importFiles }`
   - `minimumTouchTarget = 44`
   - `utilityButtonMinHeight = minimumTouchTarget`
   - `usesMinimumTouchTarget(for:)`
3. 在 `ArtifactActionPanel` 的“扫描本地”和“导入文件”按钮 label 上添加 `minHeight`，复用 `ModelArtifactActionLayoutPolicy.utilityButtonMinHeight`。
4. 新增 `testModelArtifactActionLayoutPolicyMaintainsUtilityTouchTargets`：
   - 最小触控目标为 44。
   - utility button min height 大于等于最小触控目标。
   - action 覆盖 scan/import 两个 utility 按钮。
   - 所有 utility action 都满足最小触控目标。
   - 可选断言：布局 action 和 `ModelDeploymentControlAccessibilityMetadata.ArtifactAction` 中 scan/import 的数量/映射一致。
5. 更新文档，把测试函数数从 79 改为 80，并加入 `ModelArtifactActionLayoutPolicy` 说明。
