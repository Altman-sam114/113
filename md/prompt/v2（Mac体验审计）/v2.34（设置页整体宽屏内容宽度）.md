# Agent B 提示词：v2.34 设置页整体宽屏内容宽度

## 版本号和依据

- 版本号：v2.34。
- 版本分配依据：`update_log.md` 最新正式版本为 v2.33，本轮按顺序递增。
- 总目标 X 当前小目标：继续优化 UI、Mac 和 iPad 体验，修正设置页在 iPad/Mac 超宽窗口下整体内容横向拉满的问题。

## 背景

v2.20、v2.23 和 v2.32 已分别为设置页 Apple Silicon 指标网格、运行策略开关网格和图标动作触控目标建立策略；v2.33 为提示词页建立整体宽屏内容宽度策略。并发只读子 agent 指出 `SettingsWorkspace` 当前仍是 `ScrollView` 内部 `VStack` 加固定 18pt padding，没有页面级最大内容宽度。Mac Catalyst / iPad 超宽窗口下，设置页标题、外观面板、壁纸面板、芯片准备度卡片、指标网格和运行策略开关整体会横向拉满，阅读行宽与控件跨度不够稳定。

## 目标

1. 新增 `SettingsWorkspaceLayoutPolicy`，集中定义设置页整体水平 padding、最小可读宽度、最大内容宽度和无效容器宽度 clamp。
2. 让 `SettingsWorkspace` 在 iPhone 和窄 split view 保持原有可用宽度，在 iPad/Mac 超宽窗口中整体居中并限制最大内容宽度。
3. 保留现有主题切换、相册壁纸导入、本地压缩、恢复系统背景、optimizer toggle、局部网格策略、图标触控目标和辅助语义。
4. 新增 XCTest 锁住设置页整体宽屏内容宽度策略。
5. 同步 README、测试规范、核心流程文档、流程图、AGENTS 和 update_log。

## 非目标

- 不新增模型下载、云端推理或外部服务。
- 不接真实 Gemma runtime，不把 `RealGemmaRuntimePlaceholder` 描述成真实推理。
- 不绕过 artifact `verified` 门禁。
- 不重构设置页状态流、壁纸导入流程或 optimizer 数据结构。
- 不新增原生 macOS target、完整 UI Test target 或大体积测试数据。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `SettingsWorkspace` 定义设置页滚动内容。
  - `SettingsIconActionLayoutPolicy` 只控制设置页三个图标动作的 44pt 触控目标。
  - `OptimizerMetricGridLayoutPolicy` 和 `OptimizationToggleGridLayoutPolicy` 只控制设置页/优化 dashboard 局部网格列数。
  - `PromptTemplatesWorkspaceLayoutPolicy` 是页面级宽度策略的参考模式。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试函数数为 77。
  - 已有设置页图标动作、运行策略网格、优化指标网格和提示词页整体宽度策略测试。

## 实现步骤

1. 在开始前执行：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short --branch`
2. 在 `SettingsWorkspace` 内用 `GeometryReader` 读取容器宽度。
3. 将滚动内容 `VStack` 使用：
   - `SettingsWorkspaceLayoutPolicy.contentWidth(forContainerWidth:)`
   - `.frame(maxWidth: .infinity, alignment: .center)`
   - `SettingsWorkspaceLayoutPolicy.horizontalPadding`
4. 在 `SettingsWorkspace` 之后新增 `SettingsWorkspaceLayoutPolicy`：
   - `horizontalPadding = 18`
   - `minimumReadableWidth = 320`
   - `maximumContentWidth = 760`
   - 非有限或非正宽度返回 `minimumReadableWidth`
   - 有效宽度先扣除左右 padding；窄于 minimum 时返回可用宽度；宽屏时 clamp 到 maximum。
5. 在 `LocalGemmaTests.swift` 新增 `testSettingsWorkspaceLayoutPolicyConstrainsWideSettingsContent`：
   - 锁住 padding、minimum、maximum。
   - 锁住 maximum 足够容纳运行策略两列和优化指标两列阈值。
   - 锁住 320、390、834、1200、负数、NaN 的输出。
6. 更新文档，把测试函数数从 77 改为 78，并加入 `SettingsWorkspaceLayoutPolicy` 说明。
