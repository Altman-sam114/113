# v2.49 设置偏好行文本动态排版

## 版本号与分配依据

- 版本号：v2.49。
- 分配依据：`update_log.md` 最新正式轮次为 v2.48，Agent C 已验收 `d299798` / run `29138462221`；本轮在 Agent X “全面优化 UI、Mac 与 iPad 体验”总目标下继续递增。

## 背景

设置页 `ThemePreferencePanel` 与 `WallpaperPreferencePanel` 已通过 `SettingsIconActionLayoutPolicy` 锁住 44pt 图标触控目标，页面整体宽度也由 `SettingsWorkspaceLayoutPolicy` 约束。但两处偏好行标题/状态仍使用固定小字号，未采用 Dynamic Type 语义字体，也没有显式多行策略。iPad split view、Mac Catalyst 窄窗口和较大文字设置下，外观模式与壁纸状态说明可读性不足。

## 目标

- 新增 `SettingsPreferenceTextLayoutPolicy`，集中定义外观/壁纸偏好行标题与状态文本间距、行数和多行能力。
- 标题与状态改用 Dynamic Type 语义字体。
- 标题允许两行，状态允许两行并纵向增长。
- 新增 XCTest 锁住策略常量与多行能力，并与既有 44pt 图标策略共存。
- 同步 README、测试规范、核心流程、流程图、入口规则、更新日志和本提示词归档。

## 非目标

- 不改变主题切换、相册读取、本地压缩、恢复系统背景状态流。
- 不改变 `SettingsIconActionLayoutPolicy` 的 44pt 触控目标或 identifier。
- 不改变 `WallpaperPreferenceAccessibilityMetadata` / 主题按钮辅助语义。
- 不改变设置页整体宽度、优化指标、运行策略开关、模型文件或 runtime 状态。
- 不下载模型权重，不接入云端推理，不绕过 artifact verified 门禁，不创建原生 macOS target。
- 不在本机跑完整 Xcode / Simulator XCTest；完整验证走 GitHub Actions。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `ThemePreferencePanel` / `WallpaperPreferencePanel` 位于 `SettingsWorkspace`。
  - `SettingsIconActionLayoutPolicy` 负责图标按钮尺寸。
  - `SettingsWorkspaceLayoutPolicy` 负责页面内容最大宽度。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 已有设置页图标触控与整体宽度测试。
  - 本轮新增文本布局策略测试，测试函数数从 91 增至 92。

## 实现步骤

1. 在 `ContentView.swift` 增加 `SettingsPreferenceTextLayoutPolicy`。
2. 策略包含 `verticalSpacing`、`titleLineLimit`、`statusLineLimit` 与多行判定。
3. 标题使用语义 `headline` 黑体；状态使用语义 `subheadline` 粗体。
4. 标题/状态允许多行并使用 vertical `fixedSize`；不使用 `minimumScaleFactor` 压缩。
5. 保留图标按钮 44pt、相册/主题动作、辅助语义和设置页宽度策略。
6. 新增 `testSettingsPreferenceTextLayoutPolicySupportsDynamicTypeRows`。
7. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md`。
