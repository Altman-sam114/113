# v2.14（会话 chip 动作语义）

## 版本号和版本分配依据

- 版本号：v2.14。
- 分配依据：`update_log.md` 最新正式记录为 v2.13，本轮继续 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”，选择 v2.10-v2.13 多次遗留的推理页单个会话 chip 选择/删除动作语义作为下一轮可独立验证目标。

## 背景

推理页已经有系统 `会话` command menu、会话栏新建/导出按钮、composer 控件和聊天消息气泡的辅助语义。单个 `SessionChip` 仍只在选择按钮上暴露基础 label/value，删除按钮使用纯 `Image(systemName:)`，缺少动作 hint、Voice Control 输入标签、稳定 identifier、删除禁用原因和本地/云端/verified 边界说明。

本轮只补齐会话列表单个 chip 的选择和删除动作语义，不改变会话创建、选择、删除、导出、command menu、runtime 或布局行为。

## 目标

1. 新增 `SessionChipActionAccessibilityMetadata`。
2. 为 `SessionChip` 的选择按钮接入 label、value、hint、input labels、identifier 和选中 trait。
3. 为 `SessionChip` 的删除按钮接入 label、value、hint、input labels、identifier，并把 icon-only 纯 `Image` 改成有文本语义的 `Label(...).labelStyle(.iconOnly)`。
4. 选择动作必须说明只切换本地会话并请求 composer 输入焦点，不发送 prompt、不下载模型权重、不启动真实 runtime、不发送云端服务、不绕过 artifact verified 门禁。
5. 删除动作必须说明只删除本地会话记录，不删除模型 artifact 或权重，不发送云端服务，不改变 artifact verified 门禁。
6. 默认空白当前会话禁用删除时必须暴露清晰 value/hint 原因。
7. identifier 必须稳定且不包含会话标题或用户 prompt；建议使用 action + session UUID 前 8 位。
8. 新增 XCTest 锁住选择/删除 label、value、hint、input labels、identifier、删除禁用原因和 UUID 前缀。
9. 同步更新 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md`。

## 非目标

- 不修改 `InferenceEngine.createSession`、`selectSession`、`deleteSession`、会话命名、消息同步或导出逻辑。
- 不修改系统 `会话` command menu、`SessionCommandAction` 或 focused route。
- 不修改会话侧栏宽度策略、布局断点、可见样式、排序或滚动逻辑。
- 不新增 UI Test target。
- 不下载 Gemma 权重，不接入云端推理，不把 staged artifact 提升为 verified。
- 不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `SessionBar` 遍历 `inference.sessions` 并渲染 `SessionChip`。
  - `SessionChip` 包含选择按钮和删除按钮；删除按钮当前对默认空白当前会话禁用。
  - 已有 metadata 模式包括 `SessionBarActionAccessibilityMetadata`、`ChatMessageAccessibilityMetadata`、`PromptTemplateActionAccessibilityMetadata` 等，可复用静态 helper 设计。
- `LocalGemma/AppState.swift`
  - `ChatSession` 含 `id`、`title`、`messages`、`createdAt`、`updatedAt`。
  - `InferenceEngine.selectSession` 只切换本地 active session；`deleteSession` 只修改本地会话数组。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前基线为 57 个 `test...` 方法。
  - 新增测试建议命名为 `testSessionChipActionsExposeAccessibilityMetadata`。
- `md/flow/flow.md`
  - v2.1 的 `SessionBarActionAccessibilityMetadata` 只覆盖会话栏新建/导出按钮，本轮必须区分单个会话 chip 的选择/删除动作。
- `.github/workflows/ci-results.yml`
  - 仍负责 main push 后的静态检查、逻辑烟测、iOS build/test、Mac Catalyst build、run script contract 和 artifact 上传。

## 实现步骤

1. 在 `ContentView.swift` 的现有辅助语义 helper 区域新增 `SessionChipActionAccessibilityMetadata`。
2. 建议接口：
   - `enum Action: String, CaseIterable { case select, delete }`
   - `canDelete(session:isActive:) -> Bool`
   - `label(for:session:) -> String`
   - `value(for:session:isActive:canDelete:) -> String`
   - `hint(for:session:isActive:canDelete:) -> String`
   - `inputLabels(for:session:) -> [String]`
   - `identifier(for:session:) -> String`
3. `canDelete` 应复用现有 UI 保护逻辑：活跃、标题为 `新对话`、消息数不超过默认两条时不可删。
4. 选择按钮 value/hint 至少覆盖：
   - 当前/未选中本地会话。
   - 会话消息数量。
   - 切换本地会话并请求 composer focus。
   - 不发送 prompt、不下载模型权重、不启动真实 runtime、不发送云端服务、不绕过 verified 门禁。
5. 删除按钮 value/hint 至少覆盖：
   - 可删除当前/未选中本地会话。
   - 禁用时说明默认空白当前会话不可删除。
   - 只删除本地会话记录。
   - 不删除模型 artifact 或权重、不发送云端服务、不改变 verified 门禁。
6. 删除按钮内容改为：
   - `Label(<delete label>, systemImage: "trash.fill").labelStyle(.iconOnly)`
7. 新增 XCTest：
   - 构造固定 UUID 的默认空白会话和命名会话。
   - 覆盖 `Action.allCases == [.select, .delete]`。
   - 覆盖默认空白当前会话不可删，非活跃默认会话可删，命名活跃会话可删。
   - 覆盖选择动作 label/value/hint/input labels/identifier。
   - 覆盖删除动作 enabled/disabled value/hint。
   - 覆盖 identifier 为 `session-chip-<action>-<uuid8>` 且不包含会话标题。
8. 更新文档测试数：57 -> 58。
