# v2.23 / 运行策略开关宽屏网格

## 版本号和分配依据

- 版本号：`v2.23`
- 分配依据：`update_log.md` 最新正式版本为 `v2.22 / 聊天气泡宽屏宽度策略`，本轮继续递增到 `v2.23`。
- 所属总目标：Agent X 持续优化 UI、Mac 和 iPad 体验。

## 背景

设置页和优化 dashboard 的 `运行策略` 区域目前把四个 `OptimizationToggleRow` 固定渲染为单列。iPhone 和窄 split view 下这符合阅读习惯，但 iPad 与 Mac Catalyst 宽区域会浪费横向空间并拉长滚动距离。项目已有 `OptimizerMetricGridLayoutPolicy` 和 `PromptTemplateGridLayoutPolicy` 这类可测试宽屏策略，本轮应把运行策略开关也抽成共享网格策略，保持窄屏单列、宽屏双列。

## 目标

1. 新增 `OptimizationToggleGridLayoutPolicy`，集中定义运行策略开关卡片最小宽度、间距、最大列数和列数阈值。
2. 新增 `OptimizationToggleGrid`，由 `SettingsWorkspace` 和 `OptimizerDashboard` 复用。
3. 窄屏和窄 split view 保持单列；iPad/Mac 宽区域使用两列，减少设置页和优化 dashboard 的纵向滚动。
4. 保留 `OptimizationToggleRow` 的现有 label/value/hint/input labels/identifier，不改变 VoiceOver/Voice Control 语义。
5. 新增 XCTest 锁住运行策略开关网格在窄屏、iPad 和 Mac 宽窗口下的列数策略。
6. 同步 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 非目标

- 不修改 `DeviceOptimizer`、`OptimizationSwitch` 数据结构或默认开关状态。
- 不修改 `OptimizationToggleAccessibilityMetadata` 的文案和 identifier。
- 不修改 Apple Silicon 指标卡、指标网格、芯片准备度卡片或离线隐私保护摘要。
- 不修改模型、artifact、runtime、会话、提示词、导出、壁纸或 composer 状态流。
- 不新增 UI Test target。
- 不下载模型权重，不接入云端推理，不接入真实 runtime，不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `SettingsWorkspace` 在 `VStack` 面板中逐个渲染 `optimizer.switches`。
  - `OptimizerDashboard` 也重复相同的运行策略单列渲染。
  - `OptimizationToggleRow` 已接入稳定辅助语义，本轮只改变布局入口。
  - `OptimizerMetricGridLayoutPolicy` 可作为共享网格策略参考。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试基线为 66 个 XCTest 方法。
  - 已有 `testOptimizationToggleRowsExposeAccessibilityMetadata`，但没有锁住运行策略开关宽屏网格。

## 实现步骤

1. 在 `ContentView.swift` 中新增 `OptimizationToggleGridLayoutPolicy`。
2. 策略建议：
   - 最小卡片宽度约 `250`，避免长 subtitle 在双列中过度挤压。
   - 网格间距约 `10`，与 `OptimizerMetricGridLayoutPolicy` 保持一致。
   - 最大列数为 `2`。
   - `columnCount(for:)` 在达到两列阈值时返回 2，否则返回 1。
   - `columns(forColumnCount:)` 对输入列数做 clamp。
3. 新增 `OptimizationToggleGrid` 视图：
   - 包含标题 `运行策略`。
   - 使用 `ViewThatFits(in: .horizontal)` 在两列和单列网格之间自动回退。
   - `ForEach(items)` 顺序必须保持稳定，避免开关语义回退。
   - 每个 `OptimizationToggleRow` 继续独立可聚焦。
4. 替换 `SettingsWorkspace` 与 `OptimizerDashboard` 中重复的 `VStack` 单列实现。
5. 在 `LocalGemmaTests.swift` 新增 `testOptimizationToggleGridLayoutPolicyUsesTwoColumnsOnWideSettingsWidth`，测试函数数从 66 增加到 67。

## 风险和禁止项

- 风险：两列网格会改变视觉密度，必须保持 `ForEach` 数据顺序和每个开关独立辅助焦点。
- 风险：最小卡片宽度过小会挤压长 subtitle，必须用列数阈值测试锁住窄屏回退。
- 禁止下载模型权重、云端推理、绕过 verified 门禁。
- 禁止把未跑测试写成已通过。
- 禁止验收旧 run 或旧 artifact。
