# Agent B 提示词：v2.39 工作区导航触控目标

## 版本号和依据

- 版本号：v2.39。
- 版本分配依据：`update_log.md` 最新正式版本为 v2.38，本轮按顺序递增。
- 总目标 X 当前小目标：继续优化 UI、Mac 和 iPad 体验，补齐工作区顶部 tab 和大屏侧栏工作区按钮的 44pt 触控目标。

## 背景

v2.31 到 v2.38 已连续补齐会话栏、设置页图标、全局 Header、模型文件操作、会话 chip 删除和导出弹层分享/复制等入口的 44pt 触控目标。只读子 agent 在最新 `origin/main` 上发现 `ContentView` 的工作区顶部 tab 和大屏侧栏工作区按钮只依赖 vertical padding，没有显式最小触控目标策略。现有 `WorkspaceNavigationAccessibilityMetadata` 只锁 label/value/hint/input labels/identifier 和 `Command+1...4`，没有锁触控尺寸。

## 目标

1. 新增 `WorkspaceNavigationActionLayoutPolicy`，集中定义工作区导航按钮的 44pt 最小触控目标。
2. 让顶部工作区 tab 和大屏侧栏工作区按钮都显式满足 44pt 最小高度。
3. 保留现有 `WorkspaceNavigationAccessibilityMetadata`、`WorkspaceTab.shortcutKey`、工作区 command menu、`selectedTab` 状态流和 composer focus 策略。
4. 新增 XCTest 锁住工作区导航触控目标策略。
5. 同步 README、测试规范、核心流程文档、流程图、AGENTS 和 update_log，并顺手补齐 v2.37 / v2.38 云端验收记录。

## 非目标

- 不改 `WorkspaceTab` 标题、图标、快捷键或 command menu。
- 不改工作区辅助语义文案。
- 不改主布局断点、regular/compact 侧栏选择逻辑或各工作区业务状态。
- 不改模型文件、artifact 校验、runtime、真实推理、模型下载或云端服务。
- 不新增原生 macOS target、完整 UI Test target 或大体积测试数据。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `tabPicker` 负责单栏顶部工作区 tab。
  - `sidebarTabPicker(isDetailed:)` 负责 compact / regular 大屏侧栏工作区按钮。
  - `WorkspaceNavigationAccessibilityMetadata` 已控制辅助语义。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试函数数为 82。
  - 已有工作区快捷键、command menu、选择语义和辅助语义测试，但没有工作区导航按钮触控目标测试。

## 实现步骤

1. 在开始前执行或确认：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short --branch`
2. 在 `WorkspaceNavigationAccessibilityMetadata` 附近新增 `WorkspaceNavigationActionLayoutPolicy`：
   - `enum Placement: CaseIterable { case compactTab, sidebarTab }`
   - `minimumTouchTarget = 44`
   - `compactTabMinHeight = minimumTouchTarget`
   - `sidebarTabMinHeight = minimumTouchTarget`
   - `minimumHeight(for:)`
   - `usesMinimumTouchTarget(for:)`
3. 在 `tabPicker` 的 `Label` 上增加 `.frame(minHeight: WorkspaceNavigationActionLayoutPolicy.compactTabMinHeight)`。
4. 在 `sidebarTabPicker` 的 `HStack` label 上增加 `.frame(minHeight: WorkspaceNavigationActionLayoutPolicy.sidebarTabMinHeight, alignment: .leading)`，保留现有 `.frame(maxWidth: .infinity, alignment: .leading)` 行为。
5. 新增 `testWorkspaceNavigationActionLayoutPolicyMaintainsTouchTargets`：
   - 最小触控目标为 44。
   - compact / sidebar 最小高度均大于等于 44。
   - `Placement.allCases` 覆盖两个入口。
   - 所有 placement 都满足最小触控目标。
6. 更新文档，把测试函数数从 82 改为 83，并加入 `WorkspaceNavigationActionLayoutPolicy` 说明。
7. 修正 `update_log.md`：
   - 将 v2.35 云端验收移到独立“云端验收”段并去掉重复“遗留事项”标题。
   - 为 v2.37 和 v2.38 写入已核对的 run/artifact 记录。
