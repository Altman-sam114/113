# v2.7（芯片准备度辅助语义与隐私状态）

## 版本号和分配依据

- 版本号：v2.7。
- 分配依据：`update_log.md` 最新已完成版本为 v2.6，遗留事项明确尚未处理“芯片准备度动态摘要”；本轮继续 Agent X 总目标中的 Mac/iPad/UI 可访问性优化，版本递增为 v2.7。

## 背景

当前 `DeviceOptimizer` 预置四个运行策略开关，其中 `Offline privacy guard` 默认开启，但用户可以关闭。`ChipReadinessCard` 的可见摘要仍硬编码“离线隐私保护开启”，在关闭该开关后会与真实状态不一致。`ReadinessRing` 仍使用英文辅助标签 `Deployment readiness ... percent`，与项目已建立的中文辅助语义体系不一致，也没有说明本地模拟准备度、不下载权重、不启动真实 runtime 和不发送云端服务边界。

## 本轮目标

1. 让设置页和优化 dashboard 的芯片准备度卡片根据 `Offline privacy guard` 实际开关状态展示“开启/关闭”摘要。
2. 为芯片准备度卡片和准备度圆环建立可测试的中文辅助语义 metadata，覆盖 label、value、hint、Voice Control 输入标签和稳定 identifier。
3. 保持 `ReadinessRing` 的可见视觉结构不变，仅替换英文 VoiceOver 文案。
4. 新增 XCTest 锁住默认隐私保护开启、toggle 后关闭、百分比摘要、热状态、模拟 Metal 预热、本地边界和云端边界。
5. 同步 `AGENTS.md`、`README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 非目标

- 不改变 `DeviceOptimizer.deploymentReadiness` 公式。
- 不新增真实 Core ML / Metal / ANE 推理实现。
- 不下载 Gemma 或其他模型权重。
- 不接入云端推理或外部网络模型服务。
- 不改变 artifact `verified` 门禁。
- 不做模型胶囊、模型详情右栏、完整 UI Test target 或全局视觉重设。

## 当前架构依据

- `md/flow/flow.md` 说明 `DeviceOptimizer` 是 Apple Silicon 优化指标和开关的 source of truth。
- `LocalGemma/AppState.swift` 中 `DeviceOptimizer.switches` 包含 `Offline privacy guard`，`toggle(_:)` 会翻转开关并重新计算 readiness。
- `LocalGemma/ContentView.swift` 中 `SettingsWorkspace` 和 `OptimizerDashboard` 都渲染 `ChipReadinessCard`。
- `ReadinessRing` 是准备度百分比圆环，目前只接收 `progress`。
- `OptimizationToggleAccessibilityMetadata` 已为运行策略开关提供中文辅助语义，本轮应沿用同类纯 metadata enum + XCTest 的模式。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 新增或扩展 metadata/helper：
   - 提供 readiness 百分比格式化，建议对 `progress` 做 `0...1` clamp。
   - 提供动态摘要：`热状态 \(thermalState) · 模拟 Metal 预热 · 离线隐私保护开启/关闭`。
   - 提供 card/ring label、value、hint、input labels 和 identifier。
   - hint 必须说明这是本地芯片准备度和运行策略摘要，不会下载模型权重，不会启动真实 runtime，也不会发送到云端服务。
2. 为 `DeviceOptimizer` 增加最小状态读取能力：
   - 可使用计算属性读取 `Offline privacy guard` 是否开启，或在 view 层用现有 `switches` 查找。
   - 不改变开关标题、默认顺序或 readiness 公式。
3. 修改 `SettingsWorkspace` 和 `OptimizerDashboard`：
   - 调用 `ChipReadinessCard(progress:thermalState:privacyGuardEnabled:)` 或等价接口。
   - 确保 toggle 后摘要随 `optimizer.switches` 更新。
4. 修改 `ReadinessRing`：
   - 接入中文辅助 label/value/hint/identifier。
   - 保持可见 `READY` 字样和布局不变，避免扩大视觉范围。
5. 修改 `ChipReadinessCard`：
   - 使用动态摘要替代硬编码“离线隐私保护开启”。
   - 对整卡补充辅助语义和 Voice Control 输入标签。
6. 修改 `LocalGemmaTests/LocalGemmaTests.swift`：
   - 新增 `testChipReadinessCardDescribesPrivacyGuardAndAccessibilityMetadata` 或类似测试。
   - 覆盖默认 `Offline privacy guard` 开启、toggle 后关闭、summary/value/hint/inputLabels/identifier。
   - 当前测试函数数应从 51 增加到 52。

## 云端 CI 触发

Agent B 必须基于最新 `origin/main` 在 `main` 上实现，提交信息格式为：

```text

```

```sh
git push origin main
```

触发 `.github/workflows/ci-results.yml`。

## CI 结果包要求

结果包必须是最新 `origin/main` commit 对应的未加密 artifact，至少包含：

- `ci-artifact-manifest.json`
- `artifact-name.txt`
- `ci-failure-summary.md`
- `junit.xml`
- iOS build/test 日志
- Mac Catalyst build 日志
- Mac Catalyst baseline notes
- `.xcresult` 或等价项目结果

manifest 必须包含并对齐：`branch=main`、`commitSha`、`runId`、`runAttempt`、`artifactName`、各阶段 outcome。

## Agent C 验收标准

Agent C 必须：

1. 确认本地 `main`、`origin/main`、Agent B commit SHA 和 GitHub Actions run 对齐。
2. 使用 `gh run download` 下载最新 run 的结果包，不得验收旧 run 或旧 artifact。
3. 核对 manifest、`artifact-name.txt`、JUnit、主日志、Mac Catalyst notes 和 `.xcresult`。
4. 确认新增测试覆盖芯片准备度动态摘要和中文辅助语义。
5. 确认本轮没有下载模型权重、没有接入云端推理、没有绕过 artifact `verified` 门禁。
6. 如不通过，退回 Agent B 在 `main` 上追加修复 commit；如通过，向 Agent X 报告版本号、commit、run id、attempt、artifact 名称和核对文件。

- 禁止把关闭后的 `Offline privacy guard` 继续描述为开启。
- 禁止把 `ReadinessRing` 保持为英文 VoiceOver 标签。
- 禁止改变 `DeviceOptimizer` readiness 公式导致既有测试漂移。
- 禁止下载模型权重、历史 artifact 或大体积测试数据。
- 禁止云端推理、外部网络模型服务和绕过 verified 门禁。
- 禁止伪造本地测试或云端 CI 结果。
- 禁止无关重构或拆分大文件。
