# v2.21 / 提示词模板宽屏布局策略

## 版本号和分配依据

- 版本号：`v2.21`
- 分配依据：`update_log.md` 最新正式版本为 `v2.20 / 优化指标网格宽度策略`，本轮继续递增到 `v2.21`。
- 所属总目标：Agent X 持续优化 UI、Mac 和 iPad 体验。

## 背景

提示词页的模板网格当前使用 `LazyVGrid(columns: [GridItem(.adaptive(minimum: 230), spacing: 12)])`，但 `PromptTemplateCard` 自身固定 `.frame(width: 230)`。这让 iPad 和 Mac Catalyst 宽窗口中的每一列无法充分利用可用宽度，长标题、副标题和提示词预览主要依靠缩放或截断。需要将提示词模板网格宽度策略集中管理，保留窄屏稳定单列，同时让 iPad/Mac 宽区域的卡片可以跟随列宽伸展。

## 目标

1. 新增 `PromptTemplateGridLayoutPolicy`，集中定义提示词模板卡片最小宽度、最大宽度、间距和列数计算。
2. 新增或调整共享网格视图，让 `PromptTemplatesWorkspace` 使用策略化 `LazyVGrid`。
3. 去掉 `PromptTemplateCard` 的固定 `width: 230`，改为可在网格列内横向伸展，同时用策略限制过宽文本行。
4. 窄屏保持单列和现有卡片内容层级；iPad/Mac 宽区域支持多列并提高卡片宽度利用率。
5. 新增 XCTest 锁住提示词模板网格在窄屏、iPad 宽屏和 Mac 宽窗口下的列数与卡片宽度策略。
6. 同步 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 非目标

- 不修改 `PromptTemplateLibrary` 的模板内容、分类、标题、subtitle 或 prompt 文案。
- 不修改 `PromptTemplateActionAccessibilityMetadata`、`PromptCategoryAccessibilityMetadata` 或既有辅助语义文案。
- 不修改 `InferenceEngine.applyTemplate`、`InferenceEngine.useTemplate`、composer 聚焦策略或发送逻辑。
- 不修改模型、artifact、runtime、会话、导出、壁纸或 Apple Silicon 优化状态。
- 不新增 UI Test target。
- 不下载模型权重，不接入云端推理，不接入真实 runtime，不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `PromptTemplatesWorkspace` 读取 `PromptTemplateLibrary.templates(in:)` 并用 `LazyVGrid` 渲染 `PromptTemplateCard`。
  - `PromptTemplateCard` 使用固定 `.frame(width: 230)`，这是本轮要解除的宽屏瓶颈。
  - `PromptCategorySelector` 与模板动作已有辅助语义，本轮不改。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试基线为 64 个 XCTest 方法。
  - 已有提示词模板库、分类筛选辅助语义和模板动作辅助语义测试，但没有锁住提示词模板网格宽屏策略。

## 实现步骤

1. 在 `ContentView.swift` 中新增 `PromptTemplateGridLayoutPolicy`。
2. 策略建议：
   - 最小卡片宽度保留约 `230`，用于 iPhone 和窄 split view。
   - 最大卡片宽度约 `320`，避免 Mac 宽窗口中文本行过长。
   - 网格间距沿用 `12`。
   - `columnCount(for:)` 根据可用宽度计算并 clamp 到至少 1 列。
   - `cardWidth(for:)` 根据可用宽度、列数和最大宽度计算卡片宽度。
   - `columns(for:)` 返回 `GridItem(.flexible(minimum: minimumCardWidth, maximum: maximumCardWidth), spacing: spacing)` 的数组。
3. 新增 `PromptTemplateGrid` 或等价共享视图，用 `ViewThatFits`、`GeometryReader` 或等价 SwiftUI 写法应用策略列。
4. 将 `PromptTemplatesWorkspace` 的固定 `LazyVGrid(.adaptive(minimum: 230))` 替换为共享网格。
5. 将 `PromptTemplateCard` 固定宽度改为：
   - `.frame(maxWidth: .infinity, alignment: .topLeading)`
   - `.frame(minHeight: 168, alignment: .topLeading)`
   - 如共享网格需要，可由外层限制 `maxWidth`，避免卡片无限拉伸。
6. 在 `LocalGemmaTests.swift` 新增 `testPromptTemplateGridLayoutPolicyExpandsCardsOnWidePromptWorkspace`，测试函数数从 64 增加到 65。

## 风险和禁止项

- 风险：过多列会让卡片变窄或文本拥挤，必须用策略测试锁住最小宽度和最大宽度。
- 风险：去掉固定宽度后需要确认卡片仍在网格列内稳定伸展，不破坏提示词分类筛选和模板动作按钮。
- 禁止下载模型权重、云端推理、绕过 verified 门禁。
- 禁止把未跑测试写成已通过。
- 禁止验收旧 run 或旧 artifact。
