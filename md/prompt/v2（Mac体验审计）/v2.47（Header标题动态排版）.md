# v2.47 Header 标题动态排版

## 版本号与分配依据

- 版本号：v2.47。
- 分配依据：`update_log.md` 最新正式轮次为 v2.46，本轮在 Agent X “持续优化 UI、Mac、iPad 版体验”总目标下继续递增。

## 背景

顶部 `HeaderView` 同时用于 iPhone 单栏顶部和 iPad / Mac Catalyst 双栏侧栏。Header 图标动作已经由 `HeaderActionLayoutPolicy` 锁住 44pt 触控目标，但标题文本仍使用固定 `.system(size:)`，主标题限制单行并通过 `minimumScaleFactor` 压缩。iPad split view、Mac Catalyst 窄窗口和较大 Dynamic Type 下，`端侧大模型工作台` 存在压缩、截断和可读性下降风险。

## 目标

- 新增 `HeaderTitleTextLayoutPolicy`，集中定义顶部 Header eyebrow 和主标题的 Dynamic Type 文本策略。
- `LOCAL GEMMA` eyebrow 改用语义字体，保留 tracking 和单行策略。
- `端侧大模型工作台` 主标题改用语义字体并允许两行，移除单行压缩。
- 新增 XCTest 锁住策略常量和多行能力。
- 同步 README、测试规范、核心流程、流程图、入口规则、更新日志和本提示词归档。

## 非目标

- 不改变 `HeaderActionLayoutPolicy`、Header 图标按钮尺寸或辅助语义。
- 不改变主题切换、模型工作区入口、工作区切换、模型胶囊、模型文件、runtime 状态或 verified 门禁。
- 不修改模型页、提示词页、设置页的业务状态流。
- 不下载模型权重，不接入云端推理，不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `HeaderView` 在单栏顶部和双栏侧栏复用。
  - `HeaderActionLayoutPolicy` 只负责两个 Header 图标动作的 44pt 触控目标。
  - `SectionHeaderTextLayoutPolicy` 已为工作区内部标题提供 Dynamic Type 模式，本轮采用相同小策略风格。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 已有 `testHeaderActionLayoutPolicyMaintainsTouchTargets`。
  - 本轮新增 Header 标题文本策略测试，测试函数数从 89 增至 90。

## 实现步骤

1. 在 `ContentView.swift` 增加 `HeaderTitleTextLayoutPolicy`。
2. 策略包含 `verticalSpacing`、`eyebrowTracking`、`eyebrowLineLimit`、`titleLineLimit` 和 `allowsMultilineTitle`。
3. `HeaderView` 使用策略常量替换裸写 spacing、tracking、lineLimit。
4. eyebrow 使用 `.caption.weight(.bold)`；主标题使用 `.title2.weight(.heavy)`，允许两行并保持 vertical fixed size。
5. 移除主标题 `minimumScaleFactor`。
6. 在 `LocalGemmaTests.swift` 新增 `testHeaderTitleTextLayoutPolicySupportsDynamicTypeHeadings`。
7. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md`。
