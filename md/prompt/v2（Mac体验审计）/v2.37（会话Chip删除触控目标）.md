# Agent B 提示词：v2.37 会话 Chip 删除触控目标

## 版本号和依据

- 版本号：v2.37。
- 版本分配依据：`update_log.md` 最新正式版本为 v2.36，本轮按顺序递增。
- 总目标 X 当前小目标：继续优化 UI、Mac 和 iPad 体验，修正推理页会话 chip 删除图标按钮缺少 44pt 触控目标策略的问题。

## 背景

v2.31、v2.32、v2.35 和 v2.36 已分别为会话栏、设置页图标动作、全局 Header 图标动作和模型文件操作建立 44pt 触控目标策略。只读子 agent 指出 `SessionChip` 的删除按钮只有小号 `trash.fill` 图标，没有显式命中尺寸。该按钮在 iPad/Mac 大屏会话栏和 iPhone 横向会话栏中都可见，删除本地会话是高风险动作，应保证稳定命中区并保留现有删除禁用语义。

## 目标

1. 新增 `SessionChipActionLayoutPolicy`，集中定义会话 chip 删除动作的 44pt 触控目标。
2. 将 `SessionChip` 删除按钮显式保持 44x44 命中尺寸。
3. 保留现有会话选择、删除、本地会话边界、默认空白当前会话不可删除原因、辅助语义和 Voice Control input labels。
4. 新增 XCTest 锁住会话 chip 删除动作触控目标策略。
5. 同步 README、测试规范、核心流程文档、流程图、AGENTS 和 update_log。

## 非目标

- 不改 `SessionChipActionAccessibilityMetadata` 文案。
- 不改会话创建、选择、删除、导出、composer focus 或 command menu 状态流。
- 不改 `SessionBarActionLayoutPolicy`、会话栏新建/导出按钮或会话侧栏宽度策略。
- 不下载模型权重，不接入真实 runtime，不引入云端推理。
- 不新增原生 macOS target、完整 UI Test target 或大体积测试数据。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `SessionChip` 负责单个会话 chip 的选择和删除按钮。
  - `SessionChipActionAccessibilityMetadata` 已覆盖 select/delete 辅助语义、删除禁用原因和稳定 identifier。
  - `SessionBarActionLayoutPolicy` 是本轮布局策略命名和测试模式参考。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试函数数为 80。
  - 已有 `testSessionChipActionsExposeAccessibilityMetadata`，但没有删除按钮触控目标测试。

## 实现步骤

1. 在开始实现前执行：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short --branch`
2. 在 `SessionChipActionAccessibilityMetadata` 或 `SessionChip` 附近新增 `SessionChipActionLayoutPolicy`：
   - `minimumTouchTarget = 44`
   - `deleteButtonSize = minimumTouchTarget`
   - `usesMinimumTouchTarget(for:)`，只接受 `.delete` 时返回尺寸是否达标。
3. 在 `SessionChip` 删除按钮 label 上添加 44x44 frame，必要时使用 `.contentShape(Circle())` 保证命中区稳定；不要改变 select 按钮布局。
4. 新增 `testSessionChipActionLayoutPolicyMaintainsTouchTargets`：
   - 最小触控目标为 44。
   - delete button size 大于等于最小触控目标。
   - `.delete` 满足最小触控目标。
   - `.select` 不由该策略接管，避免误改选择按钮布局。
5. 更新文档，把测试函数数从 80 改为 81，并加入 `SessionChipActionLayoutPolicy` 说明。
