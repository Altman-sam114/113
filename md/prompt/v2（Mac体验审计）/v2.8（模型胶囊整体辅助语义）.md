# v2.8（模型胶囊整体辅助语义）

## 版本号和分配依据

- 版本号：v2.8。
- 分配依据：`update_log.md` 最新完成版本为 v2.7，遗留事项仍包含“模型胶囊辅助语义”；本轮继续 Agent X 总目标中的 Mac/iPad/UI 可访问性优化，版本递增为 v2.8。

## 背景

顶部 `ModelCapsule` 集中展示当前模型、安装状态、SIM/REAL 标记、artifact 状态、后端、速度、内存和准备度。当前这些内容由多个文本、badge、metric chip 和 `ReadinessRing` 组合而成，但缺少整卡的统一辅助语义。VoiceOver 用户需要逐个扫 UI 元素才能拼出当前运行状态，Voice Control 也没有稳定输入标签或 identifier 可用于定位顶部模型状态。

## 本轮目标

1. 为 `ModelCapsule` 建立可测试的整体辅助语义 metadata，覆盖 label、value、hint、Voice Control 输入标签和稳定 identifier。
2. metadata value 必须合并当前模型名、参数量、量化、SIM/REAL 标记、artifact availability、后端、生成状态、速度、内存和准备度百分比。
3. hint 必须说明模型胶囊只展示本地状态摘要；不会下载模型权重，不会启动真实 runtime，不会发送到云端服务，也不会绕过 artifact `verified` 门禁。
4. `ModelCapsule` 接入 `.accessibilityElement(children: .combine)` 和统一 metadata，不改变视觉结构、状态流或 `ReadinessRing` 可见表现。
5. 新增 XCTest 锁住 missing/staged/verified、simulated/real、generating/idle、速度/内存/准备度和本地边界文案。
6. 同步 `AGENTS.md`、`README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 非目标

- 不改变 `ModelCatalog`、artifact validation、deployment 或 runtime plan 行为。
- 不改变 `DeviceOptimizer.deploymentReadiness` 公式。
- 不下载 Gemma 或其他模型权重。
- 不接入云端推理或外部网络模型服务。
- 不绕过 artifact `verified` 门禁。
- 不做模型页详情右栏、完整 UI Test target、真实 runtime 接入或全局视觉重设。

## 当前架构依据

- `md/flow/flow.md` 当前核心数据流第 6 步说明顶部模型胶囊展示当前模型、速度、内存、后端、artifact 状态和模拟/真实标记。
- `LocalGemma/ContentView.swift` 中 `HeaderView` 调用 `ModelCapsule`，传入 `model`、`readiness`、`tokensPerSecond`、`memoryUsageMB`、`backend`、`availability`、`isGenerating` 和 `isSimulated`。
- `ModelCapsule` 内部已经有 `statusText`、`compactMemoryValue`、`availabilityMetricValue`、`statusTint` 和 `availabilityIcon`，本轮只抽出统一 metadata，不改状态来源。
- `ChipReadinessAccessibilityMetadata` 已提供准备度百分比 clamp/format，可复用来避免百分比文案分叉。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 新增 `ModelCapsuleAccessibilityMetadata` 或等价 helper：
   - `label(model:)` 例如“当前模型 Gemma 1.5B Local”。
   - `value(...)` 合并模型名、参数量、量化、SIM/REAL、artifact availability、本地状态、后端、生成状态、速度、内存和准备度。
   - `hint` 明确只展示本地状态摘要，不下载权重、不启动真实 runtime、不发送云端服务、不绕过 verified 门禁。

   - `md/flow/flow.md` 当前 UI 工作区、关键对象、用户入口和铁律加入 `ModelCapsuleAccessibilityMetadata`。
   - `md/flow/flowchart.md` 的 UI 流程节点/说明同步顶部模型胶囊辅助语义。

## 云端 CI 触发


## CI 结果包要求

结果包必须是最新 `origin/main` commit 对应的未加密 artifact，至少包含：

- `ci-artifact-manifest.json`
- `artifact-name.txt`
- `ci-failure-summary.md`
- `junit.xml`
- iOS build/test 日志
- Mac Catalyst build 日志
- Mac Catalyst baseline notes
- `.xcresult` 或等价项目结果

manifest 必须包含并对齐：`branch=main`、`commitSha`、`runId`、`runAttempt`、`artifactName`、各阶段 outcome。

## Agent C 验收标准

Agent C 必须：

1. 确认本地 `main`、`origin/main`、Agent B commit SHA 和 GitHub Actions run 对齐。
2. 使用 `gh run download` 下载最新 run 的结果包，不得验收旧 run 或旧 artifact。
3. 核对 manifest、`artifact-name.txt`、JUnit、主日志、Mac Catalyst notes 和 `.xcresult`。
4. 确认新增测试覆盖模型胶囊整体辅助语义、artifact 状态、SIM/REAL、速度/内存/准备度和本地边界。
5. 确认本轮没有下载模型权重、没有接入云端推理、没有绕过 artifact `verified` 门禁。
6. 如不通过，退回 Agent B 在 `main` 上追加修复 commit；如通过，向 Agent X 报告版本号、commit、run id、attempt、artifact 名称和核对文件。

## 风险和禁止项

- 禁止为了辅助语义改变模型选择、deployment、runtime plan 或 artifact validation 行为。
- 禁止把 `RealGemmaRuntimePlaceholder` 描述成真实推理已完成。
- 禁止将 `missing` / `staged` artifact 描述为真实权重可运行。
- 禁止下载模型权重、历史 artifact 或大体积测试数据。
- 禁止云端推理、外部网络模型服务和绕过 verified 门禁。
- 禁止伪造本地测试或云端 CI 结果。
- 禁止无关重构或拆分大文件。
