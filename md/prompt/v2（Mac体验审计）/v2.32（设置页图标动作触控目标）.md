# v2.32（设置页图标动作触控目标）

## 版本号与分配依据

- 版本号：`v2.32`
- 分配依据：`update_log.md` 最新正式版本为 `v2.31 / 会话栏操作触控目标`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版”下继续递增。

## 背景

v2.25、v2.28 和 v2.31 已分别锁住提示词模板动作、提示词分类筛选和会话栏操作的 44pt 触控目标。上一轮只读子 agent 继续审计设置页后指出，`ThemePreferencePanel` 的设置页主题切换按钮仍为 `42x42`，`WallpaperPreferencePanel` 的相册选择和恢复系统背景按钮仍为 `40x40`，低于 Apple 44pt 触控目标。它们是设置页高频入口，出现在 iPhone、iPad split view 和 Mac Catalyst 设置工作区中，适合本轮小步优化。

## 目标

1. 新增设置页图标动作布局策略，集中定义设置页图标动作最小触控目标。
2. 将设置页“切换外观主题”“选择相册壁纸”“恢复系统背景”三个图标按钮提升到至少 `44x44`。
3. 保留现有主题切换、相册读取、本地压缩、恢复系统背景、`AppStorage` 写入和错误弹层逻辑。
4. 保留 `HeaderActionAccessibilityMetadata` 和 `WallpaperPreferenceAccessibilityMetadata` 的 label/value/hint/input labels/identifier。
5. 新增 XCTest 锁住设置页图标动作触控目标，并确认主题切换、相册壁纸选择和恢复系统背景三个图标动作覆盖关系。
6. 同步 README、测试规范、核心流程文档、流程图、入口规则和更新日志。

## 非目标

- 不改变全局顶部 header 的主题/模型库按钮尺寸；本轮只处理设置页偏好面板。
- 不改变主题枚举、主题存储、相册导入、壁纸压缩、恢复系统背景或错误处理。
- 不改变设置页 Apple Silicon 指标网格、运行策略开关网格或芯片准备度语义。
- 不改变任何辅助语义文案、Voice Control 输入标签或稳定 identifier。
- 不新增真实模型 runtime，不下载模型权重，不接入云端推理。
- 不创建原生 macOS target，不新增 UI Test target。

## 当前架构依据

- `md/flow/flow.md` 中设置页由 `SettingsWorkspace`、`ThemePreferencePanel`、`WallpaperPreferencePanel`、优化指标和运行策略开关组成。
- `HeaderActionAccessibilityMetadata` 负责设置页外观主题按钮的辅助语义。
- `WallpaperPreferenceAccessibilityMetadata` 负责选择相册壁纸和恢复系统背景动作的辅助语义。
- 当前测试只覆盖这些动作的辅助语义，没有锁住设置页图标按钮触控目标。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的设置页偏好面板附近新增 `SettingsIconActionLayoutPolicy`。
2. 为策略定义 `minimumTouchTarget = 44`、`iconButtonSize` 和覆盖主题/壁纸三个入口的 `Action`。
3. 将 `ThemePreferencePanel` 中主题切换按钮从 `.frame(width: 42, height: 42)` 改为使用策略常量。
4. 将 `WallpaperPreferencePanel` 中 `PhotosPicker` 标签和恢复系统背景按钮从 `.frame(width: 40, height: 40)` 改为使用策略常量。
5. 不改 action、图标、颜色、背景、disabled 状态、`PhotosPicker` selection、错误 alert、辅助语义或 identifier。
6. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增 `testSettingsIconActionLayoutPolicyMaintainsTouchTargets`，锁住常量、图标按钮尺寸和三个设置页图标动作覆盖关系。
7. 同步文档和日志，将测试函数数从 75 更新到 76。
