# v2.28 提示词分类筛选换行布局 - Agent B 实现提示词

## 版本号与分配依据

- 版本号：v2.28。
- 分配依据：`update_log.md` 当前最新正式版本为 v2.27，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增。

## 背景

提示词页已经有 `PromptTemplateGridLayoutPolicy` 控制模板卡片宽屏布局，也有 `PromptTemplateActionLayoutPolicy` 保证模板动作达到 44pt 触控目标。但 `PromptCategorySelector` 目前仍使用横向 `ScrollView` + `HStack` 展示“全部 / 部署 / 隐私 / 性能 / 写作 / 产品 / 排障”筛选 chip。这个实现能避免溢出，但在 iPad split view、Mac Catalyst 窄窗口和较大文字环境下会把筛选项藏进横向滚动区域，降低可发现性，也没有显式锁住筛选 chip 的 44pt 触控高度。

## 目标

- 新增 `PromptCategoryLayoutPolicy` 或等价布局策略，集中定义筛选 chip 的水平间距、行间距、最小触控目标、水平/垂直 padding 和最小 chip 宽度。
- 将 `PromptCategorySelector` 从横向滚动 `HStack` 改为可换行布局；窄屏和窄 split view 自动换行，iPad/Mac 宽区域尽量单行展示。
- 筛选 chip 显式达到至少 44pt 触控高度。
- 保留 `PromptCategoryAccessibilityMetadata` 的 label/value/hint/input labels/identifier 和 `.isSelected` 选择语义。
- 保留 `selectedCategory` 状态流、筛选结果、模板填入/发送行为和 `PromptTemplateGridLayoutPolicy` 卡片列数策略。
- 新增 XCTest 锁住筛选 chip 布局策略常量、最小触控目标、最小 chip 宽度、单行最小宽度、窄宽度换行判断和无效宽度 clamp。
- 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md`。

## 非目标

- 不改变提示词模板内容、分类枚举、筛选业务结果或模板排序。
- 不改变 `PromptCategoryAccessibilityMetadata` 文案、Voice Control 输入标签或 identifier。
- 不改变模板卡片网格列数、卡片宽度策略或模板动作辅助语义。
- 不新增真实模型 runtime，不下载 Gemma 权重，不接入外部推理服务。
- 不创建完整 UI Test target，不做无关 Swift 文件拆分。

## 当前架构依据

- `md/flow/flow.md` 规定提示词页由 `PromptTemplatesWorkspace` 提供模板筛选、填入和发送入口。
- `LocalGemma/ContentView.swift` 当前：
  - `PromptCategoryAccessibilityMetadata` 负责提示词分类筛选辅助语义。
  - `PromptTemplatesWorkspace` 在 `SectionHeader` 后渲染 `PromptCategorySelector`。
  - `PromptCategorySelector` 使用横向 `ScrollView` + `HStack(spacing: 8)`。
  - `categoryButton` 使用 `Label`、`Button`、辅助语义和 `.isSelected` trait。
  - `PromptTemplateGridLayoutPolicy` 控制模板卡片网格，不应被本轮改动破坏。
  - `PromptTemplateActionLayoutPolicy` 控制模板动作触控目标，不应被本轮改动破坏。
- `LocalGemmaTests/LocalGemmaTests.swift` 当前：
  - `testPromptCategoryAccessibilityMetadataDescribesFilterSelectionAndInputLabels` 覆盖筛选辅助语义。
  - `testPromptTemplateGridLayoutPolicyExpandsCardsOnWidePromptWorkspace` 覆盖模板网格宽度策略。
  - `testPromptTemplateActionLayoutPolicyMaintains44PointTouchTargets` 覆盖模板动作 44pt 触控目标。

## 实现步骤

1. 在 `PromptCategoryAccessibilityMetadata` 附近或 `PromptCategorySelector` 附近新增 `PromptCategoryLayoutPolicy`。
2. 建议常量：
   - `minimumTouchTarget: CGFloat = 44`
   - `horizontalSpacing: CGFloat = 8`
   - `verticalSpacing: CGFloat = 8`
   - `horizontalPadding: CGFloat = 12`
   - `verticalPadding: CGFloat = 9`
   - `minimumChipWidth: CGFloat = 74`
3. 新增纯策略函数，例如：
   - `minimumSingleRowWidth(forCategoryCount:)`
   - `usesWrapping(availableWidth:categoryCount:)`
   - `clampedAvailableWidth(_:)`
   - 无效、负数或非有限宽度安全 clamp，不产生负宽度。
4. 新增一个小型 `Layout` 或等价 SwiftUI 布局，例如 `PromptCategoryFlowLayout`，复用 `PromptCategoryLayoutPolicy.horizontalSpacing` 和 `verticalSpacing`，根据可用宽度把子视图换行。
5. 将 `PromptCategorySelector` 改为使用可换行布局，不再把所有 chip 放进横向滚动 `ScrollView`。
6. 在 `categoryButton` 中：
   - 保留 `Button` + `Label`，不要改成 `onTapGesture`。
   - 保留现有 accessibility label/value/hint/input labels/identifier 和 `.isSelected` trait。
   - 增加 `.frame(minWidth: PromptCategoryLayoutPolicy.minimumChipWidth, minHeight: PromptCategoryLayoutPolicy.minimumTouchTarget)` 或等价约束。
   - 使用策略常量控制 padding 和 spacing。
7. 不修改 `PromptTemplateGrid`、`PromptTemplateCard`、模板动作状态流或 composer 聚焦逻辑。
