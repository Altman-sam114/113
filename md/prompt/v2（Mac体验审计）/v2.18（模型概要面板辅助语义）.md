# v2.18 / 模型概要面板辅助语义

## 版本号和分配依据

- 版本号：`v2.18`
- 分配依据：`update_log.md` 最新正式版本为 `v2.17 / 模型详情行级辅助语义`，本轮继续递增到 `v2.18`。
- 所属总目标：Agent X 持续优化 UI、Mac 和 iPad 体验。

## 背景

模型页已经具备模型详情整体摘要、详情行级语义、模型选择器、状态徽章和部署控件辅助语义。Mac/iPad 宽屏模型页中，`ModelSummaryPanel` 是模型详情列的第一个视觉锚点，展示模型名称、简介、能力标签和 artifact validation summary，但目前没有独立的面板级 VoiceOver / Voice Control 入口。用户可以听到详情列整体摘要和后续行级信息，但不能单独快速扫描“这个模型概要卡片包含哪些能力标签和当前 artifact 摘要”。

## 目标

1. 为 `ModelSummaryPanel` 建立独立的模型概要面板辅助语义。
2. 新增 `ModelSummaryAccessibilityMetadata`，暴露 label、value、hint、Voice Control 输入标签和稳定 identifier。
3. value 必须合并模型名称、简介、能力标签、artifact availability、validation summary、文件格式和包体大小。
4. hint 必须明确概要面板只展示本地模型概要、能力标签和校验摘要；不下载模型权重、不启动真实 runtime、不发送云端服务、不绕过 artifact verified 门禁。
5. `ModelSummaryPanel` 接入 metadata，同时不改变视觉布局、模型状态流、artifact 校验、runtime plan 或部署动作。
6. 新增 XCTest 覆盖 missing / staged / verified validation summary、能力标签、input labels、identifier 和安全边界。
7. 同步 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 非目标

- 不修改 `ModelCatalog`、`LocalArtifactValidator`、`LocalRuntimePlanner` 或部署状态流。
- 不改变模型页布局、视觉样式、颜色、断点或详情行级语义。
- 不修改 `ModelDetailAccessibilityMetadata` 或 `ModelDetailRowAccessibilityMetadata` 的既有语义，除非为兼容概要面板做最小调整。
- 不新增 UI Test target。
- 不下载模型权重，不接入云端推理，不接入真实 runtime，不创建原生 macOS target。
- 不绕过 artifact `verified` 门禁。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `ModelDetailColumn` 汇总 `ModelSummaryPanel`、参数、性能和建议面板。
  - `ModelSummaryPanel` 展示模型名称、简介、能力标签和 `validation.summary`。
  - 已有同类模式：`ModelDetailAccessibilityMetadata`、`ModelDetailRowAccessibilityMetadata`、`ModelStatusBadgeAccessibilityMetadata`。
- `LocalGemma/AppState.swift`
  - `LocalModel` 提供 `summary`、`capabilities`、`artifactManifest.fileFormat` 和 `sizeOnDisk`。
  - `ArtifactValidationResult` 提供 `availability` 和 `summary`。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试基线为 61 个 XCTest 方法。

## 实现步骤

1. 在 `ContentView.swift` 中新增 `ModelSummaryAccessibilityMetadata`。
2. 元数据建议包含：
   - `static let hint`
   - `static let identifier`
   - `static func label(model:)`
   - `static func value(model:validation:)`
   - `static func inputLabels(model:)`
3. 将 `ModelSummaryPanel` 设置为独立只读辅助元素，接入 label/value/hint/input labels/identifier。
4. 不改变 `FlowLayout`、capability chip 视觉、`validation.summary` 文案或任何部署动作。
5. 在 `LocalGemmaTests.swift` 新增 `testModelSummaryPanelExposesAccessibilityMetadata`，测试函数数从 61 增加到 62。

## 风险和禁止项

- 风险：模型概要面板语义与模型详情整体摘要会有少量重复；本轮定位为能力标签与 validation summary 的面板级扫描入口。
- 风险：identifier 必须稳定，不能依赖动态 prompt、validation summary 或模型能力全文。
- 禁止下载模型权重、云端推理、绕过 verified 门禁。
- 禁止把未跑测试写成已通过。
- 禁止验收旧 run 或旧 artifact。
