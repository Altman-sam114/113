# v2.42 / 导出弹层宽屏内容宽度

## 版本号和分配依据

- 版本号：v2.42。
- 分配依据：`update_log.md` 最新正式版本为 v2.41，本轮继续 Agent X 总目标“优化 UI、Mac、iPad 版”的下一项独立可验证小目标。

## 背景

当前导出弹层已经通过 `ExportSessionActionAccessibilityMetadata` 锁住分享 Markdown、文本兜底和复制全文的辅助语义，并通过 `ExportSessionActionLayoutPolicy` 锁住底部分享/复制与 toolbar 分享入口的 44pt 触控目标。但 `ExportSessionView` 的会话摘要、Markdown 预览和底部动作仍随 sheet 宽度无限拉伸，在 Mac Catalyst 或 iPad 宽 sheet 中可读性下降。

## 目标

- 新增导出弹层整体宽屏内容宽度策略。
- iPhone 和窄 split view 保持原有可用宽度。
- iPad/Mac 宽 sheet 中，让会话摘要、Markdown 预览和底部分享/复制动作整体居中并限制最大内容宽度。
- 新增测试锁住策略常量、窄屏扣 padding、宽屏封顶和无效宽度 clamp。
- 同步 README、测试规范、核心流程、流程图、入口规则、更新日志和本提示词归档。

## 非目标

- 不修改 `ExportPayload`。
- 不修改 `InferenceEngine` 导出文本或 Markdown 文件生成。
- 不修改 ShareLink 文件优先 / 文本兜底逻辑。
- 不修改 `UIPasteboard` 复制行为。
- 不修改 toolbar 分享入口。
- 不修改 `ExportSessionActionAccessibilityMetadata` 的语义边界。
- 不修改 `ExportSessionActionLayoutPolicy` 的 44pt 触控目标。
- 不接入真实 runtime，不下载模型权重，不发送到云端服务，不绕过 verified 门禁。

## 当前架构依据

- `md/flow/flow.md` 规定导出弹层分享动作必须处理 Markdown 文件不存在时的文本兜底，分享/复制不得暗示云端上传。
- `LocalGemma/ContentView.swift` 中 `ExportSessionView` 负责导出 sheet UI，`ExportPayload.existingFileURL` 负责文件存在性判断。
- `LocalGemmaTests/LocalGemmaTests.swift` 已有 `testExportPayloadOnlySharesExistingFileURL`、`testExportSessionActionsExposeAccessibilityMetadata` 和 `testExportSessionActionLayoutPolicyMaintainsTouchTargets`。
- 项目已有 `SettingsWorkspaceLayoutPolicy`、`PromptTemplatesWorkspaceLayoutPolicy`、`ModelLibraryWorkspaceLayoutPolicy` 等宽屏内容宽度策略可作为实现模式。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 新增 `ExportSessionLayoutPolicy`：
   - `horizontalPadding = 18`
   - `minimumReadableWidth = 320`
   - `maximumContentWidth = 760`
   - `contentWidth(forContainerWidth:)` 对无效宽度返回最小可读宽度，对窄屏扣除左右 padding，对宽屏封顶。
2. 调整 `ExportSessionView`：
   - 在 `NavigationStack` 内用 `GeometryReader` 获取 sheet 宽度。
   - 将会话摘要、Markdown 预览、底部分享/复制动作放入同一个内容列。
   - 内容列使用 `ExportSessionLayoutPolicy.contentWidth` 固定宽度并居中。
   - 保留 `AppBackground`、toolbar、ShareLink、复制、辅助语义和 44pt 触控目标。
3. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增 `testExportSessionLayoutPolicyConstrainsWideContent`：
   - 断言常量。
   - 断言 `320 -> 284`、`390 -> 354`。
   - 断言 `834`、`1200` 等宽屏封顶到 `760`。
   - 断言 `-1` 和 `.nan` clamp 到 `320`。
4. 同步文档：
   - `AGENTS.md`
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`

- manifest 的 `branch=main`、`version=v2.42`、`commitSha`、`runId`、`runAttempt` 必须与下载 run 对齐。

- `test.log` 必须包含 `testExportSessionLayoutPolicyConstrainsWideContent`。
- 日志不得出现模型权重下载、云端推理、绕过 verified 门禁或导出内容上传。

## 风险和禁止项

- 宽度策略不能让窄屏内容小于可用宽度。
- 不要把整体宽度策略混入 `ExportSessionActionLayoutPolicy`，避免触控目标职责混乱。
- 不要修改导出文本、文件生成、分享兜底、剪贴板写入或会话状态流。
- 禁止下载模型权重、接入云端推理、伪造测试结果或跳过云端 artifact 验收。
