# v2.2 / 导出分享复制辅助语义

## 版本号与分配依据

- 版本号：`v2.2`
- 分配依据：`update_log.md` 当前最新正式版本为 `v2.1 / 会话栏操作辅助语义`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增到 `v2.2`。

## 背景

v2.1 已为推理页会话栏的新建和导出入口补齐辅助语义，并通过云端 CI artifact 验收。但用户进入 `ExportSessionView` 后，导出弹层内部的分享和复制动作仍只有可见文本或 icon-only toolbar 图标，缺少统一的 `accessibilityValue`、`accessibilityHint`、Voice Control 输入标签和稳定 identifier。

这会让 Mac Catalyst 和 iPad 外接键盘用户在“会话栏导出入口”和“导出弹层内部操作”之间得到不一致的辅助语义。本轮只补导出弹层分享/复制控件的 metadata，不改变导出文本、Markdown 文件生成、分享兜底、剪贴板写入或会话状态流。

## 目标

1. 新增可测试的 `ExportSessionActionAccessibilityMetadata` 或等价 metadata。
2. 覆盖导出弹层三类动作：
   - 分享 Markdown 文件。
   - 分享文本兜底。
   - 复制全文到剪贴板。
3. 将 metadata 应用到 `ExportSessionView` 底部 ShareLink、底部复制按钮和 toolbar 分享入口。
4. label / value / hint 必须说明：
   - Markdown 文件分享只使用本地导出的 `.md` 文件。
   - 文本分享是 Markdown 文件不存在时的本地文本兜底。
   - 复制全文写入系统剪贴板。
   - 以上动作不会把会话发送到云端服务或外部推理服务。
5. toolbar 分享入口不能只暴露模糊图标；可使用带文字的 `Label(...).labelStyle(.iconOnly)` 或等价方式保留视觉。
6. 新增纯 XCTest 锁住 label、value、hint、input labels、identifier 和本地/云端边界；测试函数数从 46 增加到 47。
7. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md` 中的当前能力、测试数量和验证记录。

## 非目标

- 不修改 `InferenceEngine.exportActiveSessionText` 或 `exportActiveSessionMarkdownFile`。
- 不修改 `ExportPayload.existingFileURL` 的文件存在性兜底逻辑。
- 不改变 `ShareLink` 分支条件、分享内容、剪贴板写入行为或导出弹层视觉布局。
- 不新增 UI Test target，不做完整导出 UI 视觉重设，不创建原生 macOS target。
- 不下载模型权重，不接入云端推理，不发送会话或提示词到外部服务。

## 当前架构依据

- `md/flow/flow.md`：分享流程要求 `ExportSessionView` 优先分享真实存在的 Markdown 文件，否则分享文本；复制全文使用系统剪贴板。
- `LocalGemma/ContentView.swift`：
  - `ExportPayload.existingFileURL` 负责判断临时 `.md` 文件是否存在。
  - `ExportSessionView` 底部有文件/文本 `ShareLink` 分支和 `复制全文` 按钮。
  - toolbar 分享入口当前使用 icon-only `Image`，需要补明确语义。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - `testExportPayloadOnlySharesExistingFileURL` 已锁住文件存在性兜底。
  - 本轮应新增导出弹层动作 metadata 测试，不重复测试文件系统兜底。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的 `ExportPayload` / `ExportSessionView` 附近新增 `ExportSessionActionAccessibilityMetadata`：
   - `Action`：`.shareMarkdownFile`、`.shareTextFallback`、`.copyFullText`。
   - `label(for:)`
   - `value(for:messageCount:)`
   - `hint(for:)`
   - `inputLabels(for:)`
   - `identifier(for:)`
2. 修改 `ExportSessionView`：
   - 文件分支底部 `ShareLink` 使用 `.shareMarkdownFile` metadata。
   - 文本分支底部 `ShareLink` 使用 `.shareTextFallback` metadata。
   - toolbar `ShareLink` 也使用同一 metadata，并将裸 `Image` 换成带文字的 `Label(...).labelStyle(.iconOnly)` 或等价实现。
   - 复制按钮使用 `.copyFullText` metadata；视觉上的“已复制”状态可以保留，但辅助 label 应稳定表达动作本身。
3. 新增 XCTest，例如 `testExportSessionActionsExposeAccessibilityMetadata`：
   - 三个 action 的 label/value/hint/input labels/identifier 均非空。
   - identifiers 稳定，例如 `export-session-action-share-markdown-file`、`export-session-action-share-text-fallback`、`export-session-action-copy-full-text`。
   - value 或 hint 包含 `本地 Markdown`、`文本分享兜底`、`剪贴板` 和 `不会发送到云端服务` 等边界。
   - input labels 包含中文动作名，例如“分享 Markdown 文件”“分享文本内容”“复制全文”。
   - 保留 `ExportPayload.existingFileURL` 测试不变，确认 metadata 不改变兜底行为。
4. 同步文档：
   - `AGENTS.md`：一句话总览和核心边界加入导出弹层分享/复制辅助语义。
   - `md/flow/flow.md`：分享流程和核心模块加入 `ExportSessionActionAccessibilityMetadata`。
   - `md/flow/flowchart.md`：会话导出与分享流注明分享/复制动作辅助语义。
   - `md/test/test.md`：测试数量从 46 更新到 47，覆盖范围和 Stage Regression 触发条件加入导出弹层分享/复制辅助语义。
   - `README.md`：当前范围、推理页导出说明、验证章节和历史记录同步 v2.2。
   - `update_log.md`：当前核心测试改为 47，追加 v2.2 记录，写明本地轻量检查命令与结果。

## 风险与禁止项

- 风险：toolbar 分享入口若仍是裸图标，会继续给 VoiceOver / Voice Control 暴露模糊动作；本轮必须补明确 label 和 input labels。
- 风险：分享 hint 不能暗示上传到云端服务；必须明确是本地 Markdown 文件或文本兜底。
- 风险：复制全文只能说明写入剪贴板，不能宣称会导出文件或发送网络请求。
- 禁止下载模型权重、云端推理、上传会话、绕过 verified 门禁、改真实 runtime、引入新第三方依赖、扩大到无关视觉重构。
- 禁止伪造本地或云端测试结果；没有跑的测试必须明确说明。
