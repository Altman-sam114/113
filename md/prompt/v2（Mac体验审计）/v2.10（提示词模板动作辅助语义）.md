# v2.10（提示词模板动作辅助语义）Agent B 实现提示词

## 版本号和分配依据

- 版本号：`v2.10`
- 分配依据：`update_log.md` 最新正式版本为 `v2.9 / 模型详情右栏辅助语义`。Agent X 继续执行“优化 UI、Mac、iPad 版”的总目标；两个只读子 agent 均指出提示词模板卡片的“填入/发送”动作仍缺稳定中文辅助语义，因此本轮递增到 `v2.10`。

## 背景

当前提示词页已经具备：

- `PromptTemplateLibrary`：内置六类预设提示词模板。
- `PromptCategoryAccessibilityMetadata`：提示词分类筛选 chip 的 label/value/hint/input labels/identifier。
- `ComposerFocusRequest` / `ComposerFocusPolicy`：模板填入或发送后切回推理页并请求 composer 聚焦。

缺口是 `PromptTemplateCard` 的两个动作按钮仍只有英文 `accessibilityLabel`，没有稳定 value、hint、Voice Control input labels 和 accessibility identifier。Mac Catalyst 和 iPad 外接键盘 / Voice Control 用户难以稳定发现“填入输入框”和“直接发送到本地模拟 runtime”的边界。

## 目标

1. 新增 `PromptTemplateActionAccessibilityMetadata`。
2. 为模板卡片动作生成稳定：
   - label
   - value
   - hint
   - Voice Control input labels
   - accessibility identifier
3. 覆盖两个动作：
   - `apply` / 填入：写入 composer、切回推理页并聚焦输入框，不发送 prompt。
   - `send` / 发送：把模板作为当前输入发送到本地模拟 runtime、切回推理页并聚焦输入框。
4. `hint` 必须明确本地边界：
   - 不下载模型权重
   - 不启动真实 runtime
   - 不发送到云端服务
   - 发送动作不绕过 verified 门禁
5. `isGenerating` 时 value 必须说明当前生成中、动作暂不可用。
6. 新增 XCTest，测试函数数从 54 增加到 55。

## 非目标

- 不修改提示词模板内容、分类、排序或筛选行为。
- 不修改 `InferenceEngine.applyTemplate` / `useTemplate` 业务逻辑。
- 不修改 composer 焦点策略、会话状态、模型状态或 artifact verified 门禁。
- 不创建原生 macOS target。
- 不下载 Gemma 或任何模型权重。
- 不接入云端推理、外部模型服务或真实 Core ML 推理。
- 不新增 UI Test target，不做无关重构。

## 当前架构依据

- `md/flow/flow.md`：提示词页只负责筛选模板、填入输入框和直接发送；发送仍走 `InferenceEngine` 和本地 runtime 边界。
- `LocalGemma/ContentView.swift`：
  - `PromptTemplateCard` 位于提示词网格内，当前两个按钮分别调用 `apply` 和 `send` 闭包。
  - `PromptCategoryAccessibilityMetadata` 可作为 metadata enum 风格参考。
  - `ComposerFocusReason.applyTemplate` / `.sendTemplate` 已保证操作后请求 composer 聚焦。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - `testPromptCategoryAccessibilityMetadataDescribesFilterSelectionAndInputLabels`
  - `testInferenceAppliesPromptTemplateWithoutSending`
  - `testInferenceUsesPromptTemplateForGeneration`
  可作为本轮测试风格参考。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 新增 `PromptTemplateActionAccessibilityMetadata`。
2. 推荐接口：
   - `enum Action: String, CaseIterable, Identifiable { case apply; case send }`
   - `static func label(for action: Action, template: PresetPromptTemplate) -> String`
   - `static func value(for action: Action, isGenerating: Bool) -> String`
   - `static func hint(for action: Action, template: PresetPromptTemplate) -> String`
   - `static func inputLabels(for action: Action, template: PresetPromptTemplate) -> [String]`
   - `static func identifier(for action: Action, template: PresetPromptTemplate) -> String`
3. 将 `PromptTemplateCard` 的“填入”和“发送”按钮接入上述 metadata。
4. 保持现有可见 UI、按钮动作、禁用态和 opacity 行为不变。
5. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增 `testPromptTemplateActionsExposeAccessibilityMetadata`：
   - 覆盖 `Action.allCases == [.apply, .send]`。
   - 覆盖每个模板两个动作的 identifier 唯一且包含模板 id。
   - 覆盖中文 label、生成中 value、本地填入/发送 hint、Voice Control input labels。
   - 覆盖 apply hint 包含“不发送 prompt”。
   - 覆盖 send hint 包含“本地模拟 runtime”“不会发送到云端服务”“verified 门禁”。
6. 同步文档：
   - `AGENTS.md`
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`

## Agent C 验收标准

Agent C 必须只验收最新 `origin/main` commit 对应的 run 和 artifact：

1. `git fetch origin`
2. 核对 `git rev-parse origin/main` 等于 v2.10 commit。
3. `gh run list --workflow ci-results.yml --branch main --limit 5` 找到最新 run。
4. `gh run download <run_id> --dir /private/tmp/localgemma-c-review-<run_id>`
5. 核对：
   - manifest `branch=main`
   - manifest `version=v2.10`
   - manifest `commitSha` 等于 `origin/main`
   - manifest `runId` / `runAttempt` 等于下载 run
   - `artifactName` 等于 `artifact-name.txt`
   - required outcomes：static、logic、iOS build、XCTest、Mac Catalyst build、Mac run script 均为 success
   - optional Codex Run environment 允许 skipped 但必须有 skipped reason
   - `junit.xml` failures 为 0
   - `xcodebuild.log`、`test.log`、`mac-catalyst-build.log` 有成功记录
   - `.xcresult` Info.plist 可通过 `plutil -lint`
