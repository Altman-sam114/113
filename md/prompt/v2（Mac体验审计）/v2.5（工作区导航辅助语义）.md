# v2.5 / 工作区导航辅助语义

## 版本号和分配依据

- 版本号：`v2.5`
- 分配依据：`update_log.md` 当前最新正式记录为 `v2.4 / 头部主题入口辅助语义`，本轮继续围绕 Mac/iPad 工作台辅助导航审计递增。

## 背景

当前工作区导航已经具备标题、快捷键、选中状态 value 和稳定 identifier：

- 顶部 tab button 使用 `WorkspaceTab.shortcutKey` 和 `SelectionAccessibilityMetadata.workspaceLabel`。
- 大屏 sidebar tab button 使用相同快捷键和 label/value。
- `WorkspaceTab.commandItems` 已覆盖系统 `工作区` command menu。

缺口是可见工作区导航按钮仍没有 `.accessibilityHint` 和 `.accessibilityInputLabels`。相比会话栏、导出、壁纸、模型部署、头部主题入口和提示词筛选等入口，Mac/iPad 的工作区导航尚未向 VoiceOver / Voice Control 说明快捷键、当前用途和“只切换本地工作区”的边界。

## 目标

1. 新增或扩展工作区导航辅助语义 metadata，例如 `WorkspaceNavigationAccessibilityMetadata`。
2. 为每个 `WorkspaceTab` 生成：
   - label：复用或等价于 `SelectionAccessibilityMetadata.workspaceLabel(for:)`。
   - value：区分已选中 / 未选中。
   - hint：包含 `Command 1...4` 快捷键、`sidebarSubtitle` 用途说明，并明确只切换本地工作区。
   - input labels：包含“推理工作区 / 模型工作区 / 提示词工作区 / 设置工作区”等中文 Voice Control 命令。
   - compact tab identifier 和 sidebar tab identifier。
3. 将 metadata 应用到：
   - `ContentView.tabPicker` 顶部工作区按钮。
   - `ContentView.sidebarTabPicker(isDetailed:)` 大屏侧栏工作区按钮。
4. 新增 XCTest 锁住四个工作区的 hint、input labels、identifier、快捷键和 `sidebarSubtitle` 对齐，测试函数数从 49 增加到 50。
5. 同步 `AGENTS.md`、`README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 非目标

- 不改 `WorkspaceTab` case、标题、图标、快捷键或 command menu 行为。
- 不改 `selectedTab`、focused scene binding、composer focus policy、会话状态、模型状态或 runtime。
- 不改布局断点、sidebar 宽度、视觉设计或 tab 排序。
- 不接入真实模型推理，不下载模型权重，不创建原生 macOS target。
- 不在本轮处理运行策略开关、优化面板、UI Test target 或真实渲染截图。

## 当前架构依据

- `WorkspaceTab.shortcutKey` 定义 `Command+1...4` 工作区键盘导航。
- `WorkspaceTab.sidebarSubtitle` 是 regular 大屏侧栏的用途说明，应该进入辅助 hint，避免视觉和辅助语义分叉。
- `LocalGemmaApp` 的 `工作区` command menu 复用 `WorkspaceTab.commandItems`。
- `SelectionAccessibilityMetadata` 当前只提供 workspace/session label/value，缺少工作区专属 hint/input labels。
- 本轮只改变 SwiftUI button 辅助语义，不改变工作区切换状态流。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 中新增 `WorkspaceNavigationAccessibilityMetadata`，或在不破坏旧测试的前提下扩展现有 selection metadata。
2. metadata 至少提供：
   - `label(for:)`
   - `value(isSelected:)`
   - `hint(for:)`
   - `inputLabels(for:)`
   - `compactIdentifier(for:)`
   - `sidebarIdentifier(for:)`
3. `hint(for:)` 必须包含：
   - `Command \(tab.shortcutKey)`
   - `tab.sidebarSubtitle`
   - “只切换本地工作区”或等价边界说明。
4. 将 `tabPicker` 和 `sidebarTabPicker` 的 label/value/identifier 切换到该 metadata，并新增 hint/input labels。
5. 保留 `.keyboardShortcut`、`.accessibilityAddTraits(.isSelected)` 和当前视觉结构。
6. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增 `testWorkspaceNavigationAccessibilityMetadataDescribesShortcutsAndVoiceControl`，覆盖：
   - 四个 tab compact/sidebar identifiers。
   - hint 包含对应 `Command 1...4` 和 `sidebarSubtitle`。
   - input labels 包含 `\(tab.title)工作区`。
   - label/value 与 selection 语义一致。
   - metadata 对 `WorkspaceTab.allCases` 全量非空。
7. 更新文档中的当前测试数量和 Mac/iPad 导航辅助语义说明。
