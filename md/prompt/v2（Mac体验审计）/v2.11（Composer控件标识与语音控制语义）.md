# v2.11（Composer控件标识与语音控制语义）Agent B 实现提示词

## 版本号和分配依据

- 版本号：`v2.11`
- 分配依据：`update_log.md` 最新正式版本为 `v2.10 / 提示词模板动作辅助语义`。Agent X 继续执行“优化 UI、Mac、iPad 版”的总目标；v2.10 遗留事项明确还有 composer 按钮 identifier 缺口，本轮按推理页主输入工作流继续递增到 `v2.11`。

## 背景

当前推理页已经具备：

- `ComposerFocusRequest` / `ComposerFocusPolicy`：切回推理页、新建/切换会话、提示词模板填入或发送后请求 composer 聚焦。
- `ComposerInputMetadata`：输入框 label/hint/input labels，发送/停止按钮 label/value/hint。
- `Command+Return`：发送 prompt 或停止生成。

缺口是 composer 主输入框没有稳定 `accessibilityIdentifier`，发送/停止按钮仍是裸 `Image`，缺少 Voice Control input labels 和稳定 identifier；发送/停止 hint 也还没有完整说明不下载模型权重、不启动真实 runtime、不发送云端服务和不绕过 verified 门禁。Mac Catalyst、iPad 外接键盘、VoiceOver、Voice Control 和后续 UI 自动化都需要这层稳定发现语义。

## 目标

1. 扩展 `ComposerInputMetadata`，为 composer 输入框和发送/停止按钮生成稳定：
   - text field identifier
   - action input labels
   - action identifier
   - 更完整的 action hint
2. 输入框必须保持现有 label/hint/input labels，并新增稳定 identifier。
3. 发送/停止按钮必须：
   - 保持现有视觉形态和 `Command+Return` 行为。
   - 使用 `Label(...).labelStyle(.iconOnly)` 或等价方式，避免纯图标按钮对辅助技术不可见。
   - 在发送、空输入禁用、生成中停止三态下有可测试 label/value/hint/input labels/identifier。
4. `hint` 必须说明：
   - 发送当前输入给本地模拟 runtime。
   - 空输入时需要输入内容后才可发送。
   - 生成中时可停止当前模拟生成。
   - 不下载模型权重。
   - 不启动真实 runtime。
   - 不发送到云端服务。
   - 不绕过 verified 门禁。
5. 新增或扩展 XCTest，测试函数数从 55 增加到 56 或保持 55 但覆盖显著增强；推荐新增独立测试或扩展现有 `testComposerInputMetadataAndFocusPolicyDescribeEntryPoints`。

## 非目标

- 不修改 `InferenceEngine.send`、`stop`、空输入保护或流式生成逻辑。
- 不修改提示词模板、会话管理、模型状态、artifact validation、runtime plan 或 verified 门禁。
- 不创建原生 macOS target。
- 不下载 Gemma 或任何模型权重。
- 不接入云端推理、外部模型服务或真实 Core ML 推理。
- 不新增 UI Test target，不做无关重构。

## 当前架构依据

- `md/flow/flow.md`：`ComposerFocusRequest`、`ComposerFocusPolicy` 和 `ComposerInputMetadata` 只管理 view 层输入焦点与辅助输入文案，不写入 `InferenceEngine` 业务状态。
- `LocalGemma/ContentView.swift`：
  - `ComposerInputMetadata` 位于 metadata enum 区域。
  - `ComposerBar` 的 `TextField` 接入 label/hint/input labels，但缺 identifier。
  - `ComposerBar` 的发送/停止按钮接入 label/value/hint 和 keyboard shortcut，但缺 input labels/identifier，且当前 label 是裸 `Image`。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - `testComposerInputMetadataAndFocusPolicyDescribeEntryPoints` 已覆盖 composer label/value/hint、空输入/生成中状态和 focus policy，可作为扩展点。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 扩展 `ComposerInputMetadata`。
2. 推荐新增接口：
   - `static let textFieldIdentifier = "composer-input-field"`
   - `static func actionInputLabels(isGenerating: Bool) -> [String]`
   - `static func actionIdentifier(isGenerating: Bool) -> String`
3. 补强 `actionHint(text:isGenerating:)`：
   - 发送可用时说明本地模拟 runtime、本地边界和 verified 门禁。
   - 空输入禁用时说明输入内容后可发送，并保留本地边界。
   - 生成中时说明停止当前模拟生成，并保留本地边界。
4. `ComposerBar`：
   - `TextField` 增加 `.accessibilityIdentifier(ComposerInputMetadata.textFieldIdentifier)`。
   - 发送/停止按钮 label 从 `Image` 改为 `Label(ComposerInputMetadata.actionLabel(...), systemImage: ...)` 并 `.labelStyle(.iconOnly)`。
   - 按钮增加 `.accessibilityInputLabels(...)` 和 `.accessibilityIdentifier(...)`。
5. 更新 XCTest：
   - 断言 text field identifier。
   - 断言发送/停止 input labels。
   - 断言发送/停止 identifier。
   - 断言空输入、可发送、生成中三个 hint 均包含必要本地/云端/verified 边界。
   - 保留现有 focus policy 覆盖。
6. 同步文档：
   - `AGENTS.md`
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`

## Agent C 验收标准

Agent C 必须只验收最新 `origin/main` commit 对应的 run 和 artifact：

1. `git fetch origin`
2. 核对 `git rev-parse origin/main` 等于 v2.11 commit。
3. `gh run list --workflow ci-results.yml --branch main --limit 5` 找到最新 run。
4. `gh run download <run_id> --dir /private/tmp/localgemma-c-review-<run_id>`
5. 核对：
   - manifest `branch=main`
   - manifest `version=v2.11`
   - manifest `commitSha` 等于 `origin/main`
   - manifest `runId` / `runAttempt` 等于下载 run
   - `artifactName` 等于 `artifact-name.txt`
   - required outcomes：static、logic、iOS build、XCTest、Mac Catalyst build、Mac run script 均为 success
   - optional Codex Run environment 允许 skipped 但必须有 skipped reason
   - `junit.xml` failures 为 0
   - `xcodebuild.log`、`test.log`、`mac-catalyst-build.log` 有成功记录
   - `.xcresult` Info.plist 可通过 `plutil -lint`
