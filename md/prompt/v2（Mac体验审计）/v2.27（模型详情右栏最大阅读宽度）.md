# v2.27 模型详情右栏最大阅读宽度 - Agent B 实现提示词

## 版本号与分配依据

- 版本号：v2.27。
- 分配依据：`update_log.md` 当前最新正式版本为 v2.26，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增。

## 背景

模型页在 `ModelLibraryLayoutMode.twoColumn` 下会把左侧选择/部署/文件操作和右侧 `ModelDetailColumn` 并列展示。当前双栏路径中 `ModelDetailColumn` 使用 `.frame(maxWidth: .infinity)`，在 Mac Catalyst 或 iPad 外接大屏宽窗口下会无限吃满剩余空间，导致概要、参数、性能和建议面板的阅读行宽过长。项目已有聊天气泡、composer、提示词模板、优化网格等宽屏布局策略，本轮应为模型详情右栏补同类可测试宽度策略。

## 目标

- 新增 `ModelDetailColumnLayoutPolicy` 或等价布局策略，集中定义模型详情列在双栏模式下的最小可读宽度、最大阅读宽度、列间距和无效宽度 clamp。
- `ModelLibraryView` 的双栏路径中，右侧 `ModelDetailColumn` 不再 `.frame(maxWidth: .infinity)`，而是使用策略计算出的宽度，并保持垂直顶部对齐。
- 单栏模型页保持现有行为，不限制为窄卡片，不改变窄屏顺序。
- 不改变 `ModelDetailAccessibilityMetadata`、`ModelDetailRowAccessibilityMetadata`、`ModelSummaryAccessibilityMetadata` 或现有行级辅助语义。
- 新增 XCTest 锁住模型详情列宽度策略：窄屏返回 0 或不启用，iPad/Mac 宽区域使用可用宽度但封顶，极宽窗口不超过最大阅读宽度，无效宽度安全 clamp。
- 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md`。

## 非目标

- 不重构 `ContentView.swift` 文件结构。
- 不修改 `ModelCatalog`、artifact 校验、卸载确认、部署状态或 runtime 计划。
- 不新增真实模型 runtime，不下载 Gemma 权重，不接入外部推理服务。
- 不改变模型详情的文案、辅助语义、Voice Control 输入标签或 identifier。
- 不改变模型页双栏触发阈值，除非测试证明必须同步；默认继续复用 `ModelLibraryLayoutMode.resolve`。

## 当前架构依据

- `md/flow/flow.md` 规定 `ModelLibraryLayoutMode` 只控制模型页内部单栏/双栏；模型详情整体和行级辅助语义由 `ModelDetailAccessibilityMetadata` 与 `ModelDetailRowAccessibilityMetadata` 控制。
- `LocalGemma/ContentView.swift` 当前：
  - `ModelLibraryLayoutMode` 位于文件顶部，提供 `resolve(for:)` 和 `controlColumnWidth(for:)`。
  - `ModelLibraryView.deploymentContent(size:)` 在双栏路径中使用 `HStack(alignment: .top, spacing: 14)`。
  - 左侧控制列使用 `layoutMode.controlColumnWidth(for: size)`。
  - 右侧 `ModelDetailColumn` 当前使用 `.frame(maxWidth: .infinity)`。
  - `ModelDetailColumn` 本身负责概要、参数、性能和建议面板，并接入整体详情辅助语义。
- `LocalGemmaTests/LocalGemmaTests.swift` 当前：
  - `testModelLibraryLayoutModeSupportsWideModelWorkflows` 覆盖模型页单栏/双栏和控制列宽度。
  - `testModelDetailColumnExposesAccessibilityMetadata` 和 `testModelDetailRowsExposeAccessibilityMetadata` 覆盖详情辅助语义和行级辅助语义。

## 实现步骤

1. 在 `ModelLibraryLayoutMode` 附近新增 `ModelDetailColumnLayoutPolicy`。
2. 建议常量：
   - `minimumReadableWidth: CGFloat = 320`
   - `maximumReadableWidth: CGFloat = 680`
   - `interColumnSpacing: CGFloat = 14`
3. 新增函数，例如：
   - `static func width(for size: CGSize, layoutMode: ModelLibraryLayoutMode) -> CGFloat`
   - 单栏返回 `0`，表示调用方不启用固定详情列宽。
   - 双栏用 `size.width - layoutMode.controlColumnWidth(for: size) - interColumnSpacing` 计算可用详情宽度。
   - 非有限、负数或 0 宽度返回 `0`。
   - 正常双栏宽度 clamp 到 `minimumReadableWidth...maximumReadableWidth`。
4. 在 `ModelLibraryView.deploymentContent(size:)` 双栏路径中：
   - `HStack` spacing 复用 `ModelDetailColumnLayoutPolicy.interColumnSpacing`。
   - `ModelDetailColumn` 使用 `.frame(width: ModelDetailColumnLayoutPolicy.width(for: size, layoutMode: layoutMode), alignment: .topLeading)` 或等价方式。
   - 不改变单栏路径的 `ModelDetailColumn`。
5. 保持 `ModelDetailColumn` 的 `.accessibilityElement(children: .contain)` 和所有 label/value/hint/input labels/identifier 不变。
