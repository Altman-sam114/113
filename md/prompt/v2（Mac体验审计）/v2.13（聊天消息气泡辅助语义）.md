# v2.13（聊天消息气泡辅助语义）

## 版本号和版本分配依据

- 版本号：v2.13。
- 分配依据：`update_log.md` 最新正式记录为 v2.12，本轮继续 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”，选择 v2.12 遗留的推理页聊天消息气泡整体辅助语义作为下一轮可独立验证目标。

## 背景

当前项目已为工作区导航、会话栏操作、导出弹层、composer 输入框和发送/停止按钮、模型页、提示词页、设置页优化控件补充了稳定辅助语义。推理页 `ChatBubble` 仍主要依赖内部 `Text` 的默认读取，VoiceOver 在 Mac Catalyst / iPad 大屏消息流中会把角色、正文和 token 数拆开读，缺少统一的消息身份、生成状态、token 摘要、Voice Control 输入标签和稳定 identifier。

本轮只补齐消息气泡的只读辅助语义，不改变会话、生成、导出、runtime 或布局行为。

## 目标

1. 新增 `ChatMessageAccessibilityMetadata`。
2. 为 `ChatBubble` 接入整体 `.accessibilityElement(children: .ignore)`、label、value、hint、input labels 和 identifier。
3. 辅助语义必须区分用户消息、本地模型消息和系统状态消息。
4. 空 assistant 文本必须描述为“正在生成”，避免 VoiceOver 读到空内容。
5. value 必须合并消息正文或生成状态、token 数和本地模拟/runtime 边界摘要。
6. hint 必须说明消息气泡只展示本地会话内容，不下载模型权重、不启动真实 runtime、不发送到云端服务、不绕过 artifact verified 门禁。
7. identifier 必须稳定且不泄露完整 UUID；可使用 message role 和 UUID 前 8 位。
8. 新增 XCTest 锁住 label/value/hint/input labels/identifier、空文本生成态和三种角色。
9. 同步更新 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md`。

## 非目标

- 不修改 `InferenceEngine` 消息创建、流式生成、停止、会话命名或导出逻辑。
- 不修改 `ChatMessage` 数据结构字段。
- 不新增 UI Test target。
- 不调整 `ChatBubble` 可见布局、配色、字体、token 显示或滚动逻辑。
- 不下载 Gemma 权重，不接入云端推理，不把 staged artifact 提升为 verified。
- 不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `ChatTranscript` 负责遍历 `ChatMessage` 并渲染 `ChatBubble`。
  - `ChatBubble` 位于推理页消息流，当前显示角色标题、消息正文和 token 数。
  - 已有 metadata 模式包括 `ComposerInputMetadata`、`PromptTemplateActionAccessibilityMetadata`、`OptimizerMetricAccessibilityMetadata` 等，可复用同类静态 helper 设计。
- `LocalGemma/AppState.swift`
  - `ChatMessage` 包含 `id`、`role`、`text`、`timestamp` 和 `tokens`。
  - 当前 runtime 仍是 `SimulatedGemmaRuntime`，真实 runtime 受 artifact verified 门禁保护。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前基线为 56 个 `test...` 方法。
  - 新增测试建议命名为 `testChatMessagesExposeAccessibilityMetadata`。
- `md/flow/flow.md`
  - 推理页和 composer 已有辅助语义边界，本轮需要把消息气泡纳入同一 Mac/iPad 可访问性基线。
- `.github/workflows/ci-results.yml`
  - 仍负责 main push 后的静态检查、逻辑烟测、iOS build/test、Mac Catalyst build、run script contract 和 artifact 上传。

## 实现步骤

1. 在 `ContentView.swift` 的现有辅助语义 helper 区域新增 `ChatMessageAccessibilityMetadata`。
2. 建议接口：
   - `label(for message: ChatMessage) -> String`
   - `value(for message: ChatMessage) -> String`
   - `hint(for message: ChatMessage) -> String`
   - `inputLabels(for message: ChatMessage) -> [String]`
   - `identifier(for message: ChatMessage) -> String`
   - `roleTitle(for role: ChatMessage.Role) -> String`
   - `spokenText(for message: ChatMessage) -> String`
3. `label` 示例：
   - 用户：`用户消息`
   - assistant：`本地模型消息`
   - system：`系统状态消息`
4. `value` 示例：
   - 非空：`<正文>。<n> tokens。本地会话消息。`
   - 空 assistant：`正在生成。本地模型正在写入模拟输出。<n> tokens。本地会话消息。`
5. `hint` 必须包含：
   - `只展示本地会话内容`
   - `不会下载模型权重`
   - `不会启动真实 runtime`
   - `不会发送到云端服务`
   - `verified 门禁`
6. `identifier` 用 `chat-message-<role>-<uuid8>`，保证稳定且短。
7. `inputLabels` 至少包含角色短语和查看动作，例如 `["本地模型消息", "查看本地模型消息", "消息 <uuid8>"]`。
8. 在 `ChatBubble` 最外层或气泡内容容器上接入：
   - `.accessibilityElement(children: .ignore)`
   - `.accessibilityLabel(...)`
   - `.accessibilityValue(...)`
   - `.accessibilityHint(...)`
   - `.accessibilityInputLabels(...)`
   - `.accessibilityIdentifier(...)`
9. 新增 XCTest：
   - 构造固定 UUID 的 user / assistant / empty assistant / system 消息。
   - 覆盖 label 三角色。
   - 覆盖 value 包含正文、`正在生成`、token 数、本地会话消息。
   - 覆盖 hint 的本地/云端/模型下载/真实 runtime/verified 门禁边界。
   - 覆盖 input labels 非空且包含角色短语。
   - 覆盖 identifier 前缀、role 和 UUID 前 8 位。
10. 更新文档测试数：56 -> 57。

## 风险和禁止项

- 不要把消息辅助语义写成可交互按钮；消息气泡是只读内容。
- 不要把 assistant 空文本读成空字符串，必须描述为正在生成。
- 不要泄露完整 UUID 或把用户 prompt 作为 identifier。
- 不要改变 token 统计、消息文本、导出文本或会话排序。
- 禁止下载模型权重、接入云端推理、绕过 artifact verified 门禁、伪造测试结果或验收旧 artifact。
