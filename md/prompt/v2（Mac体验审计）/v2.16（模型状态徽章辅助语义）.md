# v2.16 / 模型状态徽章辅助语义

## 版本号和分配依据

- 版本号：`v2.16`
- 分配依据：`update_log.md` 最新正式版本为 `v2.15 / 聊天记录容器辅助语义`，本轮继续递增到 `v2.16`。
- 所属总目标：Agent X 持续优化 UI、Mac 和 iPad 体验。

## 背景

当前模型页已经为模型选择器、部署电源、artifact 操作按钮和模型详情右栏建立了辅助语义。模型选择器面板底部仍有三个视觉状态徽章：`StatusBadge`、`AvailabilityBadge`、`DeploymentBadge`。这些徽章在 Mac/iPad 宽屏模型页中用于快速扫描安装状态、artifact 状态和部署状态，但当前只显示英文短文本，缺少独立的 VoiceOver / Voice Control 语义和稳定 identifier。

## 目标

1. 为模型状态徽章建立统一的 `ModelStatusBadgeAccessibilityMetadata`。
2. 让 `StatusBadge`、`AvailabilityBadge`、`DeploymentBadge` 暴露 label、value、hint、input labels 和稳定 identifier。
3. 文案必须明确徽章只展示本地模型状态，不下载模型权重、不启动真实 runtime、不发送云端服务、不绕过 artifact verified 门禁。
4. 新增 XCTest 覆盖三类徽章的所有状态、Voice Control 输入标签、identifier 和边界文案。
5. 同步 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 非目标

- 不修改 `ModelCatalog`、`LocalArtifactValidator`、`LocalRuntimePlanner` 或部署状态流。
- 不改变模型页布局、视觉样式、颜色或断点。
- 不新增 UI Test target。
- 不下载模型权重，不接入云端推理，不接入真实 runtime，不创建原生 macOS target。
- 不绕过 artifact `verified` 门禁。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `ModelSelectorPanel` 展示模型选择器和三个状态徽章。
  - `StatusBadge` 展示 `ModelInstallState.title`。
  - `AvailabilityBadge` 展示 `ArtifactAvailability.title`。
  - `DeploymentBadge` 展示 `ModelDeploymentState.title`。
  - 已有同类模式：`ModelCapsuleAccessibilityMetadata`、`ModelDetailAccessibilityMetadata`、`ModelDeploymentControlAccessibilityMetadata`。
- `LocalGemma/AppState.swift`
  - `ModelInstallState`、`ArtifactAvailability`、`ModelDeploymentState` 定义状态枚举。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试基线为 59 个 XCTest 方法。

## 实现步骤

1. 在 `ContentView.swift` 中新增 `ModelStatusBadgeAccessibilityMetadata`。
2. 元数据建议包含：
   - `label(for installState:)`
   - `value(for installState:)`
   - `inputLabels(for installState:)`
   - `identifier(for installState:)`
   - `label(for availability:)`
   - `value(for availability:)`
   - `inputLabels(for availability:)`
   - `identifier(for availability:)`
   - `label(for deploymentState:)`
   - `value(for deploymentState:)`
   - `inputLabels(for deploymentState:)`
   - `identifier(for deploymentState:)`
   - 统一 `hint`
3. 将 `StatusBadge`、`AvailabilityBadge`、`DeploymentBadge` 设置为独立只读辅助元素。
4. 不改变 badge 的视觉文本和原有布局。
5. 在 `LocalGemmaTests.swift` 新增 `testModelStatusBadgesExposeAccessibilityMetadata`，测试函数数从 59 增加到 60。

## 风险和禁止项

- 风险：徽章英文视觉文本保留，但辅助 value 应提供中文解释，避免 Mac/iPad VoiceOver 用户只听到短英文状态。
- 风险：identifier 必须稳定且不依赖动态模型名称，避免测试脆弱。
- 禁止下载模型权重、云端推理、绕过 verified 门禁。
- 禁止把未跑测试写成已通过。
- 禁止验收旧 run 或旧 artifact。
