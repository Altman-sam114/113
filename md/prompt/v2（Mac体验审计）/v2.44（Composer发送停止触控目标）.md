# v2.44 / Composer 发送停止触控目标

## 版本号和分配依据

- 版本号：v2.44。
- 分配依据：`update_log.md` 最新正式版本为 v2.43，本轮继续 Agent X 总目标“优化 UI、Mac、iPad 版”的下一项独立可验证小目标。

## 背景

当前推理页 composer 已通过 `ComposerBarLayoutPolicy` 锁住宽屏输入行宽，通过 `ComposerInputMetadata` 和 `ComposerFocusPolicy` 锁住输入框、发送/停止按钮辅助语义和焦点流。发送/停止主动作按钮实际使用 48x48 frame，已经满足 Apple 44pt 最小触控目标，但该尺寸仍是 `ComposerBar` 内部硬编码，缺少独立策略和测试；后续 UI 调整可能误压缩触控目标，影响 iPad/Mac Catalyst 和触控用户。

## 目标

- 新增 composer 发送/停止按钮触控目标策略。
- 发送和停止两种动作都保持至少 44pt 命中目标，并保留当前 48pt 视觉尺寸。
- 新增测试锁住最小触控目标、按钮尺寸、send/stop action 映射和 identifier 边界。
- 同步 README、测试规范、核心流程、流程图、入口规则、更新日志和本提示词归档。

## 非目标

- 不修改 `ComposerBarLayoutPolicy` 的宽屏输入宽度策略。
- 不修改 `ComposerInputMetadata` 的 label/value/hint/input labels/identifier。
- 不修改发送、停止、空输入禁用、`Command+Return` 或焦点逻辑。
- 不修改 `InferenceEngine`、会话、消息、导出、提示词模板或模型 runtime。
- 不接入真实 runtime，不下载模型权重，不发送到云端服务，不绕过 verified 门禁。

## 当前架构依据

- `md/flow/flow.md` 规定 composer 输入框和发送/停止按钮只管理本地 UI 输入、焦点、快捷键和辅助语义，不写入额外业务状态。
- `LocalGemma/ContentView.swift` 中 `ComposerBar` 持有发送/停止按钮，当前按钮保留文本语义的 `Label(...).labelStyle(.iconOnly)`。
- `LocalGemmaTests/LocalGemmaTests.swift` 已有 `testComposerBarLayoutPolicyConstrainsWideComposerInput` 和 `testComposerInputMetadataAndFocusPolicyDescribeEntryPoints`。
- SwiftUI 设计规则要求交互区域至少 44x44。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 新增：
   - `ComposerInputAction: CaseIterable`，包含 `.send` 和 `.stop`，并暴露 `isGenerating` 映射。
   - `ComposerInputActionLayoutPolicy`，包含 `minimumTouchTarget = 44`、`actionButtonSize = 48`、`buttonSize(for:)` 和 `usesMinimumTouchTarget(for:)`。
2. 调整 `ComposerBar`：
   - 新增 `currentAction` 只从 `isGenerating` 派生。
   - 将发送/停止按钮 `.frame(width: 48, height: 48)` 改为使用 `ComposerInputActionLayoutPolicy.buttonSize(for: currentAction)`。
   - 保留 `send()`、`stop()`、`.keyboardShortcut(.return, modifiers: [.command])`、`.disabled(isSendDisabled)`、opacity、focus 和 accessibility metadata。
3. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增 `testComposerInputActionLayoutPolicyMaintainsTouchTargets`：
   - 断言 `minimumTouchTarget == 44`。
   - 断言 `actionButtonSize == 48` 且不小于最小触控目标。
   - 遍历 `ComposerInputAction.allCases`，断言 button size 和 `usesMinimumTouchTarget`。
   - 断言 `.send.isGenerating == false`、`.stop.isGenerating == true`。
   - 断言两种 action 仍映射到 `composer-send-button` 和 `composer-stop-button`。
4. 同步文档：
   - `AGENTS.md`
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`
