# v2.48 优化指标卡文本动态排版

## 版本号与分配依据

- 版本号：v2.48。
- 分配依据：`update_log.md` 最新正式轮次为 v2.47，本轮在 Agent X “全面优化 UI、Mac 与 iPad 体验”总目标下继续递增。

## 背景

设置页和优化 dashboard 共用 `OptimizerMetricCard`。现有指标网格已经通过 `OptimizerMetricGridLayoutPolicy` 在窄屏回退单列，指标卡也已有完整辅助语义，但 label、value 和 detail 仍使用固定字号；label/value 还限制单行并依赖 `minimumScaleFactor` 压缩。iPad split view、Mac Catalyst 窄窗口和较大 Dynamic Type 下，这会降低指标可读性。

## 目标

- 新增 `OptimizerMetricTextLayoutPolicy`，集中定义指标卡文本间距、状态圆点尺寸、行数、detail 行距和最小卡片高度。
- label、value 和 detail 改用 Dynamic Type 语义字体。
- label/value/detail 允许多行并纵向增长，移除单行缩放压缩。
- 新增 XCTest 锁住策略常量、多行能力、44pt 以上最小高度和既有网格阈值。
- 同步 README、测试规范、核心流程、流程图、入口规则、更新日志和本提示词归档。

## 非目标

- 不改变 `DeviceOptimizer.metrics` 默认数据或优化开关状态流。
- 不改变 `OptimizerMetricAccessibilityMetadata` 文案、Voice Control 输入标签或 identifier。
- 不改变 `OptimizerMetricGridLayoutPolicy` 单列/双列阈值。
- 不改变设置页整体宽度、芯片准备度、隐私保护摘要、模型文件或 runtime 状态。
- 不下载模型权重，不接入云端推理，不绕过 artifact verified 门禁，不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `OptimizerMetricCard` 同时被设置页和优化 dashboard 使用。
  - `OptimizerMetricAccessibilityMetadata` 负责辅助语义。
  - `OptimizerMetricGridLayoutPolicy` 负责窄屏单列和宽屏双列。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 已有优化指标卡辅助语义和网格宽度策略测试。
  - 本轮新增文本布局策略测试，测试函数数从 90 增至 91。

## 实现步骤

1. 在 `ContentView.swift` 增加 `OptimizerMetricTextLayoutPolicy`。
2. 策略包含 `verticalSpacing`、`indicatorSize`、label/value/detail 行数、detail 行距和 `minimumCardHeight`。
3. label 使用语义 footnote，value 使用语义 title3，detail 使用语义 subheadline。
4. 三段文本允许多行并使用 vertical `fixedSize`，移除 `minimumScaleFactor`。
5. 保留 progress、tint、指标数据、辅助语义和网格列数状态流。
6. 新增 `testOptimizerMetricTextLayoutPolicySupportsDynamicTypeCards`。
7. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md`。
