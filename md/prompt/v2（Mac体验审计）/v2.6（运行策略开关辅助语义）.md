# v2.6 / 运行策略开关辅助语义

## 版本号和分配依据

- 版本号：`v2.6`
- 分配依据：`update_log.md` 当前最新完成版本为 `v2.5 / 工作区导航辅助语义`，本轮继续围绕 Mac/iPad 设置页和 Apple Silicon 优化面板的辅助交互基线递增。

## 背景

Agent X 在 v2.5 验收通过后并发做了下一轮 UI/Mac/iPad 审计，两个独立只读子 agent 均指出设置页“运行策略”开关是当前最小且可验证的缺口：

- `OptimizationToggleRow` 当前只是一个 `Button`，依赖图标和标题/副标题表达开关状态。
- 它没有统一的 `accessibilityLabel`、`accessibilityValue`、`accessibilityHint`、`accessibilityInputLabels` 和稳定 `accessibilityIdentifier`。
- 现有测试 `testOptimizerTogglesChangeReadiness` 只验证 `DeviceOptimizer.toggle` 会改变 readiness，没有锁住 Mac/iPad VoiceOver / Voice Control 入口语义。

本轮目标是补齐运行策略开关辅助语义，延续模型部署控件、壁纸控件、头部入口、会话栏和工作区导航已建立的 metadata 模式。

## 目标

1. 新增 `OptimizationToggleAccessibilityMetadata` 或等价 metadata。
2. 为每个 `OptimizationSwitch` 生成：
   - label：包含策略标题，例如 `运行策略 Metal graph prewarm`。
   - value：区分 `已开启` / `已关闭`，并包含开关副标题。
   - hint：明确只切换本地运行策略，不下载模型权重、不启动真实 runtime、不发送云端服务。
   - input labels：包含策略标题、`开启...`、`关闭...` 或等价中文 Voice Control 命令。
   - identifier：稳定且可预测，例如基于标题 slug 的 `optimizer-toggle-metal-graph-prewarm`。
3. 将 metadata 应用到 `OptimizationToggleRow`。
4. 保留现有按钮行为：点击后仍调用 `optimizer.toggle(item)`，不改变 `DeviceOptimizer` 状态流和 readiness 计算。
5. 新增 XCTest，测试函数数从 50 增加到 51，覆盖四个默认运行策略的 label/value/hint/input labels/identifier、开启/关闭状态和本地边界文案。
6. 同步 `AGENTS.md`、`README.md`、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 非目标

- 不修改 `DeviceOptimizer.toggle` 的 readiness 公式。
- 不修改优化指标、热状态、`ChipReadinessCard` 动态摘要或 readiness ring。
- 不改变设置页布局、颜色、按钮视觉、开关顺序或标题。
- 不接入真实模型推理，不下载模型权重，不创建原生 macOS target。
- 不扩大到 `ModelCapsule`、模型详情右栏或 UI Test target。

## 当前架构依据

- `DeviceOptimizer` 是 Apple Silicon 优化面板的状态源，拥有 `metrics`、`switches`、`thermalState` 和 `deploymentReadiness`。
- `OptimizationSwitch` 是运行策略项，包含 `title`、`subtitle` 和 `isEnabled`。
- `SettingsWorkspace` 与 `OptimizerDashboard` 都复用 `OptimizationToggleRow` 渲染运行策略，因此在 row 层接入 metadata 可覆盖设置页和 dashboard 两处入口。
- 当前项目已经通过多个 metadata enum 锁住辅助语义，例如 `ModelDeploymentControlAccessibilityMetadata`、`WallpaperPreferenceAccessibilityMetadata`、`HeaderActionAccessibilityMetadata` 和 `WorkspaceNavigationAccessibilityMetadata`。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 中新增 `OptimizationToggleAccessibilityMetadata`。
2. metadata 至少提供：
   - `label(for:)`
   - `value(for:)`
   - `hint(for:)`
   - `inputLabels(for:)`
   - `identifier(for:)`
3. `hint(for:)` 必须包含：
   - “只切换本地运行策略”或等价说明。
   - “不会下载模型权重”。
   - “不会启动真实 runtime”。
   - “不会发送到云端服务”。
4. `identifier(for:)` 必须稳定，不依赖 UUID；推荐从 `item.title` 转成小写 slug，并处理空白和非字母数字字符。
5. 将 `OptimizationToggleRow` 增加：
   - `.accessibilityLabel(...)`
   - `.accessibilityValue(...)`
   - `.accessibilityHint(...)`
   - `.accessibilityInputLabels(...)`
   - `.accessibilityIdentifier(...)`
   - 可视情况添加 `.accessibilityAddTraits(item.isEnabled ? .isSelected : [])`，但不得破坏按钮行为。
6. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增 `testOptimizationToggleRowsExposeAccessibilityMetadata`，覆盖：
   - `DeviceOptimizer().switches.count == 4`
   - 四个 identifier 分别为稳定值。
   - 开启状态 value 包含 `已开启`，关闭状态 value 包含 `已关闭`。
   - value 包含对应 subtitle。
   - hint 包含本地运行策略、不会下载模型权重、不会启动真实 runtime、不会发送到云端服务。
   - input labels 非空，且包含策略标题。
   - `DeviceOptimizer.toggle` 后同一 switch 的 metadata value 从开启变为关闭。
7. 更新文档中的当前测试数量、设置页运行策略辅助语义和 Mac/iPad 审计遗留项。
