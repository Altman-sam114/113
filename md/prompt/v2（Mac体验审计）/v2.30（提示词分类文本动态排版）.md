# v2.30（提示词分类文本动态排版）

## 版本号与分配依据

- 版本号：`v2.30`
- 分配依据：`update_log.md` 最新正式版本为 `v2.29 / 提示词模板文本动态排版`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版”下继续递增。

## 背景

v2.28 已将提示词分类筛选从横向滚动改为自适应换行，并锁住 44pt 触控目标。v2.29 已将提示词模板卡片主要文本改为 Dynamic Type 语义字体。但 `PromptCategorySelector` 内的筛选 chip 仍使用固定 `11pt` 字体，较大 Dynamic Type、iPad split view 和 Mac Catalyst 窄窗口下仍存在文本可读性风险。

## 目标

1. 新增提示词分类筛选文本排版策略，集中定义分类 chip 文本行数和多行能力。
2. 将分类筛选 chip 文本从固定小字号改为 SwiftUI Dynamic Type 语义字体。
3. 保留 v2.28 的换行布局、44pt 触控目标、最小 chip 宽度和无效宽度 clamp。
4. 新增 XCTest 锁住文本排版策略，并确认与现有 `PromptCategoryLayoutPolicy` 兼容。
5. 同步 README、测试规范、核心流程文档、流程图、入口规则和更新日志。

## 非目标

- 不改变提示词分类枚举、模板库内容或筛选结果。
- 不改变 `selectedCategory` 状态流、模板填入/发送、生成中禁用或 composer 聚焦。
- 不改变 `PromptTemplateGridLayoutPolicy` 的列数阈值和卡片宽度。
- 不改变 `PromptTemplateActionAccessibilityMetadata` 或 `PromptCategoryAccessibilityMetadata` 的 label/value/hint/input labels/identifier。
- 不新增真实模型 runtime，不下载模型权重，不接入云端推理。
- 不创建原生 macOS target，不新增 UI Test target。

## 当前架构依据

- `md/flow/flow.md` 中提示词页由 `PromptTemplatesWorkspace`、`PromptCategorySelector`、`PromptCategoryFlowLayout`、`PromptTemplateGrid` 和 `PromptTemplateCard` 负责 UI。
- `PromptCategoryLayoutPolicy` 负责分类筛选 chip 的换行布局、44pt 触控目标、spacing、padding、最小 chip 宽度和无效宽度 clamp。
- `PromptCategoryAccessibilityMetadata` 负责分类筛选的 VoiceOver / Voice Control 语义，本轮不得改变其文案和 identifier。
- `PromptTemplateTextLayoutPolicy` 和 `PromptTemplateActionLayoutPolicy` 已分别锁住模板卡片文本可读性和动作触控目标，本轮不扩大到模板卡片。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的 `PromptCategoryLayoutPolicy` 附近新增 `PromptCategoryTextLayoutPolicy`。
2. 将 `PromptCategorySelector.categoryButton` 中 `Label` 的固定 `.system(size: 11, weight: .black)` 改为 `.subheadline` + `.bold()`，并使用文本策略的 line limit。
3. 保留 `Button`、`Label`、`.titleAndIcon`、当前背景/描边、`PromptCategoryAccessibilityMetadata`、`.isSelected` trait 和所有 action。
4. 在 `LocalGemmaTests/LocalGemmaTests.swift` 新增测试，锁住分类文本 line limit、多行能力和 44pt 触控兼容性。
5. 同步文档和日志，将测试函数数从 73 更新到 74。
