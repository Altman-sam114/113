# v2.15（聊天记录容器辅助语义）

## 版本号和版本分配依据

- 版本号：v2.15。
- 分配依据：`update_log.md` 最新正式记录为 v2.14，本轮继续 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”。v2.13 已补齐单条聊天消息气泡语义，v2.14 已补齐会话 chip 动作语义；只读审计指出 `ChatTranscript` 消息列表容器仍缺列表级 VoiceOver / Voice Control 摘要，因此作为 v2.15 小步目标。

## 背景

推理页现在已经为单条 `ChatBubble` 暴露角色、正文或生成中状态、token 数和本地边界，也为会话栏和会话 chip 暴露操作语义。但 `ChatTranscript` 作为消息列表容器仍没有稳定 label/value/hint/input labels/identifier。Mac/iPad VoiceOver 用户进入推理页时，缺少“这是当前本地会话记录、共有多少条消息、最新消息是什么状态、不会联网或启动真实 runtime”的列表级摘要。

本轮只补齐聊天记录容器辅助语义，不改变消息内容、滚动、会话状态、生成状态、导出、runtime 或布局。

## 目标

1. 新增 `ChatTranscriptAccessibilityMetadata`。
2. 为 `ChatTranscript` 的 `ScrollView` 容器接入 label、value、hint、input labels 和稳定 identifier。
3. value 必须覆盖：
   - 空消息列表 / 欢迎态。
   - 普通消息总数。
   - 最新消息角色和摘要。
   - assistant 空文本时的“正在生成”状态。
4. hint 必须说明只浏览本地会话记录，不发送 prompt、不下载模型权重、不启动真实 runtime、不发送云端服务、不绕过 artifact verified 门禁。
5. identifier 必须稳定且不包含用户 prompt 或消息正文。
6. 新增 XCTest `testChatTranscriptExposesAccessibilityMetadata`，覆盖空列表、普通列表、生成中最新消息、input labels、identifier 和本地/云端/verified 边界。
7. 测试函数数从 58 增加到 59。
8. 同步更新 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md`。

## 非目标

- 不修改 `InferenceEngine`、`ChatMessage`、会话创建/选择/删除、流式输出、导出或自动滚动逻辑。
- 不改变 `ChatBubble` 单条消息辅助语义。
- 不改变 `SessionBar`、`SessionChip`、composer、command menu 或布局断点。
- 不新增 UI Test target。
- 不下载 Gemma 权重，不接入云端推理，不把 staged artifact 提升为 verified。
- 不创建原生 macOS target。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `ChatTranscript` 接收 `[ChatMessage]` 并渲染 `ScrollViewReader`、`ScrollView` 和 `LazyVStack`。
  - `ChatBubble` 已接入 `ChatMessageAccessibilityMetadata`，可以复用其中的 `roleTitle(for:)` 与 `spokenText(for:)` 形成最新消息摘要。
  - 现有 metadata 模式均为 enum 静态 helper，可继续沿用。
- `LocalGemma/AppState.swift`
  - `ChatMessage` 含 `id`、`role`、`text`、`tokens`。
  - assistant 空文本表示本地模拟输出生成中。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前基线为 58 个 `test...` 方法。
  - 新测试应放在 `testChatMessagesExposeAccessibilityMetadata` 附近。
- `.github/workflows/ci-results.yml`
  - 仍负责 main push 后的静态检查、逻辑烟测、iOS build/test、Mac Catalyst build、run script contract 和 artifact 上传。

## 实现步骤

1. 在 `ContentView.swift` 的聊天消息辅助语义 helper 附近新增 `ChatTranscriptAccessibilityMetadata`。
2. 建议接口：
   - `static let label: String`
   - `static let hint: String`
   - `static let inputLabels: [String]`
   - `static let identifier: String`
   - `static func value(for messages: [ChatMessage]) -> String`
3. `value(for:)` 建议规则：
   - 空列表：`空聊天记录，当前没有本地会话消息。`
   - 非空：`聊天记录包含 N 条本地会话消息。最新<角色>：<摘要>。`
   - 摘要复用 `ChatMessageAccessibilityMetadata.spokenText(for:)`，但不要写入 identifier。
4. 在 `ChatTranscript` 的 `ScrollView` 上接入：
   - `.accessibilityElement(children: .contain)`
   - `.accessibilityLabel(...)`
   - `.accessibilityValue(...)`
   - `.accessibilityHint(...)`
   - `.accessibilityInputLabels(...)`
   - `.accessibilityIdentifier(...)`
5. 新增 XCTest：
   - 空列表 value。
   - 用户 + assistant 普通消息 value。
   - assistant 空文本生成中 value。
   - label、hint、input labels、identifier。
   - identifier 不包含消息正文。
6. 更新文档测试数：58 -> 59。
