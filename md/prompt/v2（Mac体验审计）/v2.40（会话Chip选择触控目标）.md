# v2.40 Agent B 提示词：会话 Chip 选择触控目标

## 版本号和分配依据

- 版本号：`v2.40`
- 分配依据：`update_log.md` 当前最新正式版本为 `v2.39: 优化工作区导航触控目标`，本轮继续 Agent X 的 Mac/iPad/UI 体验审计，递增为 `v2.40`。

## 背景

当前推理页单个会话 chip 已通过 `SessionChipActionAccessibilityMetadata` 暴露选择和删除动作语义。v2.37 已让删除图标按钮保持 44pt 触控目标，但 `SessionChipActionLayoutPolicy.usesMinimumTouchTarget(for: .select)` 仍明确返回 `false`，选择动作是高频入口，尤其在 iPad/Mac 大屏会话侧栏与横向会话栏中需要同样满足 Apple 44pt 最小触控建议。

## 目标

1. 让推理页单个会话 chip 的选择动作也由布局策略保证至少 44pt 触控高度。
2. 保留删除按钮既有 44pt 行为和 v2.37 语义。
3. 增加或更新 XCTest 锁住选择和删除动作都使用最小触控目标。
4. 同步 README、测试规范、核心流程、流程图、入口记忆和更新日志。

## 非目标

- 不改变会话选择、删除、默认空白当前会话保护或 composer 聚焦状态流。
- 不修改 `SessionChipActionAccessibilityMetadata` 的 label/value/hint/input labels/identifier 文案。
- 不调整会话侧栏宽度、会话栏新建/导出按钮、聊天气泡、composer 或导出弹层。
- 不下载模型权重，不接入云端推理，不绕过 artifact `verified` 门禁。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `SessionChipActionLayoutPolicy` 当前定义 `minimumTouchTarget` 和 `deleteButtonSize`。
  - `SessionChip` 内选择按钮是 `Button(action: select)`，删除按钮使用 `deleteButtonSize`。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - `testSessionChipActionLayoutPolicyMaintainsTouchTargets` 当前断言 delete 使用 44pt，select 不使用 44pt。
- `md/flow/flow.md` / `md/flow/flowchart.md`
  - 当前文档描述会话 chip 动作语义和删除触控目标，需要升级为选择/删除触控目标。

## 实现步骤

1. 扩展 `SessionChipActionLayoutPolicy`：
   - 保留 `minimumTouchTarget = 44`。
   - 新增选择动作最小高度常量，例如 `selectButtonMinHeight = minimumTouchTarget`。
   - 保留 `deleteButtonSize = minimumTouchTarget`。
   - `usesMinimumTouchTarget(for: .select)` 改为检查选择最小高度。
2. 在 `SessionChip` 的选择按钮 label 上应用最小高度：
   - 不改变视觉文本、图标或辅助语义。
   - 只增加 `frame(minHeight: ...)` 或等价最小命中高度。
3. 更新 `testSessionChipActionLayoutPolicyMaintainsTouchTargets`：
   - 断言 `selectButtonMinHeight >= minimumTouchTarget`。
   - 断言 select/delete 都返回 `true`。
4. 同步文档：
   - `AGENTS.md`
   - `README.md`
   - `md/test/test.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `update_log.md`
