# v2.1 / 会话栏操作辅助语义

## 版本号与分配依据

- 版本号：`v2.1`
- 分配依据：`update_log.md` 当前最新正式版本为 `v2.0 / 模型选择器辅助语义`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增到 `v2.1`。

## 背景

项目已经为系统 `会话` CommandMenu 建立了 `SessionCommandAction` 和 focused route 测试，也为会话选择 chip 建立了选择语义。但推理页会话栏中的可见“导出”和“新建”按钮仍只有英文 `accessibilityLabel`，缺少 `accessibilityValue`、`accessibilityHint`、Voice Control 输入标签和稳定 identifier。

这会让 Mac Catalyst 和 iPad 外接键盘用户在可见会话栏入口和系统菜单入口之间得到不一致的辅助语义。本轮只补可见按钮的辅助技术 metadata，并让它与现有 `SessionCommandAction` 标题、快捷键和 focus policy 对齐；不改变会话创建、导出、选择、删除或 composer focus 业务行为。

## 目标

1. 新增可测试的 `SessionBarActionAccessibilityMetadata` 或等价 metadata。
2. 为 `SessionBar` 的“导出当前会话”和“新建会话”按钮添加 label、value、hint、input labels 和 identifier。
3. label 必须复用或对齐 `SessionCommandAction.title`，避免可见按钮和系统 `会话` 菜单语义漂移。
4. value / hint 必须说明：
   - 新建会话会创建本地会话并请求 composer focus。
   - 导出当前会话会打开本地 Markdown / 文本分享流程，不发送到云端服务。
5. 新增纯 XCTest 锁住 metadata、identifier、input labels、快捷键说明和 focus policy 对齐，测试函数数从 45 增加到 46。
6. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md` 中的当前能力、测试数量和验证记录。

## 非目标

- 不修改 `InferenceEngine` 会话状态流、导出文本生成、`ExportPayload` 或分享兜底逻辑。
- 不修改 `SessionCommandAction` 的快捷键、菜单标题、focused route 或 command menu 注册方式，除非为了复用现有标题常量。
- 不改变会话栏可见布局、按钮图标、按钮顺序、颜色或 closure。
- 不新增 UI Test target，不做推理页视觉重设，不创建原生 macOS target。
- 不下载模型权重，不接入云端推理，不发送会话或提示词到外部服务。

## 当前架构依据

- `md/flow/flow.md`：`LocalGemmaApp` 的 `会话` command menu 复用 `SessionCommandAction`；`ChatWorkspace` 通过 `SessionCommandActions` focused value 暴露新建会话和导出当前会话动作。
- `LocalGemma/ContentView.swift`：
  - `SessionCommandAction` 定义 `createSession` / `exportSession` 的菜单标题、快捷键、shift 修饰和 focus reason。
  - `SessionCommandRoutingPolicy.requestsComposerFocus(after:)` 已说明新建会话请求 composer focus，导出不请求 focus。
  - `SessionBar` 的导出和新建按钮当前只有英文 `accessibilityLabel`，没有 value/hint/input labels/identifier。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - `testSessionCommandMenuCoversFocusedSessionActions` 和 `testSessionCommandFocusedRouteDescribesAvailabilityAndFocusPolicy` 是本轮对齐参考。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的会话命令 metadata 附近新增 `SessionBarActionAccessibilityMetadata`：
   - `label(for:)`
   - `value(for:)`
   - `hint(for:)`
   - `inputLabels(for:)`
   - `identifier(for:)`
2. metadata 应接受 `SessionCommandAction`，覆盖 `.createSession` 和 `.exportSession`。
3. 修改 `SessionBar`：
   - 导出按钮使用 `.exportSession` metadata。
   - 新建按钮使用 `.createSession` metadata。
   - 保持现有 UI 和按钮 action 不变。
4. 新增 XCTest，例如 `testSessionBarActionsExposeAccessibilityMetadata`：
   - label 与 `SessionCommandAction.title` 对齐。
   - identifiers 为稳定字符串，例如 `session-bar-action-createSession` / `session-bar-action-exportSession`。
   - value/hint 包含 `Command N`、`Command Shift E` 或等价快捷键说明。
   - 新建会话 hint/value 包含“聚焦输入框”或 `SessionCommandRoutingPolicy.requestsComposerFocus(after: .createSession) == true` 的语义。
   - 导出 hint/value 包含“本地 Markdown / 文本分享兜底”和“不发送到云端服务”，并断言 `requestsComposerFocus(after: .exportSession) == false`。
   - input labels 非空且包含中文动作名。
5. 同步文档：
   - `AGENTS.md`：一句话总览和核心边界加入会话栏操作辅助语义。
   - `md/flow/flow.md`：UI 工作区和核心模块加入 `SessionBarActionAccessibilityMetadata`。
   - `md/flow/flowchart.md`：推理页节点注明会话栏操作辅助语义。
   - `md/test/test.md`：测试数量从 45 更新到 46，覆盖范围和 Stage Regression 触发条件加入会话栏操作辅助语义。
   - `README.md`：当前范围、运行说明、推理页交互、验证章节和历史记录同步 v2.1。
   - `update_log.md`：当前核心测试改为 46，追加 v2.1 记录，写明本地轻量检查命令与结果。

## 风险与禁止项

- 风险：可见按钮 metadata 如果和 `SessionCommandAction` 标题/快捷键不一致，会让菜单和会话栏语义漂移；测试必须锁住对齐关系。
- 风险：导出 hint 不能暗示会上传或发送到外部服务；必须明确是本地 Markdown / 文本分享兜底。
- 风险：新建会话可以说明请求输入焦点，但不能宣称会发送 prompt 或创建模型推理。
- 禁止下载模型权重、云端推理、上传会话、绕过 verified 门禁、改真实 runtime、引入新第三方依赖、扩大到无关视觉重构。
- 禁止伪造本地或云端测试结果；没有跑的测试必须明确说明。
