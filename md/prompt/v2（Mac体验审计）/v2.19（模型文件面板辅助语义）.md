# v2.19 / 模型文件面板辅助语义

## 版本号和分配依据

- 版本号：`v2.19`
- 分配依据：`update_log.md` 最新正式版本为 `v2.18 / 模型概要面板辅助语义`，本轮继续递增到 `v2.19`。
- 所属总目标：Agent X 持续优化 UI、Mac 和 iPad 体验。

## 背景

模型页左侧部署控制台已经为模型选择器、状态徽章、部署电源和四个 artifact 操作按钮建立了辅助语义。Mac/iPad 宽屏双栏中，`ArtifactActionPanel` 是用户管理模型文件的工作流区域，包含模拟暂存、卸载、扫描本地和导入文件四个入口，但面板本身没有整体 VoiceOver / Voice Control 摘要。用户可以逐个听到按钮，却无法先快速理解“这是本地模型文件工作流、当前 artifact 状态是什么、这些动作不会联网下载或绕过 verified 门禁”。

## 目标

1. 为 `ArtifactActionPanel` 建立独立的模型文件面板辅助语义。
2. 新增 `ModelArtifactPanelAccessibilityMetadata`，暴露 label、value、hint、Voice Control 输入标签和稳定 identifier。
3. value 必须合并当前 artifact availability、validation summary、四个可用动作、Files 手动导入、本地扫描和模拟暂存边界。
4. hint 必须明确面板只管理本地模型文件工作流；不会联网下载模型权重，不会启动真实 runtime，不会发送云端服务，也不会绕过 artifact verified 门禁。
5. `ArtifactActionPanel` 接入 `.accessibilityElement(children: .contain)`，保留四个按钮的单独焦点和既有 metadata。
6. 新增 XCTest 覆盖 missing / staged / verified 面板 value、hint、input labels、identifier 和不泄露动态 validation summary 到 identifier。
7. 同步 `AGENTS.md`、`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 非目标

- 不修改 `ModelCatalog`、`ModelArtifactStore`、`LocalArtifactValidator`、`LocalRuntimePlanner` 或部署状态流。
- 不改变四个 artifact 操作按钮的 label/value/hint/input labels/identifier。
- 不改变模型页布局、视觉样式、颜色或断点。
- 不新增 UI Test target。
- 不下载模型权重，不接入云端推理，不接入真实 runtime，不创建原生 macOS target。
- 不绕过 artifact `verified` 门禁。

## 当前架构依据

- `LocalGemma/ContentView.swift`
  - `ArtifactActionPanel` 展示“下载模型 / 卸载模型 / 扫描本地 / 导入文件”四个模型文件操作入口。
  - `ModelDeploymentControlAccessibilityMetadata.ArtifactAction` 已为四个按钮生成独立语义。
  - 本轮只补面板级摘要，不改按钮行为。
- `LocalGemma/AppState.swift`
  - `ArtifactValidationResult` 提供 `availability` 和 `summary`。
- `LocalGemmaTests/LocalGemmaTests.swift`
  - 当前测试基线为 62 个 XCTest 方法。

## 实现步骤

1. 在 `ContentView.swift` 中新增 `ModelArtifactPanelAccessibilityMetadata`。
2. 元数据建议包含：
   - `static let label`
   - `static let hint`
   - `static let identifier`
   - `static let inputLabels`
   - `static func value(validation:)`
3. `value(validation:)` 复用 `ModelDeploymentControlAccessibilityMetadata.availabilityDescription(for:)` 或等价现有描述，避免另起一套状态文案。
4. `ArtifactActionPanel` 在 `.panelStyle()` 后接入 `.accessibilityElement(children: .contain)` 和面板 metadata，保留子按钮焦点。
5. 在 `LocalGemmaTests.swift` 新增 `testModelArtifactPanelExposesAccessibilityMetadata`，测试函数数从 62 增加到 63。

## 风险和禁止项

- 风险：面板级语义可能与四个按钮重复；本轮通过 `.contain` 保留按钮焦点，把面板定位为工作流摘要。
- 风险：identifier 必须稳定，不能依赖 validation summary、模型名、prompt 或动态文件名。
- 禁止下载模型权重、云端推理、绕过 verified 门禁。
- 禁止把未跑测试写成已通过。
- 禁止验收旧 run 或旧 artifact。
