# v2.0 / 模型选择器辅助语义

## 版本号与分配依据

- 版本号：`v2.0`
- 分配依据：`update_log.md` 当前最新正式版本为 `v1.9 / 模型部署控件辅助语义`，本轮在 Agent X 总目标“持续优化 UI、Mac、iPad 版体验”下继续递增到 `v2.0`。

## 背景

项目已经为工作区选择、会话选择、composer 输入框、提示词分类筛选和模型部署控件建立了可测试的辅助语义 metadata。模型页的 `ModelSelectorPanel` 是 Mac Catalyst、iPad 大屏和 Voice Control 用户切换模型的入口，但当前 `Picker` 只依赖可见 label 和菜单内容，没有统一的 `accessibilityLabel`、`accessibilityValue`、`accessibilityHint`、Voice Control 输入标签和稳定 identifier。

本轮只补模型选择器入口的辅助语义，让“当前选中哪个模型、artifact 状态、部署状态、候选数量、切换动作边界”可被辅助技术稳定读取；不改变模型选择 binding、`ModelCatalog.select`、artifact 校验、部署状态或 runtime 行为。

## 目标

1. 扩展 `ModelDeploymentControlAccessibilityMetadata`，为模型选择器 `Picker` 提供可测试 metadata。
2. 为 `ModelSelectorPanel` 的 `Picker` 添加 label、value、hint、input labels 和 identifier。
3. `value` 必须说明当前模型名、参数量/量化、候选数量、artifact availability 和部署状态。
4. `hint` 必须说明这是切换当前模型的入口，不下载模型权重，不启动真实 runtime，也不绕过 verified 门禁。
5. 新增纯 XCTest 锁住模型选择器 metadata，测试函数数从 44 增加到 45。
6. 同步 README、`md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`AGENTS.md` 和 `update_log.md` 中的当前能力、测试数量和验证记录。

## 非目标

- 不修改 `ModelCatalog`、`ModelArtifactStore`、`LocalArtifactValidator`、`LocalRuntimePlanner`、`InferenceEngine` 或 runtime 行为。
- 不改变 `selectedModelID` binding、模型切换结果、部署按钮、artifact 操作按钮或布局断点。
- 不改变 `ModelLibraryLayoutMode`、`WorkspaceLayoutMode`、`SessionSidebarLayoutPolicy` 或 command menu 映射。
- 不新增 UI Test target，不做模型页视觉重设，不创建原生 macOS target。
- 不下载模型权重，不接入云端推理，不绕过 artifact `verified` 门禁。

## 当前架构依据

- `md/flow/flow.md`：模型页由 `ModelCatalog` 管理模型选择、artifact validation 和部署状态；`ContentView` 负责 UI，不应绕过状态层直接修改核心状态。
- `LocalGemma/ContentView.swift`：
  - `ModelDeploymentControlAccessibilityMetadata` 已为部署电源和 artifact 操作按钮提供 metadata。
  - `ModelSelectorPanel` 持有 `models`、`selectedModelID`、`selectedModel`、`validation` 和 `deploymentState`，正好能生成选择器辅助语义。
  - `ModelSelectorPanel` 的 `Picker(selection: $selectedModelID)` 当前没有显式辅助 metadata。
- `LocalGemma/AppState.swift`：
  - `LocalModel` 提供 `name`、`parameterCount`、`quantization`、`artifactManifest` 和 `deploymentProfile`。
  - `ArtifactValidationResult.availability` 和 `ModelDeploymentState.localizedTitle` 可用于生成稳定 value。
- `LocalGemmaTests/LocalGemmaTests.swift`：
  - `testModelDeploymentControlsExposeAccessibilityMetadata` 是本轮 metadata 测试参考。

## 实现步骤

1. 在 `LocalGemma/ContentView.swift` 的 `ModelDeploymentControlAccessibilityMetadata` 中新增模型选择器 API：
   - `modelSelectorLabel`
   - `modelSelectorIdentifier`
   - `modelSelectorValue(selectedModel:validation:deploymentState:modelCount:)`
   - `modelSelectorHint(modelCount:)`
   - `modelSelectorInputLabels(selectedModel:)`
2. 修改 `ModelSelectorPanel` 的 `Picker`：
   - 添加 `.accessibilityLabel(...)`
   - 添加 `.accessibilityValue(...)`
   - 添加 `.accessibilityHint(...)`
   - 添加 `.accessibilityInputLabels(...)`
   - 添加 `.accessibilityIdentifier(...)`
3. 保持现有可见 UI、`.pickerStyle(.menu)`、`.tint`、frame、padding、background 和 overlay 不变。
4. 新增 XCTest，例如 `testModelSelectorExposesAccessibilityMetadata`：
   - label 等于“选择当前模型”或等价稳定文案。
   - identifier 等于稳定字符串，例如 `model-selector-picker`。
   - value 包含当前模型名、参数量、量化、候选数量、artifact 状态和部署状态。
   - hint 包含“切换当前模型”，并包含“不下载模型权重”或“不启动真实 runtime / verified 门禁”边界。
   - input labels 包含“选择模型”和当前模型名。
5. 同步文档：
   - `AGENTS.md`：一句话总览和架构边界加入模型选择器辅助语义。
   - `md/flow/flow.md`：UI 工作区和核心模块加入模型选择器辅助语义。
   - `md/flow/flowchart.md`：模型页节点注明选择器/部署控件辅助语义。
   - `md/test/test.md`：测试数量从 44 更新到 45，覆盖范围和 Stage Regression 触发条件加入模型选择器辅助语义。
   - `README.md`：当前范围、运行说明、模型状态、验证章节和历史记录同步 v2.0。
   - `update_log.md`：当前核心测试改为 45，追加 v2.0 记录，写明本地轻量检查命令与结果。

## 风险与禁止项

- 风险：选择器 hint 如果写得过强，可能误导用户认为切换模型会下载或启动真实权重；测试必须锁住“不下载模型权重”和 verified 门禁边界。
- 风险：value 如果缺少当前模型名或候选数量，Mac/iPad 菜单式 Picker 对 VoiceOver 用户不够可预测。
- 风险：identifier 如果不稳定，后续 UI Test 或辅助技术命令会退化。
- 禁止下载模型权重、云端推理、绕过 verified 门禁、改真实 runtime、引入新第三方依赖、扩大到无关视觉重构。
- 禁止伪造本地或云端测试结果；没有跑的测试必须明确说明。
