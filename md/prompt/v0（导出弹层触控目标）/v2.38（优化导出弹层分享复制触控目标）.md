# Agent B 提示词：v2.38 优化导出弹层分享复制触控目标

## 版本号和版本依据

- 版本号：v2.38。
- 版本分配依据：人工明确指定本轮版本为 v2.38；当前 `update_log.md` 最新正式版本为 v2.37，本轮按顺序继续推进。
- 总目标 X 当前小目标：继续自主优化 UI、Mac 和 iPad 体验，为导出弹层分享/复制动作补齐独立 44pt 触控目标策略。

## 背景

v2.2 已为导出弹层的分享 Markdown、文本兜底和复制全文动作建立 `ExportSessionActionAccessibilityMetadata`，覆盖 label、value、hint、Voice Control input labels 和稳定 identifier。当前 `ExportSessionView` 中底部分享按钮和复制按钮主要依赖文本、padding 和样式形成高度，toolbar 分享入口则是 icon-only `ShareLink`，没有独立 LayoutPolicy 与 XCTest 锁住最小命中尺寸。iPhone、iPad 和 Mac Catalyst 上导出弹层都可能通过底部按钮或 toolbar 分享入口触发分享/复制，本轮需要把这些入口的 44pt 触控目标收敛到可测试策略。

## 目标

1. 新增 `ExportSessionActionLayoutPolicy`，集中定义导出弹层分享/复制动作的 44pt 最小触控目标。
2. 在 `ExportSessionView` 中复用该策略，保证：
   - 底部分享 Markdown 文件按钮至少 44pt 命中高度。
   - 底部文本分享兜底按钮至少 44pt 命中高度。
   - 底部复制全文按钮至少 44pt 命中高度。
   - toolbar 分享入口至少 44x44 命中尺寸。
3. 保留现有 `ExportSessionActionAccessibilityMetadata`、ShareLink 文件优先/文本兜底选择、复制全文剪贴板写入和导出弹层本地/云端边界说明。
4. 新增 XCTest 锁住导出弹层分享/复制触控目标策略。
5. 同步 README、测试规范、核心流程文档、流程图、AGENTS 和 update_log。

## 非目标

- 不改变 `ExportPayload`、`InferenceEngine.exportActiveSessionText` 或 `exportActiveSessionMarkdownFile`。
- 不改变 `ExportPayload.existingFileURL` 的文件存在性判断。
- 不改变 ShareLink 优先分享 Markdown 文件、文件不存在时分享文本的选择逻辑。
- 不改变 `UIPasteboard.general.string = payload.text` 写入行为。
- 不改变导出弹层辅助语义文案、Voice Control input labels 或 identifier，除非为复用 layout policy 做必要且等价的整理。
- 不改变会话导出入口、`SessionBarActionLayoutPolicy`、会话 command menu、composer focus 或会话状态流。
- 不下载模型权重，不接云端推理，不绕过 artifact verified 门禁。
- 不新增原生 macOS target、完整 UI Test target、大体积测试数据或无关重构。

## 实现步骤

1. 开始实现前同步远端并确认工作区：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short --branch`
2. 在 `ExportSessionActionAccessibilityMetadata` 附近新增 `ExportSessionActionLayoutPolicy`。建议保持小而可测：
   - `minimumTouchTarget: CGFloat = 44`
   - `bottomButtonMinHeight: CGFloat = minimumTouchTarget`
   - `toolbarButtonSize: CGFloat = minimumTouchTarget`
   - `enum Presentation: CaseIterable { case bottomShare, bottomCopy, toolbarShare }` 或等价命名。
   - `usesMinimumTouchTarget(for:)` 返回对应入口是否达到最小触控目标。
   - 如使用 action 映射，需明确 `.shareMarkdownFile` 和 `.shareTextFallback` 都覆盖底部分享与 toolbar 分享，`.copyFullText` 只覆盖底部复制。
3. 在 `ExportSessionView` 中复用策略：
   - 底部 Markdown `ShareLink` label 使用 `.frame(maxWidth: .infinity, minHeight: ExportSessionActionLayoutPolicy.bottomButtonMinHeight)` 或等价实现；保留现有视觉样式、文本和辅助语义。
   - 底部文本兜底 `ShareLink` label 使用同一底部分享最小高度。
   - 底部复制全文按钮 label 使用同一底部按钮最小高度；保留 `didCopyText` 状态和剪贴板写入。
   - toolbar Markdown/text `ShareLink` label 使用 `.frame(width: ExportSessionActionLayoutPolicy.toolbarButtonSize, height: ExportSessionActionLayoutPolicy.toolbarButtonSize)` 或等价 icon button 命中尺寸；保留 icon-only 视觉和现有 toolbar accessibility identifier 后缀。
4. 新增 `testExportSessionActionLayoutPolicyMaintainsTouchTargets`：
   - 断言 `minimumTouchTarget == 44`。
   - 断言 `bottomButtonMinHeight >= minimumTouchTarget`。
   - 断言 `toolbarButtonSize >= minimumTouchTarget`。
   - 断言所有 policy presentation/action 均满足最小触控目标。
   - 断言分享动作覆盖 Markdown 文件和文本兜底，复制动作只归底部复制入口；避免误把复制动作暴露为 toolbar 分享。
5. 更新文档：
   - `AGENTS.md`：项目总览、核心架构边界和需要同步测试的清单加入 `ExportSessionActionLayoutPolicy` / 导出弹层分享复制 44pt 触控目标。
   - `md/flow/flow.md`：UI 工作区、分享流程和核心状态对象列表加入新策略说明，明确只影响命中尺寸，不改变导出内容、ShareLink 选择、剪贴板或辅助语义。
   - `md/flow/flowchart.md`：会话导出与分享流、UI 布局说明中补充 44pt 触控目标。
   - `md/test/test.md`：当前测试函数数从 81 更新为 82；Stage Regression 触发条件、Full 可视检查和当前基线加入导出弹层分享/复制触控目标。
   - `README.md`：当前范围、运行方式、推理页导出说明、已完成验证更新到 v2.38 和 82 个测试函数。
   - `update_log.md`：追加 v2.38 记录，写清核心变更、关键文件、验证结果和遗留事项。

## 风险

- toolbar `ShareLink` 的 icon-only label 可能在不同平台默认命中区不一致；必须用显式 frame 或等价布局保证 44x44。
- 底部按钮若只依赖 padding，Dynamic Type 或样式调整后可能回退；必须改为 policy 常量驱动的最小高度。
- 复制按钮有 `didCopyText` 状态切换，新增 frame 不能影响复制成功后的视觉文本和辅助语义。
- 文档测试函数数、README 验证章节和 `md/test/test.md` 当前基线容易漏改；README 过期视为 bug。
